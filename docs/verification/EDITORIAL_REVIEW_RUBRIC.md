# PyWhy 출판 품질 엄격 검수 기준

> "30,000원 내고 살 만한가?" 관점에서 전수조사

## 검수 항목 (10가지)

### 1. 글쓰기 품질
- [ ] 어색한 한국어 문장 (번역체, 비문)
- [ ] 같은 의미 반복 (장황함)
- [ ] 갑자기 끊긴 흐름
- [ ] 문맥 없는 전문용어
- [ ] 일관성 없는 종결어 (~합니다 / ~한다 혼용)

### 2. 설명의 명확성
- [ ] "이게 무슨 말이지?" 모호한 부분
- [ ] 너무 빨리 진행되는 점프
- [ ] 비유가 본 개념을 가리는 경우
- [ ] 근거 없는 주장 ("일반적으로", "보통")
- [ ] 약속한 것을 안 다루고 넘어감

### 3. 코드 예제 품질
- [ ] 실제로 실행되는가
- [ ] 출력 주석이 정확한가
- [ ] 너무 길거나 너무 단순함
- [ ] 학습 목표와 일치하는가
- [ ] 코드 안의 한국어 주석이 실용적인가

### 4. 페이지 시각 품질 (PDF)
- [ ] 어색한 페이지 분리 (제목만 있는 페이지, 코드 블록 분단)
- [ ] 표가 페이지를 넘어가는데 헤더 반복 안됨
- [ ] 너무 빈 페이지 / 너무 빽빽한 페이지
- [ ] 한 줄 차이로 다음 페이지로 넘어가는 어색함
- [ ] 다이어그램 깨짐 / 잘림

### 5. 레벨 구분 적절성
- [ ] 초급에 갑자기 어려운 용어 등장
- [ ] 고급/전공자 차이가 모호
- [ ] 같은 내용 레벨 간 중복

### 6. 학습 흐름
- [ ] 이전 주차 내용 자연스럽게 연결
- [ ] 다음 주차 예고가 적절한 흥미 유발
- [ ] 한 주차 안에서 섹션 간 흐름

### 7. 연습문제 품질
- [ ] 학습한 내용을 실제로 검증
- [ ] 정답이 명확하고 충분히 설명됨
- [ ] 난이도 단계 적절
- [ ] 함정 문제와 단순 반복 문제의 균형

### 8. AI 섹션의 가치
- [ ] 단순 트렌드 따라가기가 아니라 실제 통찰 제공
- [ ] AI 생성 코드 검증 능력에 실제 도움

### 9. 용어 사전·정리·복습
- [ ] 정의가 정확하고 다른 곳과 일관됨
- [ ] 빠진 핵심 용어 없음
- [ ] 정리가 책 내용과 일치

### 10. 표지·전체 인상
- [ ] 표지가 신인 저자 첫 책으로 적절한가
- [ ] 판권면 정보 완비
- [ ] 참고문헌 형식 일관성
- [ ] 폰트 / 여백 / 색상 톤 일관성

## 등급 (각 항목 평가)

- 🟢 **PASS**: 출판 가능 수준
- 🟡 **MINOR**: 사소한 보강 필요 (출판 가능)
- 🟠 **MAJOR**: 명확한 수정 필요 (그대로 출판 시 부끄러움)
- 🔴 **BLOCKING**: 절대 그대로 출판 불가
