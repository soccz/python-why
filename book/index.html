03 · 왜 이 책인가

"문법 외우기"가 아니라 "사고법" 책

대부분의 파이썬 책은 문법을 가르칩니다. PyWhy는 파이썬 인터프리터가 어떻게 생각하는지를 가르칩니다. 같은 한 줄의 코드를 놓고 4단계로 분해합니다 — 그게 4단계 학습 시스템입니다.

De-sugaring 전개

for x in seq: 한 줄을 iter()·next()·StopIteration으로 분해. with는 __enter__·__exit__로. x = 1은 "상자"가 아니라 "이름표(Binding)"로.

비유의 일관성

1주차에 본 "이름표"는 9주차 함수 인자에서 다시 등장하고, 12주차 던더 메서드에서 또 등장합니다. 비유 한 번 잡으면 그 다음 챕터들이 자연스럽게 연결됩니다.

공식 문서 1차 자료

모든 기술적 주장은 Python 3.14 공식 문서 / PEP / CPython 소스 코드 / IEEE 754에서 인용·검증. 본문에 각주, 챕터 말미에 참고문헌 — 학술 논문 수준의 출처 표기.

AI 시대의 관점

바이브 코딩 시대에 AI가 코드를 어떻게 만드는지의 메커니즘과 한계를 통합. AI가 만든 코드를 읽고 판단할 수 있는 능력이 진짜 실력입니다.

04 · 4단계 학습 시스템

같은 책을 4번, 다른 깊이로 읽는다

각 주제마다 4단계 박스가 있습니다. 처음 읽을 땐 초급만 따라가도 OK. 나중에 다시 펼쳤을 때 같은 주제가 다른 깊이로 보입니다.

● 초급

실생활 비유 + 실행

"포스트잇"으로 변수, "자판기/배출구"로 Iterator. 초등~중학생 이해 가능.

● 중급

동작 원리 + 오류 상황

왜 이 코드가 깨지는가, 인터프리터가 실제로 어떻게 처리하는가. 고등~대학 비전공.

● 고급

원서 용어 + 내부 메커니즘

LEGB Rule, Iterator Protocol, MRO. Python 공식 문서 용어 그대로.

● 전공자

CPython 구현 + CS 이론

Part A: typeobject.c, longobject.c 같은 C 소스 인용. Part B: IEEE 754, PEG 파서, GIL 같은 CS 이론.

  • 이미 CPython 컨트리뷰터이시면 — 전공자 레벨도 익숙한 내용일 수 있음. 단 4단계 시스템의 교육적 정리 자체에 가치가 있다고 판단되시면 OK.
08 · FAQ

자주 묻는 질문

환불 정책은 어떻게 되나요?
Gumroad의 기본 환불 정책(구매 후 14일 내 환불 요청 가능)을 따릅니다. 이미 다운로드한 PDF는 환불 후에도 유효하지만, 양심에 맡기는 디지털 책의 특성을 이해해 주시기 바랍니다.
업데이트는 어떻게 받나요?
v1.x 정오표 패치 및 개정판은 구매자 무료로 제공됩니다. 정오표는 errata 페이지에 게시되며, 새 빌드가 나오면 Gumroad 라이브러리에서 다운로드 가능합니다.
물리 책으로도 나오나요?
현재는 PDF only입니다. 1쇄 100부 판매 이후 ISBN 발급 + 인쇄 출판 검토 예정. 인쇄가 결정되면 PDF 구매자에게 우선 안내합니다.
팀/기업 라이선스가 있나요?
현재 단일 PDF만 판매합니다. 팀 라이선스(5인/10인 패키지)는 출시 후 수요를 보고 결정합니다. 사내 교육 목적의 대량 구매 문의는 사이트 정오표 채널로 연락 주시기 바랍니다.
Python 3.15가 나오면 어떻게 되나요?
Python 3.15는 2026년 10월 예정입니다. PyWhy v2 (3.15 기준)는 별도 출시 예정이지만, v1 구매자에게 50% 할인 쿠폰 제공을 검토하고 있습니다.
정말 1,010쪽이나 필요한가요?
4단계 학습 시스템 때문입니다. 한 주제를 초급부터 전공자까지 4번 다른 깊이로 다루기 때문에 분량이 큽니다. 처음 읽을 땐 초급만 골라 읽으셔도 OK — 나중에 다시 펼치면 같은 책의 다른 챕터를 새로 읽는 효과가 있습니다.
무료 HTML 강의와 PDF의 차이는?
콘텐츠는 같습니다. PDF는 (1) 1,010쪽 단권 정리 (2) 색인 378개 + 책갈피 (3) 모든 인용 hyperlink (4) 오프라인/인쇄 (5) 다크 테마 코드블록 등 정리된 판본의 가치를 제공합니다.
저자 H0NG는 누구인가요?
자가 출판하는 익명 저자입니다(가운데 0은 숫자 영). 이 책의 신뢰는 저자 인지도가 아니라 출처 검증의 투명성(editorial 리포트 공개)으로 증명합니다. 모든 기술적 주장은 1차 출처와 대조 가능합니다.
09 · 저자의 말

왜 이 책을 썼는가

"파이썬을 5년 가르치면서 같은 질문을 매번 받았습니다 — 'for문은 알겠는데 왜 동작하는지 모르겠어요.'"

문법은 검색하면 나옵니다. AI도 문법은 잘 씁니다. 하지만 왜를 설명하는 자료는 흩어져 있고, 깊이는 영문 공식 문서에 가두어져 있습니다.

이 책은 한 줄의 코드가 인터프리터를 거쳐 CPU에 닿기까지의 사고 과정을 한국어로 정리한 결과입니다. 4단계 학습 시스템은 같은 책을 평생 옆에 두고 다시 펼치는 책으로 만들고 싶어 도입했습니다.

AI가 코드를 만드는 시대일수록, 그 코드를 읽고 판단할 수 있는 사람의 가치가 커집니다. 이 책이 그 길의 첫 걸음이 되길 바랍니다.
