ALGO 19

그리디 알고리즘 총정리
매 순간 최선이 언제 전역 최선이 되는가

1951년 MIT, 정보이론 수업의 기말과제. 학생 David Huffman은 기말시험 대신 교수 Fano가 낸 "최적 prefix code" 문제를 택합니다. 증명을 포기하려던 순간 "빈도순 이진 트리"를 떠올리고 — 지금도 ZIP, JPEG, MP3에 들어가는 알고리즘이 태어남.
그리디는 단순해 보이지만 "언제 작동하는가"는 수학의 아름다운 질문. Greedy-Choice Property + Optimal Substructure가 만나는 곳, Matroid라는 우아한 이론.
핵심 질문: "왜 활동 선택과 Huffman은 그리디가 최적인데, 0/1 배낭은 안 되는가?"

01 — 이번 주 이야기: Huffman의 MIT 기말과제

"The basic idea of the Huffman method came to me as I was walking home from class. I had been thinking about what Fano had told us, but I really wanted to find my own approach."
— David A. Huffman, 회고 (Scientific American, 1991)

1951 — MIT, Robert M. Fano의 정보이론 수업

Fano 교수가 학생들에게 선택권을 줌 — 기말시험을 보거나, 아니면 "최소 redundancy prefix code 찾기" 문제를 term paper로. 당시 MIT 대학원생이던 David Huffman은 term paper를 선택.

사실 Fano 자신은 Shannon-Fano coding(1949)이라는 기법을 이미 만들었지만 최적은 아님. Huffman도 몇 달간 실패했다가, 포기하고 시험 공부로 돌아가기 직전 "빈도순 이진 트리 상향식 병합"을 떠올림.

1952 — "A Method for the Construction of Minimum-Redundancy Codes"

발표 논문: Proc. IRE 40:1098-1102 (Sept 1952). 4페이지짜리 짧은 논문이지만 최적성 증명 포함. Huffman의 방법은 Shannon-Fano를 능가했고, 그 후 데이터 압축의 표준이 됨.

응용 범위: ZIP, gzip, PKZIP, JPEG, MP3, PNG, HTTP/2 HPACK — 모두 Huffman 또는 변형 사용. 70년이 넘도록 현역.

그리디의 철학 — 매 순간 최선

그리디(greedy) 알고리즘의 공통 구조:

  1. 현재 상태에서 지역적으로 가장 좋아 보이는 선택.
  2. 그 선택을 확정, 되돌아보지 않음.
  3. 반복.

놀랍게도 이게 전역 최적이 되는 경우가 많음: Dijkstra(Algo 15), Prim·Kruskal(Algo 17), Huffman, 활동 선택. 하지만 0/1 배낭엔 안 됨. 언제 작동하는가? — 이번 주의 핵심 질문.

그리디의 계보 — Whitney 1935 Matroid 이론, Borůvka 1926부터 시작된 MST 그리디, Huffman 1952 최적 압축, 그리고 1971년 Rado-Edmonds 정리가 "그리디가 최적인 구조 = Matroid"를 증명.

02 — 그리디란 — 매 순간 최선

초급

동전 거스름돈 비유

1300원을 거슬러줘야 한다. 한국 동전 {500, 100, 50, 10}. 어떻게?

  • 가장 큰 동전(500)부터 쓰기: 500·2 = 1000 → 남은 300.
  • 다음 큰 동전(100): 100·3 = 300 → 남은 0. 끝.
  • 총 5개 동전. 매 순간 "가장 큰" 동전을 욕심내는 전략 = 그리디.

한국/미국 동전 체계에선 그리디가 항상 최적. 하지만 동전이 {1, 3, 4}라면? 6원 거슬러주기:

  • 그리디: 4 + 1 + 1 = 3개.
  • 최적: 3 + 3 = 2개.

→ 그리디는 항상 작동하지 않는다. 언제 작동하느냐가 이번 주의 핵심.

중급

그리디 알고리즘의 형식

  1. 최적화 문제: 집합에서 특정 조건을 만족하는 부분집합(또는 순열)을 골라 목적함수 극대/극소.
  2. 지역적 선택 규칙: 현재 상태에서 "가장 좋아 보이는" 요소를 고르는 기준.
  3. 반복: 선택 후 문제가 작아지도록. 되돌아가지 않음(irrevocable).

핵심 조건 두 가지:

  • Greedy-Choice Property: "첫 번째 선택은 어떤 최적해에 포함됨". (전역 최적이 지역 선택과 호환)
  • Optimal Substructure: "선택 후 남은 부분 문제도 같은 형태 + 그 최적해가 전체 최적의 일부". (DP와 공유)

03 — 활동 선택 문제 (CLRS §15.1)

중급

문제 — 겹치지 않는 최대 회의 예약

회의실 하나. N개의 회의 요청 — 각각 시작 시각 si, 종료 시각 fi. 서로 겹치지 않게(한 번에 하나) 최대 몇 개를 잡을 수 있는가?

  • 활동 i와 j가 호환: fi ≤ sj 또는 fj ≤ si.
  • 목표: 호환되는 활동 부분집합의 최대 크기.
  • Brute force: 2N 부분집합 → 불가능.
고급

그리디 전략 — "가장 일찍 끝나는 것"

직관: 회의를 일찍 끝내면 남은 시간이 많다 → 더 많은 회의 여유.

  1. 활동을 종료 시각 오름차순으로 정렬.
  2. 가장 먼저 끝나는 활동 선택.
  3. 선택한 활동의 종료 이후 시작하는 활동들 중 가장 먼저 끝나는 것 선택.
  4. 반복.

왜 "일찍 끝나는" 기준이 정답인가: 어떤 최적해 O가 있을 때, O의 첫 활동을 "가장 일찍 끝나는" 활동으로 교체해도 여전히 |O|개 호환. Exchange argument.

5단계 활동 선택 애니메이션: 종료 시각 순으로 정렬 → 가장 일찍 끝나는 a1 선택 → 4 이후 시작하는 것 중 가장 일찍 끝나는 a4 → 반복. 최종 {a1, a4, a6} = 3개. Exchange argument로 최적성 증명.

04 — 활동 선택 Python 구현

고급

정렬 + 한 번 순회

activity_selection.py
def activity_selection(activities):
    """activities: [(start, finish, name), ...]. 반환: 선택된 활동 리스트."""
    # 종료 시각 오름차순 정렬
    acts = sorted(activities, key=lambda x: x[1])

    selected = [acts[0]]
    last_finish = acts[0][1]

    for s, f, name in acts[1:]:
        if s >= last_finish:
            selected.append((s, f, name))
            last_finish = f

    return selected


# 테스트
activities = [
    (1, 4, 'a1'), (3, 5, 'a2'),
    (0, 6, 'a3'), (5, 7, 'a4'),
    (6, 10, 'a5'), (8, 11, 'a6'),
]
print(activity_selection(activities))
# [(1, 4, 'a1'), (5, 7, 'a4'), (8, 11, 'a6')]
  • 복잡도: 정렬 O(n log n) + 순회 O(n) = O(n log n).
  • 정렬 이후에는 DP 없이 단 한 번 순회. DP로 풀면 O(n²) — 그리디가 훨씬 효율적.
  • 변형:
    • 가장 먼저 시작하는 것 기준? ✗ (a3처럼 긴 활동 하나만 차지할 수 있음).
    • 가장 짧은 것 기준? ✗ (짧지만 양쪽 긴 활동 2개를 쪼개는 경우 반례 가능).
    • 겹치는 활동이 가장 적은 것 기준? ✗ (반례 만들기 가능).
    • 오직 "가장 일찍 끝나는 것"만 정답.
📖 원서 인용 — CLRS 4판, §15.1 "An activity-selection problem"

"Activities are sorted in monotonically increasing order of finish time... The greedy choice is always to pick the activity with the earliest finish time. Why? Because it leaves the resource available for as many of the activities that follow it as possible."

05 — Greedy-Choice Property — 언제 성립하는가

고급

두 조건

그리디가 최적인 문제는 일반적으로 두 속성을 모두 가짐:

  1. Greedy-Choice Property (그리디 선택 속성): 전역 최적해가 지역적 최선 선택을 포함. 즉 "첫 선택을 그리디로 해도 최적을 놓치지 않음".
  2. Optimal Substructure (최적 부분구조): 선택 후 남은 부분 문제의 최적해가 전체 최적해의 일부. (DP와 공유되는 속성)

DP vs 그리디:

  • DP: 모든 선택지를 보고 그중 최선.
  • 그리디: 탐색 없이 현재 최선만 확정.
  • 그리디는 DP의 특수 케이스 — "탐색이 불필요한 경우".
전공자

Greedy-Choice 증명 — Exchange Argument의 표준 템플릿

  1. 임의 최적해 O를 가정.
  2. 그리디 선택 g가 O에 포함되지 않는다면, O의 어떤 요소 x를 g로 교환.
  3. 교환 후에도 O'는 여전히 valid이고 (제약 조건 유지) 목적함수 값이 더 나빠지지 않음.
  4. → O' 또한 최적. 그리디 선택 g가 최적해의 일부. □

활동 선택에서의 적용: O의 첫 활동 x를 그리디 선택 g1(가장 일찍 끝나는)로 교체. f(g1) ≤ f(x)라 뒤따르는 활동들이 여전히 호환. O' = (O − {x}) ∪ {g1}도 동일 크기, 유효.

06 — Huffman 코딩 — 아이디어

중급

문제 — 가변 길이 prefix code

문자열을 이진으로 압축. 문자마다 비트 코드를 할당:

  • 고정 길이 코드: 예) ASCII 8비트. 빈도와 무관하게 모두 같은 길이.
  • 가변 길이 코드: 자주 쓰는 문자엔 짧은 코드. 평균 길이가 짧아짐.
  • Prefix code: 어떤 코드도 다른 코드의 prefix가 아님 — 그래야 유일 해독 가능.

목표: 문자 빈도가 주어질 때 평균 비트 수를 최소화하는 prefix code.

고급

Huffman의 아이디어 — 상향식 트리 합성

  1. 각 문자를 빈도를 가진 leaf 노드로 시작.
  2. 최소 빈도 두 노드 x, y를 합쳐 새 내부 노드 z. freq(z) = freq(x) + freq(y). z의 왼쪽=x, 오른쪽=y.
  3. z를 힙에 다시 넣고 반복.
  4. 하나만 남을 때까지 → 이것이 Huffman 트리. leaf까지의 경로(0/1)가 각 문자의 코드.

왜 "가장 적게 나오는 두 개를 합치는 것"이 최적인가: 빈도가 낮은 문자가 트리에서 더 깊이 있어야 총 비트 수가 최소. 합치면 둘 다 같은 내부 노드의 자식이 되어 코드 길이가 같아짐. Exchange argument로 증명.

5단계 Huffman 트리 구축: heap의 최소 두 노드를 반복 병합. 자주 나오는 F가 root 근처(1비트), 드문 A/B가 가장 깊은 leaf(4비트). 평균 2.24비트 — 고정 3비트 대비 25% 압축.

07 — Huffman Python 구현 (heapq)

전공자

힙 기반 구현

huffman.py
import heapq
from collections import Counter

class Node:
    def __init__(self, freq, char=None, left=None, right=None):
        self.freq = freq
        self.char = char
        self.left = left
        self.right = right

    def __lt__(self, other):
        return self.freq < other.freq    # heap 비교 기준


def huffman(text):
    """문자열 → Huffman 코드 딕셔너리."""
    if not text: return {}
    freq = Counter(text)

    # 모든 문자를 leaf로 heap에 삽입
    heap = [Node(f, c) for c, f in freq.items()]
    heapq.heapify(heap)

    # 트리 구축: n-1번 병합
    while len(heap) > 1:
        left = heapq.heappop(heap)
        right = heapq.heappop(heap)
        merged = Node(left.freq + right.freq, left=left, right=right)
        heapq.heappush(heap, merged)

    root = heap[0]

    # DFS로 코드 추출
    codes = {}
    def walk(node, code):
        if node.char is not None:    # leaf
            codes[node.char] = code or "0"    # 단일 문자 엣지케이스
            return
        walk(node.left, code + "0")
        walk(node.right, code + "1")

    walk(root, "")
    return codes


# 테스트
text = "ABRACADABRA"
codes = huffman(text)
print(codes)
# 예: {'A': '0', 'B': '100', 'R': '101', 'C': '110', 'D': '111'}
encoded = "".join(codes[c] for c in text)
print(len(encoded), "bits")    # 23 bits (11자 × 약 2.09비트 평균, 고정 3비트=33비트 대비 30% 압축)
  • heapq + Node(__lt__): 힙이 freq 기준 비교하도록 __lt__ 정의. 동률이면 삽입 순서 유지 (Python heapq가 안정적).
  • 복잡도: 힙 초기화 O(n), 병합 n-1번 × O(log n) = O(n log n).
  • 단일 문자 케이스: codes[c] = code or "0" — 텍스트에 문자가 하나뿐이면 코드 길이 0이 되므로 강제로 "0".
  • 디코딩: 인코딩된 비트열을 root부터 따라가며 leaf에 도달하면 문자 출력, root로 돌아가 반복. prefix code라 고유 해독.

08 — Huffman 최적성 증명

전공자

Lemma 1 — Greedy-Choice Property

Claim: 알파벳 C에서 빈도가 가장 적은 두 문자 x, y는 어떤 최적 prefix code 트리의 최대 깊이에서 형제로 있을 수 있다.

증명 스케치: 임의 최적 트리 T에서 가장 깊은 두 형제 leaf를 a, b라 하자. WLOG freq(a) ≤ freq(b) 그리고 freq(x) ≤ freq(y).

  1. x와 a를 교환해 T'를 얻음. 비용 변화: (freq(x) − freq(a)) × (depth(a) − depth(x)). freq(x) ≤ freq(a), depth(a) ≥ depth(x)이므로 ≤ 0. T'는 여전히 최적.
  2. 마찬가지로 T'에서 y와 b를 교환해 T''. T''도 최적이고, 이제 x, y가 가장 깊은 형제. □
전공자

Lemma 2 — Optimal Substructure

x, y를 합친 새 문자 z (freq(z) = freq(x) + freq(y))를 도입해 C' = C − {x, y} ∪ {z}. T'가 C'의 최적 트리이면, T' 속 z를 x, y의 부모로 대체한 T는 C의 최적 트리.

증명 스케치: |T| = |T'| + freq(x) + freq(y) (z의 깊이가 1 증가하는 비용). 최적성 유지.

전공자

Theorem — Huffman 알고리즘은 최적

Lemma 1(Greedy-Choice) + Lemma 2(Optimal Substructure) → 귀납법. Huffman의 매 병합 선택이 최적 트리의 일부. □

📖 원서 인용 — CLRS 4판, Theorem 15.5 (Huffman 최적성)

"Procedure HUFFMAN produces an optimal prefix code."

09 — 분수 배낭 — 그리디 OK

중급

문제

배낭 용량 W, 아이템 n개 — 각 아이템 i는 가치 vi, 무게 wi. 아이템을 쪼갤 수 있음(fractional). 총 무게 ≤ W이면서 총 가치 최대화.

그리디: 단위 무게당 가치 vi/wi 내림차순 정렬 → 차례로 담기 → 마지막에 용량 초과하면 부분 담기.

3단계 분수 배낭: v/w 큰 순서(A→B→C)로 담기. 용량 남으면 전체, 초과하면 분수(C는 20/30=2/3). 최종 가치 240. 분수 허용 덕에 그리디가 최적 — 교환 논증으로 증명.
고급

Python 구현

fractional_knapsack.py
def fractional_knapsack(items, W):
    """items: [(value, weight, name), ...]. 반환: 총 가치."""
    items = sorted(items, key=lambda x: x[0] / x[1], reverse=True)
    total_value = 0
    for v, w, name in items:
        if W >= w:
            total_value += v
            W -= w
        else:
            total_value += v * (W / w)    # 부분 담기
            break
    return total_value


print(fractional_knapsack([(60,10,'A'),(100,20,'B'),(120,30,'C')], 50))
# 240.0

10 — 0/1 배낭 — 왜 그리디 실패

고급

같은 예제, 다른 결과

같은 아이템 A(60/10), B(100/20), C(120/30), 용량 50. 아이템을 쪼갤 수 없음(0 또는 1).

  • v/w 그리디: A(w=10, 잔여 40) + B(w=20, 잔여 20) + C는 w=30 > 20 → 못 담음. v = 160.
  • 최적: B + C = 220 (w=50 정확히).
  • 그리디 160 < 최적 220 → 그리디 실패.

왜? 0/1 배낭은 greedy-choice property가 없음. "v/w 가장 큰 A를 반드시 포함"이 최적해를 보장하지 않음. A를 포기하고 B+C를 통째로 담는 게 나음.

같은 문제, 결과 다름. 분수 배낭은 A를 담는 게 최적이지만, 0/1에선 A를 버리고 B+C가 더 나음. 0/1 배낭 = NP-hard(약), DP로 O(nW) 의사다항. 그리디는 틀림.
전공자

0/1 배낭의 복잡도

  • 의사 다항(pseudo-polynomial) DP: O(nW). 무게 W가 작으면 빠름.
  • 정확한 의미의 "다항 시간 알고리즘"은 존재하지 않음 (NP-hard, 엄밀히는 weakly NP-hard).
  • FPTAS 존재: (1-ε)-근사를 O(n²/ε) 시간.

"v/w 그리디 + 남은 아이템 중 최대 가치 하나" 휴리스틱은 2-근사. 완벽한 답은 아니지만 많은 경우 빠르고 그럭저럭 좋음.

11 — Matroid — 그리디가 정답인 구조

전공자

Hassler Whitney 1935 — 선형 독립의 추상화

매트로이드 M = (S, I)는 유한 집합 S와 독립 집합족 I ⊆ 2S를 가진 구조. 세 공리:

  1. 비어있지 않음: ∅ ∈ I.
  2. 유전적(hereditary): A ∈ I, B ⊆ A ⟹ B ∈ I. (독립 집합의 부분집합도 독립)
  3. 교환 속성(exchange property): A, B ∈ I, |A| < |B| ⟹ ∃ x ∈ B − A, A ∪ {x} ∈ I. (더 큰 독립 집합의 요소 하나를 작은 집합에 추가해도 독립)

예:

  • Graphic Matroid: S = 그래프의 간선, I = 사이클 없는 부분집합(포레스트). → MST 문제.
  • Linear Matroid: S = 벡터들, I = 선형 독립 부분집합.
  • Uniform Matroid: S = 임의 집합, I = |A| ≤ k인 모든 A.
전공자

Rado-Edmonds 정리 (Rado 1957 / Edmonds 1971)

정리: 가중 매트로이드 M = (S, I, w)에서 "가장 가중치 큰 것부터 추가(가능하면)" 그리디가 항상 최대 가중치 독립 집합을 찾는다. 역도 성립 — 그리디가 항상 최적인 유전적 구조는 반드시 매트로이드.

이게 Kruskal이 MST에 작동하는 진짜 이유: 간선이 Graphic Matroid를 이루므로 Rado-Edmonds 보장.

💡 매트로이드의 의미

"그리디가 옳은가?"라는 질문은 매트로이드 검사로 환원됨. 활동 선택, Huffman은 매트로이드가 아니지만 유사한 구조(greedoid, scheduling matroid)를 가짐. 매트로이드 이론은 조합 최적화(combinatorial optimization)의 기둥.

12 — Exchange Argument — 증명 템플릿

고급

표준 템플릿

그리디 알고리즘 G가 반환하는 해 SG가 최적임을 증명하려면:

  1. 임의 최적해 SO를 가정.
  2. SG와 SO가 다르다면, 첫 번째 다른 지점을 찾음.
  3. 그 지점에서 SO의 선택을 G의 선택으로 교환.
  4. 교환 후 해 SO'가 여전히 유효하고(제약 만족), 목적함수가 더 나빠지지 않음.
  5. → SO'도 최적. 귀납/반복으로 SG가 최적. □
전공자

적용 예제들

  • Kruskal/Prim MST: Cut Property (Algo 17 §3) — 두 간선 교환이 MST 깨지 않음.
  • Dijkstra: "pop된 노드의 dist 최적" — 더 짧은 경로가 있었다면 모순.
  • 활동 선택: 최적해의 첫 활동을 "가장 일찍 끝나는 것"으로 교체.
  • Huffman: 최적 트리의 깊은 형제를 "최소 빈도 두 문자"로 교체.
  • Task Scheduling (EDF): Earliest Deadline First. 마감일 지연 최소화.

핵심: "교환해도 손해 없음"을 보이면 그리디 선택이 최적해에 반드시 포함됨. 이 논증 패턴은 그리디 정당성 증명의 사실상 유일한 방법.

13 — 근사 알고리즘 — NP-hard 탈출구

고급

그리디로 NP-hard 문제 근사

최적은 못 찾아도 "최악 케이스 보장된 비율" 이내로 찾는 것이 근사 알고리즘. 많은 경우 그리디가 표준 도구.

  • Vertex Cover (그래프의 최소 정점 덮개): 간선을 하나 잡고 양 끝점 모두 추가 → 2-근사.
  • Set Cover (집합 덮개): 가장 많은 미덮인 요소를 덮는 집합 추가 → ln(n)-근사 (Johnson 1974).
  • Metric TSP: MST 기반 DFS 순회 → 2-근사. Christofides 1976로 1.5-근사.
  • Scheduling (Makespan): 가장 빨리 끝나는 기계에 다음 작업 → 2-근사 (List scheduling).
  • 0/1 Knapsack FPTAS: 그리디 + DP scaling → (1-ε) 근사 in O(n²/ε).
전공자

왜 그리디가 근사에 특히 어울리나

  • 구현이 단순해 실무에 쓰기 쉬움.
  • "2-근사" 같은 형식 보장을 증명하기 쉬움 (exchange argument와 유사).
  • 많은 NP-hard 문제는 LP relaxation의 그리디 rounding으로 좋은 근사율 달성.

현대 알고리즘 교과서의 근사 챕터(CLRS §35, Vazirani 2001)는 대부분 그리디 기법을 중심으로 구성.

전공자

AI가 그리디를 잘못 쓰는 패턴

  • "그리디가 항상 작동한다": 0/1 배낭, 동전 교환(특정 체계) 등에 그리디 제안 → 오답.
  • 그리디 기준 선택 실수: 활동 선택에 "가장 먼저 시작" 기준 → 반례 가능.
  • 증명 없이 신뢰: 새 문제에 "대충 그리디"로 제안, exchange argument 확인 안 함.
  • 근사율 과장: 2-근사를 "거의 최적"으로 표현. 2배 차이는 실무에선 크다.

교정 질문: "exchange argument로 증명 가능한가?", "반례 만들어보자", "이 문제는 매트로이드 구조인가?"

14 — 연습문제 (12문제)

WHY 문제 1

활동 선택에서 "가장 먼저 시작하는 것" 기준이 왜 틀리나?

풀이 펼치기
답

반례: 활동 A[0,100], B[1,2], C[3,4], D[5,6]. "가장 먼저" 기준: A 선택 → 뒤에 아무것도 못 들어감 (1개). "가장 일찍 끝나는" 기준: B, C, D 모두 가능 (3개). 일찍 시작하는 게 길 수 있음.

PREDICT 문제 2

동전 {1, 5, 10, 25} (미국)에서 그리디가 항상 최적인가?

풀이 펼치기
답

예. 이 특정 동전 체계에선 그리디 최적. 증명: 모든 금액에 대해 큰 동전이 많이 들어가는 해가 적은 동전 수. 하지만 {1, 3, 4}는 아님 (6 = 4+1+1 vs 3+3). 미국 동전은 "canonical" — Pearson 2005가 다항 시간에 canonical 여부 판정 알고리즘 제시.

BUILD 문제 3

Huffman 코드의 평균 비트 수가 엔트로피 H(X)와 같은 관계?

풀이 펼치기
답

Huffman 평균 길이 L 만족: H(X) ≤ L < H(X) + 1. 즉 엔트로피 이론 하한에서 1비트 이내. 대체 방법:

  • Arithmetic coding: 더 H(X)에 근접 (분수 비트 가능).
  • 범위 코딩(range coding): 비슷한 효과.

DESUGAR 문제 4

Exchange argument로 "활동 선택 그리디" 최적성 증명.

풀이 펼치기
답

임의 최적해 O = {o1, o2, ..., ok}를 종료 시각 순으로 정렬. 그리디 선택 g1은 가장 일찍 끝나는 활동, f(g1) ≤ f(o1). O' = {g1, o2, ..., ok}을 생각. o2, ...은 f(o1) 이후 시작 → f(g1) 이후 시작 → 여전히 호환. |O'| = |O|. O'도 최적. 귀납 적용.

WHY 문제 5

0/1 배낭에 그리디가 실패하는 근본 이유는?

풀이 펼치기
답

Greedy-Choice Property 부재. "v/w 가장 큰 아이템을 포함"이 반드시 최적의 일부가 아님. A=(60,10), B=(100,20), C=(120,30), W=50에서 A 포기 → B+C=220이 나음. 분수 배낭은 "쪼갤 수 있음" 덕에 교환 논증이 작동하지만, 0/1에선 쪼갤 수 없어 깨짐. 매트로이드 구조가 아님.

BUILD 문제 6

Task Scheduling (마감일 지연 최소화) 그리디 설계.

풀이 펼치기
답

Earliest Deadline First (EDF): 마감일 오름차순 정렬, 순서대로 처리. 지연(lateness) 최대값을 최소화. Exchange argument: 두 작업 i, j (di < dj)가 뒤바뀐 순서면 교환해도 지연 안 늘어남 → EDF가 최적. CLRS Ex. 15.2-2.

PREDICT 문제 7

Huffman 트리에서 문자 N개면 내부 노드 수는?

풀이 펼치기
답

N - 1개. Huffman 트리는 full binary tree (모든 내부 노드가 자식 2개) + N개 leaf. 수학적으로 full 이진 트리의 정리: 내부 노드 수 = leaf 수 - 1. Huffman 알고리즘이 N-1번 병합하므로 직접 셀 수도 있음.

DESUGAR 문제 8

Matroid의 "교환 속성"이 왜 그리디의 정당성과 연결되나.

풀이 펼치기
답

교환 속성: 작은 독립 집합 A에 큰 독립 집합 B의 요소를 추가해도 독립 유지. Rado-Edmonds 증명: 그리디 선택 S와 임의 최적 O를 비교, 크기 차가 있으면 교환 속성으로 S에 O의 요소 추가 가능 → S가 최대라는 가정 모순. 즉 교환 속성이 exchange argument 성공의 충분 조건.

WHY 문제 9

그리디 vs DP — 언제 어느 쪽?

풀이 펼치기
답

공통: Optimal Substructure 필요. 차이점:

  • 그리디: "지금 최선 선택" 검증 없이 확정 → Greedy-Choice Property 필수. O(n log n) 정도.
  • DP: 모든 선택지 탐색, 메모이제이션. Greedy-Choice Property 불필요. 일반적으로 O(n²) 또는 O(nW).

확신 없으면 DP가 안전. 그리디가 되면 훨씬 빠름.

BUILD 문제 10

Vertex Cover의 2-근사 그리디.

풀이 펼치기
답

임의 간선 (u,v) 선택 → C에 u, v 둘 다 추가 → (u,v) 인접 간선 모두 제거 → 반복. 정당성: 선택된 간선들은 서로 disjoint → 모든 최적 vertex cover는 각 간선의 끝점 하나는 포함. 그리디는 두 끝점 모두 → 2배. 따라서 |C| ≤ 2 · OPT. CLRS §35.1.

AI가 "동전 교환 문제에 그리디" 제안. 언제 OK, 언제 위험?

풀이 펼치기
질문

WHY 문제 12

Huffman이 "최적"이라는 말의 의미 — 뭐에 대한 최적?

풀이 펼치기
답

평균 비트 수가 가장 짧은 prefix code. 중요한 제약:

  • 각 문자가 독립적으로 빈도에 따라 코드됨 (문맥 무시).
  • prefix code 조건.
  • 정수 비트 길이.

더 좋은 압축(문맥, 분수 비트)은 arithmetic coding, LZ77/LZ78 등 다른 기법.
