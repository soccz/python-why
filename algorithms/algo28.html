계산 기하 — Convex Hull, Sweep Line
점·선·면의 알고리즘적 처리

1972년 Bell Labs의 Ronald Graham(1935-2020)이 2페이지 논문 "An efficient algorithm for determining the convex hull of a finite planar set"(Info. Proc. Letters 1:132-133)을 발표. 컴퓨터 그래픽스·GIS의 기초 볼록 껍질 O(n log n). 같은 1979년 A.M. Andrew는 Graham을 단순화한 Monotone Chain을, Jon Bentley & Thomas Ottmann은 선분 교차를 sweep line으로 — 계산 기하의 황금기.
오늘날 Google Maps의 영역 계산, 게임 충돌 감지, CAD 불린 연산, 로봇 경로 계획까지 — 기하 알고리즘은 GUI의 숨은 엔진.
핵심 질문: "왜 '세 점의 방향'이라는 단일 연산이 기하 알고리즘의 원자가 되는가? 왜 sweep line이 복잡한 기하를 1차원으로 환원하는가?"

01 — 이번 주 이야기: 1972-1979 계산 기하의 탄생

"Computational geometry"라는 용어는 1975년 Shamos & Hoey의 박사 논문에서 정립되었지만, 아이디어의 씨앗은 이미 1972년부터 자라고 있었습니다 (cf. Preparata & Shamos, Computational Geometry: An Introduction, 1985).

1972 — Ronald Graham, Bell Labs

Ronald L. Graham(1935-2020)이 2페이지 논문 "An efficient algorithm for determining the convex hull of a finite planar set"(Info. Proc. Letters 1(4):132-133, 1972)을 발표. O(n log n)의 볼록 껍질(convex hull) 알고리즘.

Graham은 Paul Erdős의 공동저자로도 유명(Erdős number 1). 저글링 애호가. 수학자·CS 연구자의 전설.

1975 — Michael Shamos의 PhD

Shamos가 Yale에서 박사 학위를 받으며 "computational geometry"를 정식 분야로 정립. 볼록 껍질, 최근접 점 쌍, Voronoi 다이어그램 등. 1985년 Preparata & Shamos의 교과서가 표준이 됨.

1979 — 계산 기하의 해

  • Andrew's Monotone Chain(A.M. Andrew, Info. Proc. Letters 9(5):216-219): Graham 스캔의 단순화. x 좌표 기준 정렬 → 상·하 chain 각각 linear pass.
  • Bentley-Ottmann(IEEE Trans. Comput. C-28(9):643-647): sweep line으로 선분 교차 O((n+k) log n) (k=교차 수). Sweep line 패러다임의 대표.
  • 같은 해 Bentley의 k-d tree, range tree도 원숙기.

1987 — Fortune의 Voronoi

Steven Fortune이 "A sweepline algorithm for Voronoi diagrams"(Algorithmica 2:153-174, 1987)에서 O(n log n) sweep line 기반 Voronoi. 이전은 분할 정복(Shamos 1975). Fortune의 방법은 구현이 더 단순 + 수치적 안정성이 좋음.

1996 — Chan's Algorithm

Timothy Chan(Waterloo 교수)의 "Optimal output-sensitive convex hull algorithms in two and three dimensions"(Discrete & Comput. Geom. 16:361-368). O(n log h) — h=볼록 껍질 정점 수. n이 크고 h가 작을 때 Graham O(n log n)보다 빠름.

계산 기하의 창설기: 1972 Graham Scan → 1975 Shamos의 분야 정의 → 1979 Andrew/Bentley-Ottmann의 sweep line → 1985 교과서 → 1987 Fortune → 1996 Chan 최적. 대부분 1970년대 Bell Labs / CMU / Yale 집중.

02 — CCW Primitive — 기하의 원자

중급

세 점의 방향

세 점 A, B, C에 대해 A → B → C가 어느 방향으로 도는가? 세 가지 경우:

  • 반시계(CCW, Counter-Clockwise): 왼쪽으로 꺾음 → 양수.
  • 시계(CW, Clockwise): 오른쪽으로 꺾음 → 음수.
  • 일직선(Collinear): 꺾지 않음 → 0.

이 값은 외적(cross product) z 성분으로 계산:

ccw(A, B, C) = (B.x − A.x)·(C.y − A.y) − (B.y − A.y)·(C.x − A.x)
  • > 0: CCW (좌회전).
  • < 0: CW (우회전).
  • = 0: collinear.

이 하나의 연산이 볼록 껍질, 선분 교차, 다각형 내부 판정, 삼각형 회전 방향, Delaunay 유효성 검사 등 거의 모든 기하 알고리즘의 원자(primitive). 기하 알고리즘은 이 값의 부호 조합.

ccw.py
def ccw(A, B, C):
    return (B[0] - A[0]) * (C[1] - A[1]) - (B[1] - A[1]) * (C[0] - A[0])

print(ccw((0,0), (1,0), (1,1)))    # 1 > 0: CCW
print(ccw((0,0), (1,1), (1,0)))    # -1 < 0: CW
print(ccw((0,0), (1,1), (2,2)))    # 0: collinear
CCW primitive의 세 경우: 좌회전(양수) / 우회전(음수) / 일직선(0). 부동소수점 대신 정수 좌표 + 큰 정수 연산으로 robust하게 계산 가능 — 부정확한 비교가 기하 알고리즘 버그의 주 원인이기 때문.

03 — Graham Scan — 볼록 껍질 1972

고급

아이디어

  1. 기준점 선택: y 좌표 최소(동률 시 x 최소)인 점 P0. 반드시 볼록 껍질 위.
  2. 각도 정렬: 나머지 점들을 P0에서 본 극각(polar angle) 오름차순 정렬. O(n log n).
  3. stack 스캔: 각 점을 순서대로 보며 stack에 push. 새 점에서 top 2개와 함께 CCW 체크 — CW 또는 collinear면 top을 pop (볼록 어긋남). CCW 되면 push.
  4. 끝나면 stack이 볼록 껍질.

시간: 정렬 O(n log n) + 스캔 O(n) (각 점 최대 한 번 push/pop) = O(n log n).

graham_scan.py
def ccw(A, B, C):
    return (B[0]-A[0])*(C[1]-A[1]) - (B[1]-A[1])*(C[0]-A[0])

def graham_scan(points):
    # 1. 기준점: y 최소 (동률 시 x 최소)
    p0 = min(points, key=lambda p: (p[1], p[0]))

    # 2. 극각 정렬 (같은 각도면 거리 작은 것 먼저)
    import math
    rest = [p for p in points if p != p0]
    rest.sort(key=lambda p: (math.atan2(p[1] - p0[1], p[0] - p0[0]),
                        (p[0] - p0[0])**2 + (p[1] - p0[1])**2))

    # 3. stack 스캔
    hull = [p0]
    for p in rest:
        while len(hull) >= 2 and ccw(hull[-2], hull[-1], p) <= 0:
            hull.pop()
        hull.append(p)
    return hull


# 테스트
pts = [(0,0), (1,1), (2,2), (4,0), (3,3), (0,4)]
print(graham_scan(pts))    # [(0,0), (4,0), (3,3), (0,4)]
⚠️ atan2의 부동소수점

math.atan2는 부동소수점이라 동률 비교에 취약. 실무엔 CCW 기반 정렬이 더 robust: cmp(p1, p2) = ccw(p0, p1, p2)의 부호. 정수 좌표면 정확 판단 가능.

Graham Scan 6단계: P0=(0,0)에서 극각 정렬 → 같은 각도(π/4)의 (1,1),(2,2),(3,3)을 거리순 처리하며 stack push/pop → 볼록 껍질 4정점. 내부 점 (1,1),(2,2)는 pop으로 제거.

04 — Andrew's Monotone Chain 1979

고급

Graham의 단순화

Graham은 극각 정렬에 atan2 필요 → 부동소수점. Andrew 1979는 이를 x 좌표 lex 정렬로 대체. 정수 비교만 필요 → 완전 robust.

  1. 점들을 (x, y) lex 순으로 정렬.
  2. 하단 체인: 왼쪽→오른쪽으로 각 점 push. CCW ≤ 0이면 pop.
  3. 상단 체인: 오른쪽→왼쪽으로 동일.
  4. 두 체인 합치기 (끝점 중복 제외).
monotone_chain.py
def convex_hull_andrew(points):
    pts = sorted(set(points))
    if len(pts) <= 1: return pts

    # 하단 체인
    lower = []
    for p in pts:
        while len(lower) >= 2 and ccw(lower[-2], lower[-1], p) <= 0:
            lower.pop()
        lower.append(p)

    # 상단 체인
    upper = []
    for p in reversed(pts):
        while len(upper) >= 2 and ccw(upper[-2], upper[-1], p) <= 0:
            upper.pop()
        upper.append(p)

    return lower[:-1] + upper[:-1]    # 끝점 중복 제거


print(convex_hull_andrew([(0,0), (1,1), (2,2), (4,0), (3,3), (0,4)]))
# [(0, 0), (4, 0), (3, 3), (0, 4)]

같은 O(n log n)이지만 극각 정렬 없이 단순 정렬로 더 robust. 경쟁 프로그래밍에서 Graham보다 선호됨.

05 — Jarvis / Chan — 출력 민감 알고리즘

전공자

Jarvis March (1973, Gift Wrapping)

"선물 포장" 은유: 가장 아래 점에서 시작해, 모든 점과의 각도를 비교해 다음 hull 정점을 매번 골라 시계/반시계 방향으로 진행. h번의 pass × n 비교 = O(nh). h=볼록 껍질 정점 수.

  • h가 작을 때(예: 점 대부분이 껍질 위가 아닐 때) O(nh) < O(n log n).
  • h=n이면(모든 점이 껍질 위) O(n²) — 최악.
  • 출력 크기 의존 → "output-sensitive" 알고리즘.
전공자

Chan's Algorithm (1996)

Timothy Chan이 Jarvis와 Graham의 장점을 결합: O(n log h) — 이론적으로 볼록 껍질의 optimal 하한.

  1. 점들을 크기 m의 그룹으로 나눔. 각 그룹의 볼록 껍질을 Graham으로 구축 → O(n log m).
  2. Jarvis 방식으로 전체 hull을 찾되, 다음 점 선택 시 각 그룹의 mini-hull에 tangent를 이진 탐색 → 각 pass O(log m) per group × (n/m) groups = O((n/m) log m).
  3. h 라운드 × 위 비용 = O(nh/m · log m).
  4. m = h로 놓으면 O(n log h). h를 모르지만 exponential search(m=2, 4, 8, ...)로 실제 h 도달.

우아하지만 실무 구현은 드묾. Chan 본인도 "교육용"이라 지칭. 이론적 최적이 중요한 증명.

06 — Bentley-Ottmann — 선분 교차 1979

고급

문제와 순진한 해법

n개 선분 중 교차하는 모든 쌍 찾기. 순진한 방법 O(n²). 교차가 k개뿐이라면 출력만으로도 Ω(k).

Bentley-Ottmann 1979: sweep line으로 O((n+k) log n).

전공자

Sweep Line 패러다임

가상의 수직선(sweep line)을 왼쪽에서 오른쪽으로 이동. 이벤트 발생 지점에서만 상태 업데이트.

  • 이벤트 큐(min-heap): 모든 선분의 왼/오른 끝점 + 발견된 교차점. x 좌표 순.
  • 활성 상태(balanced BST): 현재 sweep line을 지나는 선분들, y 순으로 정렬.
  • 세 종류 이벤트:
    • 선분 시작: 활성 집합에 추가. 상·하 이웃과 교차 체크.
    • 선분 끝: 활성 집합에서 제거. 기존 상·하 이웃이 이제 이웃끼리 교차 체크.
    • 교차: 두 선분 순서 swap. 새 이웃들과 교차 체크.
  • 각 이벤트 O(log n). 이벤트 총 O(n + k) → O((n+k) log n).
Sweep line 스냅샷: 수직선이 좌→우로 이동하며 "선분 시작/끝/교차" 이벤트마다 BST 업데이트. 각 시점의 이웃 쌍만 교차 체크 → 모든 n² 쌍을 보지 않아도 모든 교차 발견.

07 — 최근접 점 쌍 — 분할 정복

고급

Shamos-Hoey 1975, O(n log n)

n개 점 중 거리가 최소인 쌍. 순진한 O(n²)을 O(n log n)으로.

  1. x 좌표 중간으로 점을 반씩 나눔.
  2. 왼쪽·오른쪽 각각 재귀. 최소 거리 dL, dR 구함. δ = min(dL, dR).
  3. 경계 영역 (중간선에서 δ 이내) 내 점들만 고려. 이 영역에서 y 순으로 정렬 후 각 점마다 이웃 7개만 비교(기하 조밀도 주장).
  4. 전체 재귀 비용 T(n) = 2T(n/2) + O(n) → O(n log n).
closest_pair.py
import math

def dist(a, b):
    return math.hypot(a[0]-b[0], a[1]-b[1])

def closest_pair(points):
    P = sorted(points)    # x 정렬

    def rec(px):
        n = len(px)
        if n <= 3:
            return min((dist(px[i], px[j]), px[i], px[j])
                       for i in range(n) for j in range(i+1, n))
        mid = n // 2
        mx = px[mid][0]
        dl = rec(px[:mid]); dr = rec(px[mid:])
        d, p1, p2 = min(dl, dr)

        # 중간 영역
        strip = sorted([p for p in px if abs(p[0] - mx) < d], key=lambda p: p[1])
        for i in range(len(strip)):
            j = i + 1
            while j < len(strip) and strip[j][1] - strip[i][1] < d:
                nd = dist(strip[i], strip[j])
                if nd < d:
                    d, p1, p2 = nd, strip[i], strip[j]
                j += 1
        return d, p1, p2

    return rec(P)

print(closest_pair([(0,0), (1,1), (2,2), (4,0), (3,3)]))
# (1.414..., (0,0), (1,1)) 또는 동등한 최근접 쌍

핵심 통찰: strip 내 각 점 주변 δ×2δ 영역에 최대 7개의 다른 점만 있음 (pigeonhole 논증). 따라서 O(n).

08 — Point-in-Polygon — Ray Casting

중급

"홀수번 건너면 내부"

점 P가 다각형 내부인가 외부인가? Ray casting:

  1. P에서 한 방향으로 ray(보통 +x 방향)를 쏨.
  2. ray와 다각형 변(edge)의 교차 수 k를 셈.
  3. k가 홀수면 내부, 짝수면 외부. Jordan curve theorem.
  • 시간 O(n) (n=다각형 변 수).
  • 경계 처리: ray가 정점을 정확히 지나는 경우 주의 (정점은 두 번 카운트 방지).
point_in_polygon.py
def point_in_polygon(point, polygon):
    """polygon: 꼭짓점 리스트 (순서대로)."""
    x, y = point
    n = len(polygon)
    inside = False
    j = n - 1
    for i in range(n):
        xi, yi = polygon[i]
        xj, yj = polygon[j]
        # y 범위 내 + x 방향 교차 체크
        if ((yi > y) != (yj > y)) and (x < (xj - xi) * (y - yi) / (yj - yi) + xi):
            inside = not inside
        j = i
    return inside


# 테스트
square = [(0,0), (4,0), (4,4), (0,4)]
print(point_in_polygon((2, 2), square))    # True
print(point_in_polygon((5, 5), square))    # False
print(point_in_polygon((2, 0), square))    # 경계 — 구현에 따라 True/False

Winding number 방법도 있음 — ray 주변을 "감는 횟수" 계산. 자기 교차 다각형에도 작동. 경계 처리가 더 robust.

09 — Voronoi / Delaunay

전공자

Voronoi Diagram

우크라이나 수학자 Georgy Voronoy(1868-1908)의 1908년 정수론 연구에서 기원. 이후 평면 기하로 일반화.

평면의 점 집합 S. 각 점 s에 대한 Voronoi cell = "s가 S 중 가장 가까운 영역". 평면이 다각형 cell로 분할됨.

  • 응용: "가장 가까운 지하철역 찾기", 전화 기지국 할당, 생태학(영역), 재료과학(결정 구조).
  • 구축: Fortune 1987 sweep line O(n log n). 또는 dual인 Delaunay 삼각화에서 유도.
전공자

Delaunay Triangulation

러시아 수학자 Boris Delaunay(1890-1980)의 1934년 논문 "Sur la sphère vide"에서 정의.

n개 점에 대한 삼각화 중, 각 삼각형의 외접원 내부에 다른 점이 없는 삼각화. Voronoi의 dual.

  • 최소 각을 최대화 — "뾰족한" 삼각형 회피. 지형/메시 생성의 이상적 결과.
  • Fortune 1987 기반 O(n log n). Flip 기법.
  • 응용: FEM(유한 요소법), 지형 mesh, 3D 모델링.

10 — Sweep Line 응용 — 직사각형 합집합

전공자

Klee's Measure Problem (1977)

n개 축 정렬 직사각형의 합집합 면적. 가장 단순한 sweep line 응용.

  1. 이벤트: 각 직사각형의 왼/오른 변. x 좌표로 정렬.
  2. 활성 상태: 현재 sweep line을 지나는 y 구간들 (segment tree로 관리).
  3. x가 Δ 이동하는 동안 "활성 y 구간 총 길이 × Δ"만큼 면적 누적.
  4. 총 O(n log n).

Klee's measure problem: 2D는 O(n log n)이 달성되었고 최적 (Bentley 1977 하한과 일치). d ≥ 3은 여전히 open problem — 현재 최선은 Chan 2013의 nd/2 · 2O(log* n).

Bentley 자신이 1977년에 최초 O(n log² n) 해법 제시.

11 — 부동소수점의 저주

전공자

기하 알고리즘의 악몽

기하 알고리즘은 부동소수점 오차에 특히 취약. 세 점이 "수학적으로 collinear"인데 floating point로는 약간 CCW 또는 약간 CW로 판정되어 결과가 엉망.

예: Graham scan에서 ccw(A, B, C) = 0(collinear)을 엉뚱하게 양수 또는 음수로 → 잘못된 점을 hull에 포함 / 제외.

  • 해결책 1: 정수 좌표 + 큰 정수 연산. Python은 native bigint 지원 — 기하 알고리즘에 유리.
  • 해결책 2: Exact arithmetic (CGAL). 유리수 또는 구간 산술.
  • 해결책 3: Adaptive precision (Shewchuk 1997). 필요할 때만 정밀도 증가.
  • 해결책 4: SoS (Simulation of Simplicity, Edelsbrunner-Mücke 1990). 기호적 섭동으로 degeneracy 처리.

실무 교훈: 순진한 float 기반 기하 구현은 거의 반드시 버그를 품음. CGAL, Shapely 같은 검증된 라이브러리 사용 권장.

중급

Python: Shapely

Shapely는 Python의 표준 2D 기하 라이브러리. 내부는 C++ GEOS(Geometry Engine Open Source, JTS Java Topology Suite의 포팅).

shapely_example.py
from shapely.geometry import Point, Polygon, LineString

p = Polygon([(0,0), (4,0), (4,4), (0,4)])

print(Point(2, 2).within(p))             # True
print(p.convex_hull)                      # 이미 볼록이므로 자신
print(p.intersection(Polygon([(2,2), (6,2), (6,6), (2,6)])))
# POLYGON ((4 2, 2 2, 2 4, 4 4, 4 2))

line = LineString([(0,0), (5,5)])
print(line.length)                        # 7.07...

  • 성능: 내부 C++이라 순수 Python 대비 100배 이상 빠름.
전공자

C++: CGAL

Computational Geometry Algorithms Library(1996-). 가장 완성도 높은 기하 라이브러리. 학계·산업에서 표준.

  • Exact arithmetic kernel: 기하 알고리즘의 정확성 보장.
  • Voronoi, Delaunay, convex hull, Boolean, mesh 등 수백 알고리즘.
  • Autodesk, Bentley, Dassault 등 상용 제품의 내부.

Python 바인딩은 성숙도 낮음. 상용 소프트웨어는 주로 C++ CGAL 또는 자체 구현.

전공자

AI가 기하를 잘못 쓰는 패턴

  • Float 직접 비교: ccw(A, B, C) == 0 — 거의 절대 참 아님. abs(ccw) < eps도 eps 선택이 문제.
  • 정렬 → O(n log n)인데 정렬 안 하고 "O(n)" 주장: Graham scan 등에서 자주 오류.
  • 다각형이 self-intersecting일 때: ray casting도 깨짐. winding number 쓰거나 검증.
  • 2D에만 유효한 알고리즘을 3D에 적용: 볼록 껍질 3D는 훨씬 복잡 (divide-conquer, gift wrapping 등 별도).
  • 순진한 O(n²) 선분 교차: n=10⁴면 10⁸ — 가능하지만 큰 문제엔 Bentley-Ottmann 필수.

교정 질문: "정수 좌표로 할 수 있나?", "degeneracy 처리?", "Shapely/CGAL로 충분?"

14 — 연습문제 (12문제)

WHY 문제 1

CCW 외적 공식이 왜 "좌회전/우회전"을 말해주나?

풀이 펼치기
답

2D 벡터 AB와 AC의 외적 z 성분 = |AB|·|AC|·sin θ. θ는 AB → AC의 각도. sin θ > 0은 반시계 방향(CCW), < 0은 시계(CW), = 0은 parallel. 즉 외적의 부호가 회전 방향. 3D 외적의 z 성분만 취한 것.

PREDICT 문제 2

Graham scan에서 극각이 같은 점이 여러 개면?

풀이 펼치기
답

기준점에서 거리 가까운 것 먼저 오도록 정렬. 가장 먼 점만 hull에 포함됨 (같은 방향의 중간 점들은 collinear이므로 pop됨). 반대로 정렬(먼 것 먼저)이면 collinear 처리 로직이 복잡해짐. 실무 구현은 거리 근접 우선.

BUILD 문제 3

두 선분 (p1,p2), (p3,p4)의 교차 여부 판정 (CCW 기반).

풀이 펼치기
답

교차 조건: ccw(p1,p2,p3)와 ccw(p1,p2,p4)가 부호가 다름 AND ccw(p3,p4,p1)와 ccw(p3,p4,p2)가 부호가 다름. 즉 각 선분이 다른 선분의 양쪽을 가름. Collinear + 중첩 케이스는 별도 처리. 4번의 CCW 호출로 판정.

DESUGAR 문제 4

Andrew 알고리즘에서 "상·하 체인" 조합이 왜 올바른 볼록 껍질이 되나?

풀이 펼치기
답

점들을 x 순으로 정렬하면 가장 왼쪽과 가장 오른쪽 점은 반드시 hull 위. 하단 체인: 이 두 점을 y 작은 쪽으로 잇는 사슬. 상단: y 큰 쪽. 두 체인이 왼쪽·오른쪽 끝점에서 만나 전체 hull 완성. 각 체인 내부에서 stack pop으로 오목 부분 제거.

WHY 문제 5

최근접 점 쌍의 strip 내 "7개 이웃" 논증.

풀이 펼치기
답

strip 너비 2δ (중간선 기준 ±δ). 한 점 p 주변 2δ × 2δ 박스를 8개 작은 δ/2 × δ 부분으로 나누면 각 부분에 최대 1점 (δ 미만 거리 없으므로). 따라서 p 주변 박스엔 최대 7개 다른 점 → y 순 다음 7개만 검사.

BUILD 문제 6

다각형의 면적을 CCW로 계산 (Shoelace formula).

풀이 펼치기
답

Shoelace (Gauss): area = |Σ (x_i · y_{i+1} − x_{i+1} · y_i)| / 2. 또는 CCW 해석: 삼각형 (O, p_i, p_{i+1})의 부호 있는 면적 합. 부호 = CCW이면 양수. 자기 교차 없는 단순 다각형에 정확.

def area(poly):
    n = len(poly)
    s = 0
    for i in range(n):
        x1, y1 = poly[i]
        x2, y2 = poly[(i+1) % n]
        s += x1*y2 - x2*y1
    return abs(s) / 2
PREDICT 문제 7

n=10만 점의 볼록 껍질 — Graham vs Chan 실측 예상.

풀이 펼치기
답

Graham O(n log n) = 10⁵ · 17 ≈ 2·10⁶ 연산. C++ 수십 ms. Chan O(n log h): h가 작을수록 빠름. 랜덤 점에선 h ≈ log n ≈ 12 → 10⁵ · 4 = 4·10⁵ 연산. 하지만 상수가 큼, 실측에선 비슷. 이론적 최적인 h가 정말 작을 때만 유리. 실무엔 Andrew's monotone chain이 표준.

DESUGAR 문제 8

Bentley-Ottmann에서 교차 이벤트가 큐에 동적으로 추가되는 이유.

풀이 펼치기
답

선분 시작 시 이웃끼리 교차 점검 → 미래 교차점을 큐에 추가. 선분 끝 시 사라진 선분의 원래 이웃들이 이제 직접 이웃 → 교차 체크. 즉 sweep line이 만날 교차점만 큐에. 모든 O(n²)쌍이 아닌 활성 이웃만 검사하는 것이 효율의 핵심.

WHY 문제 9

Voronoi가 왜 Delaunay의 dual인가?

풀이 펼치기
답

Voronoi cell의 정점 = 세(이상) cell의 공통 경계점 = 세 점과 같은 거리 = 세 점의 외접원 중심. Delaunay 삼각형의 외접원 중심도 같음. Voronoi 간선이 두 Delaunay 점을 잇는 수직 이등분선, Voronoi 정점이 Delaunay 삼각형의 외심 — 정확한 dual 관계. 하나 구하면 다른 것도 O(n)으로.

BUILD 문제 10

"직사각형 합집합 면적" sweep line 구현 스케치.

풀이 펼치기
답
  1. 각 직사각형 (x1, x2, y1, y2)에서 (x1, +, y1, y2), (x2, −, y1, y2) 이벤트.
  2. x 순 정렬. 스위프하며 segment tree에 y 구간 유지.
  3. 각 이벤트 사이 구간 Δx 동안 "현재 덮인 y 총 길이" × Δx 누적.
  4. Segment tree는 "y 좌표 좌표 압축 + lazy count"로 구현. O(n log n).

WHY 문제 12

컴퓨터 그래픽스의 "Z-buffer"와 계산 기하의 관계.

풀이 펼치기
답

3D 장면을 2D 스크린에 투영 시 "가장 가까운 표면만 보이는" 문제. 기하학적으로는 3D hidden surface removal — sweep line의 변형(스크린을 pixel 단위로 탐색). 현대 GPU는 하드웨어 z-buffer로 초당 수십억 pixel 처리. 계산 기하의 정교한 이론보다 단순 무식한 O(screen · objects)가 실무에 채택된 사례.
