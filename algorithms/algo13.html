01 — 이번 주 이야기: 가족 족보와 디렉토리

가족 족보

"내 후손 모두를 나열하라"고 했을 때, 어떤 순서로 적을까요?

  • 방식 1: 나 → 큰아들 → 큰아들의 큰아들 → … (한쪽으로 끝까지) → 백트랙 → 큰아들의 둘째아들 → … DFS.
  • 방식 2: 나 → (내 자식들 모두) → (자식들의 자식들 모두) → … 세대별로 나열. BFS.

디렉토리 트리

find /home이 디렉토리를 어떻게 순회할까요? — DFS 기본. 한 디렉토리 안으로 들어가서 끝까지 본 뒤 형제 디렉토리로.

  • pre-order: 디렉토리 출력 → 그 안 파일들 (디렉토리 먼저).
  • post-order: 파일들 먼저 → 디렉토리 (자식 처리 후 부모). rm -rf가 이걸 사용 — 자식부터 지워야 부모 지울 수 있음.
  • BFS: find -maxdepth 1처럼 같은 깊이 먼저.
WHY — 4가지 순회의 의미
순회방문 순서대표 사용처
Pre-order노드 → 왼쪽 → 오른쪽트리 복사, 표현식 prefix 변환, 직렬화
In-order왼쪽 → 노드 → 오른쪽BST 정렬 순서 (Algo 11)
Post-order왼쪽 → 오른쪽 → 노드트리 삭제, 표현식 평가, dependency resolution
Level-order (BFS)깊이별 (위에서 아래)최단 경로 (Algo 14), 직렬화, 깊이별 분석

4가지 모두 O(n) 시간이지만 "무엇을 언제 처리하는가"가 다름. 문제마다 적합한 순회가 다릅니다.

같은 트리, 4가지 방문 순서. Pre-order(직렬화), In-order(BST 정렬), Post-order(트리 삭제·표현식 평가), Level-order(BFS 최단 경로). 각자 특별한 사용처.
DFS Pre-order = "한 가지 끝까지 들어갔다가 나옴". 1번 방문 → 왼쪽 자식(2) 방문 → 또 왼쪽(4) → 더 갈 곳 없으면 백트랙 → 오른쪽(5) → 1로 돌아와서 오른쪽(3). 자연스러운 가족 족보 순회.

02 — 트리의 정의와 용어

초급

트리란

연결된 비순환 그래프(connected acyclic graph). 또는 더 직관적으로:

  • 한 개의 루트(root) 노드.
  • 각 노드는 0개 이상의 자식(child) 노드를 가짐.
  • 자식이 없는 노드 = 리프(leaf).
  • 루트를 제외한 모든 노드는 정확히 하나의 부모를 가짐.
  • 사이클 없음.
중급

핵심 용어

용어정의
Root최상위 노드. 부모 없음.
Leaf자식 없는 노드.
Internal node자식이 있는 노드.
Depth루트로부터의 거리. 루트의 depth = 0.
Height해당 노드에서 leaf까지의 최장 거리. 트리 height = root height.
Subtree한 노드와 그 모든 후손.
Ancestor한 노드의 부모, 부모의 부모, … 루트까지.
Sibling같은 부모를 가진 노드들.
Binary tree각 노드가 최대 2개 자식 (left, right).
Complete binary tree마지막 레벨 제외 모두 꽉 참, 마지막은 왼쪽부터.
Full binary tree모든 내부 노드가 정확히 2개 자식.
Perfect binary tree모든 leaf가 같은 depth + full.
⚠️ "Complete"와 "Full" 혼동 주의

한국어로는 둘 다 "완전 이진 트리"로 번역되기도 하지만 다른 개념:

  • Complete: 모양이 꽉 찬. 마지막 레벨만 미완성 가능. 힙(heap)이 이거.
  • Full / Strict: 각 노드의 자식 수가 0 or 2. 모양 무관.
  • Perfect: 모든 leaf가 같은 depth + 모든 내부 노드 자식 2개. 가장 엄격.

03 — 트리 표현 방식 (CLRS §10.3)

중급

방식 1 — 자식 포인터 (Binary tree)

tree_node.py
class TreeNode:
    def __init__(self, val, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

가장 흔한 방식. LeetCode 트리 문제의 표준. 단, 각 노드가 정확히 2개 자식 슬롯이라 일반 트리(자식 수 가변)에는 부적합.

고급

방식 2 — 자식 리스트 (General tree)

general_tree.py
class TreeNode:
    def __init__(self, val):
        self.val = val
        self.children = []   # list of TreeNode

자식 수 무제한. 디렉토리 구조, JSON tree, DOM tree 등.

고급

방식 3 — Left-Child Right-Sibling (CLRS §10.3)

일반 트리를 이진 트리로 표현하는 영리한 기법.

각 노드:
     left_child → 첫 번째 자식
     right_sibling → 같은 부모의 다음 형제

장점: 노드 크기 고정 (포인터 2개). 단점: "k번째 자식 접근"이 O(k).

OS의 프로세스 트리, 파일 시스템 디렉토리 표현에 자주 사용.

전공자

방식 4 — Parent 포인터만

parent_only.py
class Node:
    def __init__(self, val, parent=None):
        self.val = val
        self.parent = parent

Union-Find(Algo 17)의 핵심 표현. 자식 정보 없이 "조상 따라가기"만 필요할 때.

📖 원서 요약 — CLRS 4판, §10.3 "Representing rooted trees"

CLRS §10.3은 "branching factor가 무제한이거나 알 수 없는 루트 트리를 O(n) 공간에 표현"하는 방법으로 left-child, right-sibling representation을 소개합니다. 각 노드는 parent, left-child (첫 자식), right-sibling (같은 부모의 다음 형제) 3개 포인터만 갖습니다 — 이진 트리처럼 보이지만 실제는 일반 트리.

04 — DFS 재귀: pre / in / post-order

중급

3가지 순회의 정의

예제 트리:
           [1]
       /     \
     [2]     [3]
   /  \
[4]  [5]

Pre-order (Root → L → R): 1, 2, 4, 5, 3
In-order (L → Root → R): 4, 2, 5, 1, 3
Post-order (L → R → Root): 4, 5, 2, 3, 1
중급

재귀 구현

dfs_recursive.py
def preorder(node):
    if node is None: return
    print(node.val)         # root
    preorder(node.left)
    preorder(node.right)


def inorder(node):
    if node is None: return
    inorder(node.left)
    print(node.val)         # root 처리는 left 다음
    inorder(node.right)


def postorder(node):
    if node is None: return
    postorder(node.left)
    postorder(node.right)
    print(node.val)         # root는 마지막

세 코드의 차이는 print 위치 한 줄뿐. 본질이 같다는 증거.

고급

복잡도 분석

  • 시간: O(n) — 각 노드 정확히 한 번 방문.
  • 공간 (재귀 스택): O(h), h = 트리 높이. 균형이면 O(log n), 한쪽 기울면 O(n).

Python의 함수 호출 스택은 약 1KB/frame. 깊이 1000이면 ~1MB. 매우 깊은 트리에서 위험.

⚠️ Python의 RecursionError

Python의 기본 재귀 한계는 sys.getrecursionlimit() = 1000. 한쪽으로 기울어진 트리(혹은 매우 깊은 트리)에서 RecursionError: maximum recursion depth exceeded.

  • sys.setrecursionlimit(10**6)으로 늘릴 수 있지만 위험 (스택 오버플로 = 즉시 SIGSEGV).
  • 안전한 해결: iterative + 명시적 스택. §05에서.

05 — DFS 반복: 명시적 스택

고급

Pre-order 반복

가장 단순. 오른쪽을 먼저 push하면 왼쪽이 먼저 pop됨.

preorder_iter.py
def preorder_iter(root):
    if root is None: return
    stack = [root]
    while stack:
        node = stack.pop()
        print(node.val)
        if node.right: stack.append(node.right)   # 오른쪽 먼저!
        if node.left:  stack.append(node.left)
고급

In-order 반복 — 약간 더 복잡

inorder_iter.py
def inorder_iter(root):
    stack = []
    cur = root
    while cur or stack:
        # 왼쪽 끝까지 내려가며 stack에 쌓기
        while cur:
            stack.append(cur)
            cur = cur.left
        # 가장 왼쪽 노드 pop, 출력 후 오른쪽으로
        cur = stack.pop()
        print(cur.val)
        cur = cur.right
전공자

Post-order 반복 — 가장 까다로움

Post-order는 "자식 모두 처리 후 부모"라 단순 stack으로는 어려움. 두 가지 접근:

방법 A — 두 스택 (리버스 트릭)

postorder_two_stacks.py
def postorder_iter(root):
    if root is None: return
    stack = [root]
    out = []
    while stack:
        node = stack.pop()
        out.append(node.val)        # Root → Right → Left 순서
        if node.left: stack.append(node.left)
        if node.right: stack.append(node.right)
    print(out[::-1])              # reverse → Left → Right → Root

Pre-order의 변형 (R/L 순서 뒤바꾼 후 결과 reverse).

방법 B — 방문 표시 (visited flag)

postorder_visited.py
def postorder_iter2(root):
    if root is None: return
    stack = [(root, False)]
    while stack:
        node, visited = stack.pop()
        if visited:
            print(node.val)
        else:
            stack.append((node, True))      # 자기 자신 (visited=True)
            if node.right: stack.append((node.right, False))
            if node.left:  stack.append((node.left, False))

더 명시적이고 일반적. 이 패턴은 "자식 후 부모 처리" 어디에나 적용 가능.

06 — BFS: Level-Order, deque 활용

중급

Queue 사용

DFS는 stack, BFS는 queue. Algo 09에서 본 collections.deque 사용.

level_order.py
from collections import deque

def level_order(root):
    if root is None: return
    q = deque([root])
    while q:
        node = q.popleft()         # O(1) — list.pop(0)는 O(n)!
        print(node.val)
        if node.left:  q.append(node.left)
        if node.right: q.append(node.right)

list 사용 금지: list.pop(0)은 O(n) → 전체 BFS가 O(n²) 됨. deque.popleft는 O(1).

고급

레벨별로 묶기

"각 깊이의 노드들을 별도 리스트로" — LeetCode #102.

level_order_grouped.py
def level_order_grouped(root):
    if root is None: return []
    result = []
    q = deque([root])
    while q:
        level = []
        for _ in range(len(q)):    # 현재 레벨 크기 고정
            node = q.popleft()
            level.append(node.val)
            if node.left:  q.append(node.left)
            if node.right: q.append(node.right)
        result.append(level)
    return result

핵심 트릭: 한 레벨 처리 직전 len(q)를 캡처. 그 만큼만 처리하면 그 레벨이 끝.

WHY — BFS의 특별한 성질
  • 최단 경로 보장: 첫 발견이 최단 거리. Algo 14의 BFS 그래프 알고리즘 핵심.
  • 깊이별 분석: 각 레벨을 그룹으로 다루기 쉬움.
  • 메모리 사용: 한 레벨의 최대 너비만큼 큐 크기. 균형 트리에서 O(n/2) — DFS의 O(log n)보다 큼.

메모리 트레이드오프 — 트리가 매우 wide하면 BFS가 DFS보다 메모리 많이 씀.

07 — Morris Traversal: O(1) 공간 (Joseph M. Morris 1979)

전공자

Knuth의 의문에서 Morris의 답까지

Donald Knuth의 TAOCP §2.3.1은 threaded tree(Perlis & Thornton, 1960)를 다룹니다: "NULL 포인터를 in-order 후계자를 가리키는 실(thread)로 바꾸면 스택 없이 순회 가능하다." 다만 원본 트리 자체를 영구적으로 수정해야 한다는 비용이 있었습니다.

Joseph M. Morris가 1979년 논문 "Traversing Binary Trees Simply and Cheaply" (IPL 9(5), 197–200)에서 답: 트리를 순회 중에 임시로 수정해 thread를 만들고, 지나간 뒤 원상복구한다 — Morris traversal.

전공자

Morris In-order 알고리즘

cur = root
while cur:
     if cur.left == None:
         visit(cur)
         cur = cur.right
     else:
         predecessor = cur.left에서 가장 오른쪽 (in-order predecessor)
         if predecessor.right == None:
            predecessor.right = cur # threading 만들기
            cur = cur.left
         else:
            predecessor.right = None # threading 제거 (복구)
            visit(cur)
            cur = cur.right
morris_inorder.py
def morris_inorder(root):
    cur = root
    while cur:
        if cur.left is None:
            print(cur.val)
            cur = cur.right
        else:
            # in-order predecessor 찾기
            pred = cur.left
            while pred.right and pred.right is not cur:
                pred = pred.right
            if pred.right is None:
                pred.right = cur          # threading 만들기
                cur = cur.left
            else:                         # 두 번째 방문 = 왼쪽 다 처리됨
                pred.right = None           # threading 제거
                print(cur.val)
                cur = cur.right
전공자

복잡도와 보장

  • 시간: O(n). 각 edge가 최대 3번 순회됨 (predecessor 탐색에서 중복).
  • 공간: O(1). cur, pred 두 포인터만.
  • 재귀 없음: stack overflow 안전.
  • 트리 일시 수정: 순회 도중 leaf의 right 포인터가 잠시 달라짐. 순회 끝나면 원상 복구.
⚠️ Morris의 트레이드오프
  • 트리가 변경됨 (일시적으로). 멀티스레드 환경에서 다른 스레드가 트리 읽으면 잘못된 결과 가능.
  • 중간에 예외 발생 시 트리 꼬임: 복구 안 되면 영구적으로 망가짐.
  • 실용에선 거의 안 쓰지만 — "O(1) 공간이 가능하다"는 알고리즘적 통찰의 표본. 인터뷰 단골.
📖 원문 인용 — Joseph M. Morris (1979)

"Traversing Binary Trees Simply and Cheaply" (Information Processing Letters 9(5)): "We present an algorithm for traversing a binary tree using only O(1) extra storage by temporarily threading the tree during traversal. The threads are removed before the traversal completes, leaving the tree in its original state."

— Morris, J.M. (1979). Information Processing Letters, 9(5), 197-200.

08 — 트리 직렬화 / 역직렬화

고급

문제 — 트리를 문자열로, 다시 트리로

LeetCode #297. "Codec.serialize(root)와 deserialize(data)를 작성하라."

파일 저장, 네트워크 전송, 캐싱 — 모든 곳에 필요.

고급

방법 1 — Pre-order + null marker

serialize_preorder.py
def serialize(root):
    def helper(node):
        if node is None:
            vals.append("#")            # null marker
            return
        vals.append(str(node.val))
        helper(node.left)
        helper(node.right)
    vals = []
    helper(root)
    return ",".join(vals)


def deserialize(data):
    def helper():
        v = next(it)
        if v == "#": return None
        node = TreeNode(int(v))
        node.left = helper()
        node.right = helper()
        return node
    it = iter(data.split(","))
    return helper()


# 예: tree [1, 2, 3, null, null, 4, 5] →
#     "1,2,#,#,3,4,#,#,5,#,#"

장점: 단순. 단점: null marker로 크기 증가 (n개 노드 → 2n+1 토큰).

전공자

방법 2 — Level-order (BFS) + null marker

LeetCode 트리 표기법으로 익숙한 방식 ([1,2,3,null,null,4,5]).

serialize_levelorder.py
from collections import deque

def serialize_bfs(root):
    if root is None: return ""
    q = deque([root])
    out = []
    while q:
        node = q.popleft()
        if node is None:
            out.append("#")
        else:
            out.append(str(node.val))
            q.append(node.left)        # None도 큐에 들어감
            q.append(node.right)
    return ",".join(out)


def deserialize_bfs(data):
    if not data: return None
    vals = data.split(",")
    root = TreeNode(int(vals[0]))
    q = deque([root])
    i = 1
    while q and i < len(vals):
        node = q.popleft()
        if vals[i] != "#":
            node.left = TreeNode(int(vals[i]))
            q.append(node.left)
        i += 1
        if i < len(vals) and vals[i] != "#":
            node.right = TreeNode(int(vals[i]))
            q.append(node.right)
        i += 1
    return root

Pre-order보다 사람이 읽기 쉬움. 트리 구조 시각적으로 명확.

주어진 순회유일한 트리 결정?

Post + In✅ 가능
Pre + Post❌ 일반적으로 불가능 (full binary tree만 가능)
Level + In✅ 가능

왜 In-order가 필수인가: in-order는 "왼쪽 서브트리 / 노드 / 오른쪽 서브트리" 경계를 보여줌. Pre/Post는 모든 노드가 한 sequence라 경계 모호.

전공자

Pre-order + In-order 재구성 (LeetCode #105)

build_tree.py
def build_tree(preorder, inorder):
    in_idx = {v: i for i, v in enumerate(inorder)}   # O(1) 조회
    pre_iter = iter(preorder)

    def helper(left, right):
        if left > right: return None
        val = next(pre_iter)
        node = TreeNode(val)
        mid = in_idx[val]              # in-order에서 root 위치
        node.left = helper(left, mid - 1)
        node.right = helper(mid + 1, right)
        return node

    return helper(0, len(inorder) - 1)

O(n) 시간, O(n) 공간 (in_idx dict). 재귀 깊이 O(h).

핵심 통찰: pre-order의 첫 원소는 항상 root. in_idx로 root 위치를 O(1)에 찾고 좌/우 분할.

10 — LCA (Lowest Common Ancestor)

고급

문제 정의

두 노드 u, v의 최저 공통 조상 — u와 v 모두를 후손으로 가지는 가장 깊은 노드.

고급

알고리즘 1 — 재귀 (single query)

lca_recursive.py
def lca(root, p, q):
    if root is None or root is p or root is q:
        return root
    left = lca(root.left, p, q)
    right = lca(root.right, p, q)
    if left and right:        # p와 q가 양쪽 서브트리에 있음
        return root
    return left or right     # 한쪽에만 있으면 그쪽이 LCA

O(n) 시간, O(h) 공간. 단일 쿼리에 적합. 여러 쿼리면 매번 O(n)이라 비효율.

전공자

알고리즘 2 — Binary Lifting (multiple queries)

q번의 쿼리가 있다면 매번 O(n)은 비효율. 전처리 후 각 쿼리 O(log n).

아이디어: 각 노드의 2ᵏ번째 조상을 미리 저장 (k=0,1,2,…,log n).

binary_lifting_lca.py
import math

class LCA:
    def __init__(self, n, parent, root=0):
        # parent[root] = root 규약 (경계 고정점)
        self.LOG = math.ceil(math.log2(n)) + 1
        self.up = [[0] * n for _ in range(self.LOG)]
        self.depth = [0] * n
        # BFS로 depth 채우기
        children = [[] for _ in range(n)]
        for v in range(n):
            if v != root: children[parent[v]].append(v)
        from collections import deque
        q = deque([root])
        while q:
            u = q.popleft()
            for c in children[u]:
                self.depth[c] = self.depth[u] + 1
                q.append(c)
        # up[0][v] = parent[v]
        for v in range(n):
            self.up[0][v] = parent[v]
        # up[k][v] = up[k-1][up[k-1][v]]
        for k in range(1, self.LOG):
            for v in range(n):
                self.up[k][v] = self.up[k - 1][self.up[k - 1][v]]

    def query(self, u, v):
        # u가 더 깊다고 가정
        if self.depth[u] < self.depth[v]:
            u, v = v, u
        diff = self.depth[u] - self.depth[v]
        # u를 v 깊이로 끌어올림
        for k in range(self.LOG):
            if (diff >> k) & 1:
                u = self.up[k][u]
        if u == v: return u
        # 동시에 위로
        for k in range(self.LOG - 1, -1, -1):
            if self.up[k][u] != self.up[k][v]:
                u, v = self.up[k][u], self.up[k][v]
        return self.up[0][u]

전처리: O(n log n). 각 쿼리: O(log n). q개 쿼리 총 O(n log n + q log n).

예제 2 — Diameter (LeetCode #543)

트리에서 두 노드 간 가장 긴 경로의 길이.

diameter.py
def diameter(root):
    ans = [0]
    def depth(node):
        if node is None: return 0
        l = depth(node.left)
        r = depth(node.right)
        # 이 노드를 거치는 경로의 최대 길이
        ans[0] = max(ans[0], l + r)
        return 1 + max(l, r)
    depth(root)
    return ans[0]

O(n) 시간. 핵심 통찰: 각 노드는 두 가지 다른 정보를 반환 — "자기를 거치는 경로 길이" (전역 ans 갱신)와 "자기 위로 뻗어가는 경로 길이" (반환값).

전공자

예제 3 — House Robber III (LeetCode #337)

"트리에서 인접한 두 노드를 동시에 선택하지 않고, 합이 최대가 되도록."

rob_tree.py
def rob(root):
    def helper(node):
        # return (취하지 않은 경우 max, 취한 경우 max)
        if node is None: return (0, 0)
        l_skip, l_take = helper(node.left)
        r_skip, r_take = helper(node.right)
        skip = max(l_skip, l_take) + max(r_skip, r_take)
        take = node.val + l_skip + r_skip
        return (skip, take)
    return max(helper(root))

각 노드에서 2개 상태를 가진 DP. 자식 결과만으로 결정 → post-order. O(n) 시간.

def is_symmetric(root):
    def mirror(a, b):
        if a is None and b is None: return True
        if a is None or b is None: return False
        return (a.val == b.val
                and mirror(a.left, b.right)
                and mirror(a.right, b.left))
    return root is None or mirror(root.left, root.right)

def is_valid_bst(root):
    def helper(node, lo, hi):
        if node is None: return True
        if node.val <= lo or node.val >= hi:
            return False
        return (helper(node.left, lo, node.val)
                and helper(node.right, node.val, hi))
    return helper(root, float('-inf'), float('inf'))

    if root is None: return []
    paths = []
    def dfs(node, path):
        path = path + str(node.val)
        if not node.left and not node.right:
            paths.append(path)

            if node.left: dfs(node.left, path + "->")
            if node.right: dfs(node.right, path + "->")

    return paths

15 — 정리 + 용어 사전 + 다음 주 맛보기

핵심 정리

🌲

4가지 순회

Pre / In / Post (DFS) + Level (BFS). 각자 다른 사용처. 복잡도는 모두 O(n).

📚

재귀 vs 명시적 스택

Python 재귀 한계 1000. 깊은 트리는 iterative + stack. setrecursionlimit은 OS 한계 따로.

🪄

Morris O(1) 공간

1979 Morris의 threading 트릭. 트리 일시 수정 후 복구. 실용보다 통찰의 가치.

🔄

BFS는 deque

list.pop(0) = O(n) → 전체 O(n²) 함정. collections.deque 필수.

🔧

Tree 재구성

한 순회로는 부족. Pre+In, Post+In, Level+In = 유일 결정. Pre+Post = 일반적 불가.

🎯

Post-order 패턴. 자식 결과로 부모 결정. tuple 반환으로 다중 상태.
