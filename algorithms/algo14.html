01 — 이번 주 이야기: 지하철 노선도와 소셜 네트워크

지하철 노선도

서울 지하철 노선도는 그래프입니다. 역(vertex) + 두 역을 잇는 선로(edge). "가산디지털단지에서 강남까지 가장 빨리 가는 법은?" — 이게 최단 경로 문제.

두 역의 거리를 안 따지면 (모든 정거장 = 1역) BFS로 풀림. 거리(시간)을 따지면 Dijkstra(다음 주).

소셜 네트워크

Facebook 친구 관계 = 그래프. 사람(vertex) + 친구 관계(edge). "6단계 분리 이론": 임의의 두 사람이 평균 6명의 친구로 연결됨 — BFS로 검증 가능.

"내 친구의 친구 (2단계 친구)" = BFS 깊이 2까지의 노드들.

컴파일러의 빌드 순서

A.cpp가 B.h를 include, B.h가 C.h를 include. 의존성 그래프: A → B → C (방향 있음, 사이클 없음). "어떤 순서로 컴파일해야 하나?" — 위상 정렬(topological sort): C, B, A 순.

Python의 importlib, GCC make, Bazel 빌드 시스템 — 모두 위상 정렬 기반.

WHY — 그래프가 모든 곳에 있는 이유

"두 개체 사이의 관계"가 있는 모든 데이터는 그래프로 표현 가능.

  • 지도 → 도시(노드) + 도로(간선) → Dijkstra, A*
  • 웹 → 페이지(노드) + 링크(간선) → PageRank
  • 분자 구조 → 원자(노드) + 결합(간선) → 화학 시뮬레이션
  • 회로 → 게이트(노드) + 와이어(간선) → 회로 합성
  • 영화 추천 → 사용자/영화(노드) + 평점(간선) → 협업 필터링
  • 인터넷 → 라우터(노드) + 링크(간선) → 라우팅

이번 주의 4가지 도구(BFS, DFS, topological sort, SCC)로 위 모든 문제의 90%를 풀 수 있습니다.

BFS = 시작점에서 가까운 순서. 큐가 거리(=깊이) 순서를 보장 → 처음 발견 시점 = 최단 거리. 비가중 그래프 최단 경로의 정답.
BFS는 호수의 물결처럼 퍼진다. 시작점 → 1칸 거리 노드들 (초록) → 2칸 거리 노드들 (노랑) → ... 큐가 거리 순서를 자동으로 보장하므로 첫 발견 = 최단 경로.

02 — 그래프의 정의와 용어

초급

정의

그래프 G = (V, E):

  • V (vertex / node): 정점들의 집합. n = |V|.
  • E (edge): 간선들의 집합. m = |E|. 각 간선은 두 정점의 쌍.

트리(Algo 13)는 "연결되어 있고 사이클이 없는 그래프". n−1 edges. 그래프가 더 일반적.

중급

핵심 용어

용어정의
Adjacent (인접)두 정점이 간선으로 직접 연결.
Degree (차수)한 정점에 연결된 간선 수.
In-degree / Out-degree방향 그래프에서 들어오는/나가는 간선 수.
Path (경로)v₁ → v₂ → ... → vₖ. 간선들의 연결.
Cycle (사이클)시작과 끝이 같은 path.
Connected component서로 연결된 정점들의 부분집합.
Strongly connected (방향)모든 두 정점 간 양방향 path 존재.
Sparse / Densem이 n에 가까우면 sparse, n²에 가까우면 dense.
Bipartite정점을 두 그룹으로 나눠 같은 그룹 안엔 간선 없음.
Self-loop자기 자신으로의 간선 (v, v).
Multigraph같은 두 정점 간 여러 간선.
고급

중요한 부등식

Undirected graph:
   Σ deg(v) = 2m (각 간선이 양 끝 정점의 degree에 1씩 기여)
   m ≤ n(n−1)/2

Directed graph:
   Σ in-deg(v) = Σ out-deg(v) = m
   m ≤ n(n−1) (self-loop 제외)

Sparse: m = O(n). Dense: m = Θ(n²). 이 구분이 자료구조 선택을 좌우.

04 — 표현 방식: 인접 리스트 vs 인접 행렬 (CLRS §20.1)

중급

인접 리스트 (Adjacency List)

각 정점에 대해 "인접한 정점들의 리스트"를 유지.

adj_list.py
# 방법 1: dict of lists (가장 흔한 Python 패턴)
graph = {
    0: [1, 2],
    1: [0, 3],
    2: [0, 3],
    3: [1, 2],
}

# 방법 2: list of lists (정점이 0..n-1 정수일 때)
n = 4
graph = [[] for _ in range(n)]
edges = [(0, 1), (0, 2), (1, 3), (2, 3)]
for u, v in edges:
    graph[u].append(v)
    graph[v].append(u)    # undirected는 양쪽 추가

# 방법 3: defaultdict(list) — 정점이 sparse하거나 임의 hashable
from collections import defaultdict
graph = defaultdict(list)
for u, v in edges:
    graph[u].append(v)
    graph[v].append(u)
  • 공간: O(V + E).
  • "v에 인접한 정점들 순회": O(deg(v)).
  • "u-v 간선 존재?": O(deg(u)) (리스트 선형 탐색).
고급

인접 행렬 (Adjacency Matrix)

adj_matrix.py
n = 4
matrix = [[0] * n for _ in range(n)]
edges = [(0, 1), (0, 2), (1, 3), (2, 3)]
for u, v in edges:
    matrix[u][v] = 1
    matrix[v][u] = 1    # undirected는 대칭

# NumPy로 더 빠르게
import numpy as np
M = np.zeros((n, n), dtype=int)
for u, v in edges:
    M[u, v] = M[v, u] = 1
  • 공간: O(V²). 작은 그래프나 dense에 OK, 큰 sparse엔 낭비 큼.
  • "v에 인접한 정점들 순회": O(V). (모든 V열을 봐야)
  • "u-v 간선 존재?": O(1). matrix[u][v] 한 번 조회.
  • 가중치: 1 대신 weight 저장. -1 또는 ∞로 "간선 없음" 표시.
고급

선택 가이드 — 거의 항상 인접 리스트

특성인접 리스트인접 행렬
공간O(V + E)O(V²)
인접 정점 순회O(deg(v))O(V)
간선 존재 확인O(deg(v))O(1)
BFS/DFS 시간O(V + E)O(V²)

실세계 그래프는 대부분 sparse: 소셜 네트워크, 도로망, 웹. 따라서 거의 항상 인접 리스트 사용. 행렬은 dense이거나 "간선 존재 확인이 매우 빈번"한 특수 상황에만.

📖 원서 인용 — CLRS 4판, §20.1 "Representations of graphs"

"There are two standard ways to represent a graph G = (V, E): as a collection of adjacency lists or as an adjacency matrix. Either way applies to both directed and undirected graphs. Because the adjacency-list representation provides a compact way to represent sparse graphs ... it is usually the method of choice."

05 — BFS (CLRS §20.2): 비가중 최단 경로

중급

BFS 알고리즘

Breadth-First Search: 시작 정점에서 가까운 (간선 수 적은) 순서대로 방문.

bfs.py
from collections import deque

def bfs(graph, start):
    visited = {start}
    queue = deque([start])
    order = []          # 방문 순서

    while queue:
        u = queue.popleft()       # O(1) — list.pop(0)는 O(n)!
        order.append(u)
        for v in graph[u]:
            if v not in visited:
                visited.add(v)
                queue.append(v)

    return order
  • 시간: O(V + E) — 각 정점 한 번 visited, 각 간선 한 번 검토.
  • 공간: O(V) — visited set + queue (최대 V).
고급

최단 경로 BFS

bfs_shortest.py
def bfs_shortest(graph, start):
    """start에서 모든 정점까지의 최단 거리(간선 수) + 부모 추적."""
    dist = {start: 0}
    parent = {start: None}
    queue = deque([start])

    while queue:
        u = queue.popleft()
        for v in graph[u]:
            if v not in dist:
                dist[v] = dist[u] + 1
                parent[v] = u
                queue.append(v)

    return dist, parent


def reconstruct_path(parent, target):
    path = []
    while target is not None:
        path.append(target)
        target = parent[target]
    return path[::-1]

BFS의 핵심 보장: 처음 정점 v를 발견할 때의 거리 = start에서 v까지의 최단 거리 (간선 수 기준).

📖 원서 인용 — CLRS 4판, §20.2 "Breadth-first search"

"Given a graph G = (V, E) and a distinguished source vertex s, breadth-first search systematically explores the edges of G to discover every vertex that is reachable from s. ... Breadth-first search produces a 'breadth-first tree' with root s that contains all reachable vertices. ... The algorithm computes the distance (smallest number of edges) from s to each reachable vertex."

06 — BFS의 성질과 응용

고급

증명 — BFS는 최단 거리

정리: BFS가 정점 v에 처음 방문할 때 dist[v] = 진짜 최단 거리 δ(s, v).

증명 스케치 (귀납법, 거리 d별):

  • Base: dist[s] = 0 = δ(s, s). ✓
  • 귀납: 거리 ≤ d-1인 모든 정점이 정확한 dist를 가진다고 가정. v를 거리 d 정점이라 하자.
    • v의 BFS 부모 u는 거리 d−1. dist[u] = d−1 (귀납 가정).
    • u가 큐에서 처리될 때 v를 발견 → dist[v] = d−1+1 = d. ✓

증명이 동작하는 핵심: 큐가 거리 순서 보장. 거리 d 정점이 모두 큐에 들어간 뒤에야 d+1 정점들이 들어감.

고급

BFS 응용

  • 비가중 최단 경로: 모든 간선 비용이 같을 때.
  • Connected component 찾기: visited 집합이 한 component.
  • Bipartite 검사: 두 색으로 alternating 색칠. 같은 색 인접하면 not bipartite.
  • 최소 단계 퍼즐: 슬라이딩 퍼즐, 워드 ladder 등 — 상태가 정점, 한 동작이 간선.
  • 웹 크롤러: 시작 페이지에서 BFS로 같은 도메인 내 페이지 발견.
  • 0-1 BFS (변형): 간선 가중치가 0 또는 1뿐이면 deque 양 끝 사용으로 최단 경로 O(V+E).
WHY — 가중 그래프엔 BFS 못 쓴다

간선마다 가중치가 다르면 BFS의 "큐가 거리 순서" 가정이 깨짐. 가중치 7 간선보다 가중치 1 간선 두 번이 더 가까움.

해법: deque 대신 heap으로 거리 우선 처리 → Dijkstra (다음 주, Algo 15).

07 — DFS (CLRS §20.3): discovery / finish time

고급

DFS 알고리즘

Depth-First Search: 한 방향으로 끝까지 들어간 뒤 백트랙. 재귀 또는 명시적 stack.

dfs_recursive.py
def dfs(graph, start):
    visited = set()
    order = []

    def visit(u):
        visited.add(u)
        order.append(u)
        for v in graph[u]:
            if v not in visited:
                visit(v)

    visit(start)
    return order


# Iterative — 깊은 그래프에 안전
def dfs_iter(graph, start):
    visited = {start}
    stack = [start]
    order = []
    while stack:
        u = stack.pop()
        order.append(u)
        for v in graph[u]:
            if v not in visited:
                visited.add(v)
                stack.append(v)
    return order
  • 시간: O(V + E).
  • 공간: O(V) — visited + stack.
전공자

Discovery / Finish Time (CLRS 표준)

CLRS §20.3에서는 각 정점에 두 개의 timestamp 부여:

  • d[u] (discovery time): u를 처음 발견 (gray로 칠함).
  • f[u] (finish time): u의 모든 후손 처리 완료 (black으로 칠함).
dfs_with_times.py
def dfs_timestamps(graph):
    n = len(graph)
    color = ['WHITE'] * n      # white = 미방문, gray = 발견, black = 완료
    d = [0] * n              # discovery time
    f = [0] * n              # finish time
    parent = [-1] * n
    time = [0]

    def visit(u):
        time[0] += 1
        d[u] = time[0]
        color[u] = 'GRAY'
        for v in graph[u]:
            if color[v] == 'WHITE':
                parent[v] = u
                visit(v)
        color[u] = 'BLACK'
        time[0] += 1
        f[u] = time[0]

    for u in range(n):
        if color[u] == 'WHITE':
            visit(u)

    return d, f, parent

Timestamps는 단순 순회 이상의 정보 — edge classification, topological sort, SCC의 토대.

📖 원서 인용 — CLRS 4판, §20.3 "Depth-first search"

"Besides creating a depth-first forest, depth-first search also timestamps each vertex. Each vertex v has two timestamps: the first timestamp v.d records when v is first discovered (and grayed), and the second timestamp v.f records when the search finishes examining v's adjacency list (and blackens v). These timestamps provide important information about the structure of the graph and are generally helpful in reasoning about the behavior of depth-first search."

08 — Edge Classification: Tree / Back / Forward / Cross

전공자

DFS가 만드는 4가지 간선 종류

DFS 도중 (u, v) 간선을 마주칠 때 v의 색상으로 분류:

v의 색간선 종류의미
WHITETree edgeDFS 트리의 일부. v를 처음 발견.
GRAYBack edgev는 u의 조상. 사이클 존재 신호!
BLACK + d[u] < d[v]Forward edgev는 u의 후손 (Tree edge 아닌 다른 path).
BLACK + d[u] > d[v]Cross edgev는 다른 subtree에 속함.

Undirected graph에서는 Tree edge와 Back edge만 (Forward, Cross 없음).

전공자

Back edge = 사이클의 직접 증거

Theorem: 그래프에 사이클 ⟺ DFS에 back edge 존재.

증명:

  • (⟸) Back edge (u, v)는 v → ... → u라는 tree path와 u → v라는 back edge를 합쳐 cycle 형성.
  • (⟹) 사이클 v₀ → v₁ → ... → vₖ = v₀이 있다면, DFS에서 가장 먼저 발견되는 vᵢ를 보자. vᵢ가 vᵢ₋₁을 발견하기 전에 다 처리해야 vᵢ₋₁이 white에서 시작 — 모순. 따라서 (vᵢ₋₁, vᵢ) 마주칠 때 vᵢ는 gray → back edge.

이 성질이 사이클 감지(§11) 알고리즘의 토대.

09 — 위상 정렬 (CLRS §20.4): DFS vs Kahn

고급

정의

위상 정렬(topological sort): DAG의 정점들을 일렬로 나열, 모든 간선 (u, v)에 대해 u가 v보다 앞에 오도록.

존재 조건: DAG여야 함. 사이클 있으면 위상 정렬 불가능 (a→b→a라면 둘 다 앞일 수 없음).

유일하지 않음: 일반적으로 여러 valid한 위상 정렬 존재.

고급

방법 1 — DFS 기반 (CLRS §20.4)

아이디어: DFS 후 finish time 내림차순으로 정렬. Post-order의 reverse.

topo_dfs.py
def topological_sort_dfs(graph):
    visited = set()
    order = []

    def dfs(u):
        visited.add(u)
        for v in graph[u]:
            if v not in visited:
                dfs(v)
        order.append(u)         # post-order

    for u in graph:
        if u not in visited:
            dfs(u)

    return order[::-1]            # reverse

왜 작동하는가: DFS finish 순서가 토폴로지 역순. u→v 간선이면 v가 먼저 finish (u의 DFS 안에서 v 처리됨). 그래서 reverse하면 u가 v보다 앞.

주의: 사이클 있으면 잘못된 결과. 사전에 사이클 검사 또는 DFS 중 back edge 감지.

고급

방법 2 — Kahn's Algorithm (BFS 기반)

아이디어: in-degree 0인 정점부터 순차 처리. 그 정점 제거 후 인접 정점들의 in-degree 감소. 반복.

topo_kahn.py
from collections import deque

def topological_sort_kahn(graph, n):
    in_degree = [0] * n
    for u in range(n):
        for v in graph[u]:
            in_degree[v] += 1

    queue = deque(u for u in range(n) if in_degree[u] == 0)
    order = []

    while queue:
        u = queue.popleft()
        order.append(u)
        for v in graph[u]:
            in_degree[v] -= 1
            if in_degree[v] == 0:
                queue.append(v)

    if len(order) != n:
        raise ValueError("사이클 존재! DAG가 아님")
    return order

장점: 사이클 감지 자동 (n개 미만이면 사이클). 명확하고 직관적. iterative라 깊은 그래프 안전.

Arthur B. Kahn (1962) "Topological sorting of large networks" (Communications of the ACM, 5(11), 558–562)에서 제시.

WHY — DFS와 Kahn 비교
측면DFS 기반Kahn (BFS 기반)
구현 복잡도중간 (재귀)중간 (in_degree 계산)
사이클 감지별도 검사 필요자동 (order < n)
출력 순서finish time 역순가능한 한 early 정점 먼저
병렬화어려움가능 (in-degree 0 정점들 동시 처리)

10 — SCC (CLRS §20.5): Strongly Connected Components

전공자

정의

SCC: directed graph에서 "서로 도달 가능한" 정점들의 최대 부분집합. 즉 모든 u, v ∈ C에 대해 u → ... → v와 v → ... → u 모두 존재.

Undirected에서는 connected component와 같음. Directed에서는 더 미묘 — 두 정점이 한 방향으론 이어져도 반대 방향이 없으면 다른 SCC.

전공자

Kosaraju 알고리즘 — 두 번의 DFS

  1. 원본 graph G에 DFS, finish time 계산.
  2. 모든 간선 뒤집어 transpose graph G^T 만들기.
  3. G^T에 DFS — finish time 내림차순으로 시작 정점 선택. 한 DFS에서 도달한 정점들이 한 SCC.
kosaraju.py
def kosaraju(graph, n):
    # Step 1: 첫 DFS로 finish time 순서
    visited = [False] * n
    order = []

    def dfs1(u):
        visited[u] = True
        for v in graph[u]:
            if not visited[v]: dfs1(v)
        order.append(u)        # finish 시점

    for u in range(n):
        if not visited[u]: dfs1(u)

    # Step 2: transpose
    rev_graph = [[] for _ in range(n)]
    for u in range(n):
        for v in graph[u]:
            rev_graph[v].append(u)

    # Step 3: 역순으로 두 번째 DFS
    visited = [False] * n
    sccs = []

    def dfs2(u, comp):
        visited[u] = True
        comp.append(u)
        for v in rev_graph[u]:
            if not visited[v]: dfs2(v, comp)

    while order:
        u = order.pop()
        if not visited[u]:
            comp = []
            dfs2(u, comp)
            sccs.append(comp)

    return sccs

O(V + E). Edges 두 번 순회, 정점 두 번 visited.

전공자

Tarjan SCC — 한 번의 DFS

Robert Tarjan (1972). DFS 한 번 + low-link 값으로 SCC 식별. 이론적으로 우아하지만 구현 복잡.

  • 각 정점에 low-link 값: "DFS 트리에서 도달 가능한 가장 낮은 (먼저 발견된) 정점의 d 값".
  • low-link[u] = d[u]면 u는 SCC의 root.
  • stack에 정점 push, root 발견 시 그 정점들 pop = 한 SCC.

O(V + E). 한 번의 DFS만 — Kosaraju보다 약 2배 빠름. 실무에서 더 자주 사용.

📖 원서 인용 — CLRS 4판, §20.5 "Strongly connected components"

"Many algorithms that work with directed graphs begin with such a decomposition. After decomposing the graph into strongly connected components, such algorithms run on each strongly connected component separately and then combine the solutions according to the structure of connections among components."

11 — 사이클 감지: directed vs undirected

고급

Directed graph — back edge로 감지

cycle_directed.py
def has_cycle_directed(graph, n):
    color = ['WHITE'] * n

    def dfs(u):
        color[u] = 'GRAY'
        for v in graph[u]:
            if color[v] == 'GRAY':
                return True          # back edge → 사이클
            if color[v] == 'WHITE' and dfs(v):
                return True
        color[u] = 'BLACK'
        return False

    for u in range(n):
        if color[u] == 'WHITE' and dfs(u):
            return True
    return False
고급

Undirected graph — 부모 추적

Undirected는 더 미묘. (u, v) 간선을 따라 v로 갔는데 v에서 u로 돌아오는 간선은 같은 간선이라 사이클 아님.

cycle_undirected.py
def has_cycle_undirected(graph, n):
    visited = [False] * n

    def dfs(u, parent):
        visited[u] = True
        for v in graph[u]:
            if not visited[v]:
                if dfs(v, u): return True
            elif v != parent:        # 부모 아닌 visited는 사이클
                return True
        return False

    for u in range(n):
        if not visited[u] and dfs(u, -1):
            return True
    return False

또는 Union-Find로 — Algo 17에서 다룸. 새 간선 추가 시 양 끝이 이미 같은 component이면 사이클.

AI — 4가지 흔한 실수
  1. BFS에 list.pop(0): O(V·E) 만들기. collections.deque 필수.
  2. Visited 체크 누락: 무한 루프 가능. visited 추가는 enqueue 시점이 좋음 (dequeue 시점은 같은 정점 여러 번 enqueue 가능).

word_ladder.py
from collections import deque

def word_ladder(begin, end, word_list):
    word_set = set(word_list)
    if end not in word_set: return 0

    q = deque([(begin, 1)])
    visited = {begin}

    while q:
        word, steps = q.popleft()
        if word == end: return steps
        for i in range(len(word)):
            for c in "abcdefghijklmnopqrstuvwxyz":
                if c == word[i]: continue
                next_word = word[:i] + c + word[i+1:]
                if next_word in word_set and next_word not in visited:
                    visited.add(next_word)
                    q.append((next_word, steps + 1))

    return 0


print(word_ladder("hit", "cog", ["hot","dot","dog","lot","log","cog"]))
# 출력: 5 (hit → hot → dot → dog → cog)

실습 2 — Course Schedule (LeetCode #207)

"n개의 강의와 선수과목 관계가 주어졌을 때 모든 강의를 들을 수 있는가?"

course_schedule.py
from collections import deque

def can_finish(num_courses, prerequisites):
    # prereq[i] = [course, prerequisite]
    graph = [[] for _ in range(num_courses)]
    in_degree = [0] * num_courses
    for course, prereq in prerequisites:
        graph[prereq].append(course)
        in_degree[course] += 1

    # Kahn's algorithm
    q = deque(c for c in range(num_courses) if in_degree[c] == 0)
    completed = 0
    while q:
        c = q.popleft()
        completed += 1
        for next_c in graph[c]:
            in_degree[next_c] -= 1
            if in_degree[next_c] == 0:
                q.append(next_c)

    return completed == num_courses

토폴로지 정렬 가능 ⟺ DAG ⟺ 모든 강의 수강 가능. 사이클 있으면 일부 강의 in-degree가 영원히 0 안 됨.

풀이 펼치기
답

인접 리스트 (C 언어 기준): O(V+E), 각 edge 8바이트 × 2(undirected) ≈ 16MB. Pure Python list[list[int]]은 int 객체 헤더(~28B) 때문에 수배 큼.

from collections import deque
def is_bipartite(graph, n):
    color = [-1] * n
    for start in range(n):
        if color[start] != -1: continue
        color[start] = 0
        q = deque([start])
        while q:
            u = q.popleft()
            for v in graph[u]:
                if color[v] == -1:
                    color[v] = 1 - color[u]
                    q.append(v)
                elif color[v] == color[u]:
                    return False
    return True

def count_components(graph, n):
    visited = set()
    count = 0
    def dfs(u):
        stack = [u]
        while stack:
            x = stack.pop()
            if x in visited: continue
            visited.add(x)
            for y in graph[x]:
                if y not in visited: stack.append(y)
    for u in range(n):
        if u not in visited:
            dfs(u)
            count += 1
    return count

from collections import deque
def distances_from(graph, n, source):
    dist = [-1] * n
    dist[source] = 0
    q = deque([source])
    while q:
        u = q.popleft()
        for v in graph[u]:
            if dist[v] == -1:
                dist[v] = dist[u] + 1
                q.append(v)
    return dist   # -1 = unreachable

정점들의 시퀀스 (간선으로 연결) / 시작=끝인 path.
Connected Component
서로 도달 가능한 정점 집합 (undirected).
Strongly Connected (SCC)
directed에서 양방향 도달 가능. Tarjan/Kosaraju O(V+E).
Adjacency List
각 정점의 인접 정점들 list. O(V+E) 공간. Sparse 표준.
Adjacency Matrix
V×V matrix. O(V²) 공간. Dense에 적합.
BFS
Breadth-First Search. queue 기반. 비가중 최단 경로. O(V+E).
DFS
Depth-First Search. stack/재귀. timestamps로 다양한 분석.
Discovery / Finish Time
DFS에서 정점 발견/완료 시점. d[u], f[u].
Tree Edge
DFS 트리의 일부. 새 정점 발견.
Back Edge
조상으로의 간선. 사이클의 직접 증거.
Forward / Cross Edge
후손/다른 subtree로의 간선. Directed에서만.
Topological Sort
DAG 정점의 선형 순서. 모든 (u, v)에서 u가 v 앞.
Kahn's Algorithm
BFS 기반 위상 정렬. in-degree 0부터. 1962 Arthur Kahn.
Tarjan SCC
1972 Tarjan. 한 번의 DFS + low-link로 SCC.
Kosaraju SCC
두 번의 DFS (원본 + transpose) 로 SCC.
Transpose Graph G^T
모든 간선 방향 반전. Kosaraju에 사용.
Bipartite
정점을 두 그룹으로 나눠 같은 그룹 내 간선 없음. BFS 색칠로 검사.
Sparse / Dense
m=O(n) / m=Θ(n²). 자료구조 선택의 기준.
CLRS §20
Elementary Graph Algorithms (4판). BFS/DFS/topological/SCC 모두 포함.
