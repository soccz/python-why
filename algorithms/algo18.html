ALGO 18

네트워크 플로우 — Max-Flow, Min-Cut
냉전 철도망에서 태어난 쌍대성의 알고리즘

1955년, 미 공군 연구자 T. E. Harris가 RAND에 문제 하나를 던집니다: "소련 철도망의 병목은 어디인가?" 이를 수학화한 것이 Ford & Fulkerson의 "Maximal Flow through a Network"(1956). Bellman-Ford의 그 Ford, 그 RAND — 같은 시기 같은 장소에서 태어난 두 번째 그래프 걸작.
Max-Flow = Min-Cut이라는 놀라운 쌍대성. 그리고 이 정리가 이분 매칭, 이미지 segmentation, 스포츠 순위, LP duality까지 연결.
핵심 질문: "왜 '증가 경로가 없으면 최적'이 참인가? 왜 흐름의 최댓값이 컷의 최솟값과 같은가?"

01 — 이번 주 이야기: RAND의 철도망 문제

"Consider a rail network connecting two cities by way of a number of intermediate cities, where each link of the network has a number assigned to it representing its capacity. Assuming a steady state condition, find a maximal flow from one given city to the other."
— T. E. Harris, 문제 제기 (1955, RAND)

1955 — T. E. Harris, 미 공군 연구자

냉전 초기. 미 공군은 "소련 철도망에서 가장 얇은 병목"을 알고 싶었음. Harris는 이를 수학 문제로 추상화 — 각 철도 구간에 최대 열차 수(용량)가 있고, 발산 도시에서 도착 도시까지 총 얼마나 많은 열차를 보낼 수 있는가?

1956 — Lester R. Ford Jr. & Delbert R. Fulkerson, RAND Corporation

Harris의 문제에 대한 답: "Maximal Flow through a Network"(Canadian J. Math. 8:399-404, 1956). 같은 Ford 이름에 주목 — Algo 16의 Bellman-Ford의 그 Ford Jr. 동일인. RAND에서 같은 시기에 네트워크 흐름 이론의 두 축(최단 경로, 최대 흐름)을 함께 구축.

이 논문에서 Max-Flow Min-Cut 정리가 처음 증명. "가장 많이 흘릴 수 있는 양 = 가장 얇은 컷의 용량" — 수학적으로 아름답고 실무에 강력한 쌍대성.

1970 — Yefim Dinitz, 소련

모스크바의 대학원생 Dinitz가 Ford-Fulkerson의 지수 시간 가능성을 극복한 layer graph + blocking flow 접근으로 O(V²E) 달성. 소련 수학 저널에 발표돼 서방에 늦게 알려짐 — 1974년 Shimon Even & Alon Itai가 영어권 회의 논문에서 소개하면서 널리 전파. 교과서에 "Dinic's algorithm"로 표기되는 경우가 많지만, 올바른 Roman 표기는 Dinitz.

1972 — Jack Edmonds & Richard Karp

"Theoretical Improvements in Algorithmic Efficiency for Network Flow Problems"(J. ACM 19:248-264, 1972). Ford-Fulkerson에 "증가 경로는 BFS로 찾으라"는 단 한 줄의 수정으로 O(VE²)의 다항 보장. Dinitz 1970을 몰랐던 서구가 독립적으로 도달한 결과.

1988 — Andrew V. Goldberg & Robert E. Tarjan

Push-Relabel. 증가 경로 패러다임을 버리고 preflow를 도입. 각 정점에 "초과분"을 허용, 국소적 push와 relabel 연산만으로 O(V²E) 달성. STOC 1986에서 발표되고 J. ACM 1988로 공식화. 현대 고성능 max-flow 라이브러리의 기반 (예: LEMON, BGL).

2013년에는 James Orlin이 O(VE) — 이론적으로 sparse 그래프 최고 — 을 달성 (J. ACM 60:1).

Max-Flow 알고리즘의 진화 — 냉전 초기 군사 응용에서 시작, Dinitz/EK로 다항 보장, Goldberg-Tarjan push-relabel로 실무 표준, Orlin 2013 O(VE)로 이론적 정점. 같은 Ford가 1956에 BF·FF 둘 다 만든 해.

02 — 문제 정의 — 용량, 흐름, 보존 (CLRS §24.1)

초급

"파이프와 물" 비유

도시들 사이에 물 파이프가 깔려있다. 각 파이프는 최대 허용량(용량)이 있다. 발원지(source) s에서 출발해 하구(sink) t까지 얼마나 많은 물을 보낼 수 있는가?

  • 각 파이프의 흐름은 용량을 넘을 수 없음.
  • 중간 도시에서는 들어온 만큼 나가야 함 (보존 법칙).
  • s에서 나가는 총량 = t로 들어오는 총량 = "네트워크 전체 흐름량".
  • 목표: 흐름량 최대화.
중급

형식 정의

흐름 네트워크: 유향 그래프 G = (V, E), 용량 함수 c: E → ℝ≥0, 소스 s, 싱크 t.

흐름(flow) f: E → ℝ는 두 조건을 만족:

  1. 용량 제약: 모든 간선 (u,v)에 대해 0 ≤ f(u,v) ≤ c(u,v).
  2. 흐름 보존: 모든 정점 v ≠ s, t에 대해 ∑u f(u,v) = ∑w f(v,w) (들어옴 = 나감).

흐름의 값: |f| = ∑v f(s,v) − ∑v f(v,s) (s에서 나가는 순 흐름). 이를 최대화.

왼쪽은 용량만, 오른쪽은 최대 흐름. 모든 간선이 용량을 꽉 채워 사용됨 (saturated). s에서 나가는 5 = t로 들어오는 5. 중간 정점 A, B도 들어옴=나감.

03 — 증가 경로 (Augmenting Path)

중급

"아직 여유 있는 길" 찾기

현재 흐름이 어떻든, s에서 t까지 가는 경로에서 모든 간선이 아직 용량 여유가 있다면, 그 경로로 추가 흐름을 보낼 수 있다. 경로의 병목 간선 = 가장 작은 여유 용량 = 얼마만큼 더 보낼 수 있는지.

이 경로를 증가 경로(augmenting path)라 부름. Ford-Fulkerson의 핵심 아이디어: "증가 경로가 없을 때까지 반복해서 흘리기".

  • 흐름 초기화: 모든 f(u,v) = 0.
  • 반복: s→t 증가 경로 P를 찾는다. 병목 b = min(u,v)∈P (c(u,v) − f(u,v)).
  • P 따라 b만큼 흐름 증가.
  • 증가 경로가 없으면 종료.
고급

왜 "반대 방향"도 고려해야 하나

단순히 "아직 여유 있는 정방향 간선만" 따라가면 잘못된 선택을 되돌릴 수 없음. 예: s→A→B→t를 먼저 꽉 채우면, 나중에 s→B→C→t 같은 더 좋은 흐름을 찾아도 A-B 간선을 되돌려야 할 수 있음.

해결: "이미 흐르고 있는 간선의 반대 방향"도 "상쇄" 경로로 허용. 이것을 위한 개념이 잔여 그래프(residual graph) — 다음 섹션.

04 — 잔여 그래프 (Residual Graph)

고급

정의

흐름 f에 대한 잔여 그래프 Gf는 두 종류의 간선을 가짐:

  • 전방(forward) 잔여 간선 (u,v): cf(u,v) = c(u,v) − f(u,v). 아직 보낼 수 있는 여유.
  • 후방(backward) 잔여 간선 (v,u): cf(v,u) = f(u,v). 이미 흐르는 만큼 "취소" 가능.

Gf에서 s→t 경로가 있으면 그것이 증가 경로. 반대 방향 잔여 간선을 타고 가면 기존 흐름을 "상쇄"하는 효과.

5단계 Ford-Fulkerson 진행: 증가 경로를 반복 찾아 흐름을 누적. 3번의 경로 (2+2+1) = 5. 마지막에 증가 경로 없음 → 최적성 보장 (Max-Flow Min-Cut 정리).

05 — Ford-Fulkerson 알고리즘 (CLRS §24.2)

고급

의사코드

FORD-FULKERSON(G, s, t)
1. for each edge (u,v) ∈ G.E: f(u,v) = 0
2. while Gf에 s→t 경로 P 존재
3.    cf(P) = min(u,v)∈P cf(u,v)
4.    for each edge (u,v) ∈ P:
5.      if (u,v) ∈ E: f(u,v) += cf(P)
6.      else: f(v,u) −= cf(P) // 후방 간선
7. return f
  • 경로 탐색 방법은 명시 안 함: DFS/BFS/임의 선택 모두 유효 — 각각 복잡도 다름.
  • 정수 용량이면 매 반복에 흐름이 최소 1 증가 → 최대 흐름 |f*|번 반복 → O(E · |f*|).
  • 무리수 용량: 수렴 안 할 수 있음 (Zwick 반례).
전공자

정당성 — 증가 경로 없음 ⟹ 최적

보조정리: 흐름 f에 대해 Gf에 s→t 경로가 없다면, f는 최대 흐름이다.

증명: Gf에서 s에서 도달 가능한 정점 집합을 S, 나머지를 T = V − S라 하자. t ∈ T (경로 없음). 컷 (S, T)를 고려:

  1. S→T 간선은 잔여 용량 0 → f(u,v) = c(u,v) (꽉 참).
  2. T→S 간선은 잔여 그래프에 반대 방향이 없어야 함 → f(v,u) = 0 (0 흐름).
  3. 따라서 |f| = ∑(u,v)∈(S,T) c(u,v) = 컷 용량.
  4. 임의 흐름 ≤ 임의 컷 용량 (weak duality) → f가 이 컷 달성 → 최적. □

06 — Ford-Fulkerson Python 구현

전공자

DFS 기반 구현

ford_fulkerson.py
from collections import defaultdict

def ford_fulkerson(graph_cap, s, t):
    """
    graph_cap: {u: {v: capacity}} — 인접 딕셔너리.
    반환: max_flow (int). 잔여 그래프가 필요하면 residual도 반환하도록 확장.
    """
    # 잔여 그래프로 복사 (원본 불변)
    residual = defaultdict(lambda: defaultdict(int))
    for u in graph_cap:
        for v, cap in graph_cap[u].items():
            residual[u][v] += cap    # 병렬 간선 용량 합산

    def dfs(u, bottleneck, visited):
        if u == t: return bottleneck
        visited.add(u)
        for v, cap in list(residual[u].items()):
            if v not in visited and cap > 0:
                pushed = dfs(v, min(bottleneck, cap), visited)
                if pushed > 0:
                    residual[u][v] -= pushed
                    residual[v][u] += pushed    # 후방 잔여 간선
                    return pushed
        return 0

    max_flow = 0
    while True:
        pushed = dfs(s, float('inf'), set())
        if pushed == 0: break
        max_flow += pushed

    return max_flow


# 테스트
graph = {
    's': {'A': 3, 'B': 2},
    'A': {'B': 1, 't': 2},
    'B': {'t': 3},
    't': {},
}
print(ford_fulkerson(graph, 's', 't'))    # 5
  • 잔여 그래프: residual[u][v]는 u→v의 잔여 용량. 원래 없는 간선도 후방으로 생길 수 있어 defaultdict.
  • 후방 간선 업데이트: residual[v][u] += pushed — 상쇄 가능하도록.
  • DFS 재귀: 경로의 병목을 재귀적으로 전달. 최악 O(E · |f*|) 반복.
⚠️ DFS의 함정

DFS는 때로 최악 경로를 고르기도 함. 예: 용량이 1, 1000, 1000인 간선들로 구성된 특정 그래프에서 1인 병목을 1번 찾고, 반대 방향을 이용해 1을 왕복 — 각 반복이 1씩만 증가. 해결: BFS (Edmonds-Karp) — 최단 경로 우선 → O(VE²).

07 — Max-Flow Min-Cut 정리 (쌍대성)

고급

정리 — 흐름과 컷의 쌍대성

Max-Flow Min-Cut Theorem (Ford & Fulkerson 1956):

maxf |f| = min(S,T) : s∈S, t∈T   ∑(u,v)∈(S,T) c(u,v)

즉, "가장 많이 보낼 수 있는 흐름 = 가장 얇은 컷의 용량". 이는 선형 계획법 쌍대성(LP duality)의 한 인스턴스.

3프레임 쌍대성: 최대 흐름 5 = CUT 1 용량 5 = CUT 2 용량 5. 최대 흐름은 어떤 컷 용량보다도 작거나 같고 (weak duality), 정확히 하나의 컷과 일치한다 (strong duality). LP duality의 그래프 이론적 구현.
📖 원서 인용 — CLRS 4판, Theorem 24.7

"The following are equivalent: (1) f is a maximum flow in G. (2) The residual network Gf contains no augmenting paths. (3) |f| = c(S, T) for some cut (S, T) of G."

08 — Edmonds-Karp — BFS로 다항 보장

고급

핵심 통찰

Edmonds & Karp 1972의 단 한 줄 수정: "증가 경로를 BFS로 찾아 항상 최단". 이게 왜 중요한가?

BFS는 정점 u의 s로부터의 거리를 단조 증가시킴 (흐름 갱신 시). 각 간선이 "포화(saturated)"되는 횟수 ≤ V/2번. 총 증가 경로 수 ≤ O(VE). 각 BFS O(E) → 전체 O(VE²).

  • 용량이 무리수여도 유한 종료 보장.
  • 구현은 FF의 DFS를 BFS로만 바꾸면 됨.
  • 실무에서 자주 쓰이는 "깨끗한" 구현.
전공자

Python 구현

edmonds_karp.py
from collections import defaultdict, deque

def edmonds_karp(graph_cap, s, t):
    residual = defaultdict(lambda: defaultdict(int))
    for u in graph_cap:
        for v, cap in graph_cap[u].items():
            residual[u][v] += cap

    def bfs():
        """s→t 최단 증가 경로 (parent 맵 반환)."""
        parent = {s: None}
        q = deque([s])
        while q:
            u = q.popleft()
            if u == t: return parent
            for v, cap in residual[u].items():
                if v not in parent and cap > 0:
                    parent[v] = u
                    q.append(v)
        return None

    max_flow = 0
    while True:
        parent = bfs()
        if parent is None or t not in parent: break

        # 병목 찾기
        bottleneck = float('inf')
        cur = t
        while parent[cur] is not None:
            bottleneck = min(bottleneck, residual[parent[cur]][cur])
            cur = parent[cur]

        # 흐름 갱신
        cur = t
        while parent[cur] is not None:
            u = parent[cur]
            residual[u][cur] -= bottleneck
            residual[cur][u] += bottleneck
            cur = u

        max_flow += bottleneck

    return max_flow


print(edmonds_karp(graph, 's', 't'))    # 5
  • 복잡도: O(VE²). V=1000, E=10000이면 ~10¹¹ — 엄청난 상한. 실측은 훨씬 빠름.
  • BFS로 최단 경로 선택 → 각 간선의 "포화 횟수"가 O(V)로 제한.

09 — Dinitz — Blocking Flow와 Layer Graph

전공자

1970년의 한 단계 더

Dinitz의 아이디어: Edmonds-Karp는 한 반복에 하나의 경로만 처리. 하지만 같은 길이의 증가 경로들을 한꺼번에 처리하면 더 빠름.

  1. Layer Graph: BFS로 각 정점의 s-거리 계산, 거리가 정확히 1 증가하는 간선만 남긴 DAG.
  2. Blocking Flow: Layer Graph에서 더 이상 s→t 경로가 없을 때까지 최대한 흘리기.
  3. 각 phase마다 s-t 최단 거리가 증가 → 최대 V phase → 전체 O(V²E).
  4. 특수 그래프에서 더 빠름: 단위 용량 + sparse → O(E√V), 이분 매칭 → O(E√V) (Hopcroft-Karp의 근원).
💡 이름 주의 — Dinitz vs Dinic

원저자 Yefim Dinitz(러시아어 Диниц). 서방 교재에 "Dinic"로 잘못 표기된 것이 굳어짐. 2006년 그가 직접 "Dinitz' algorithm: the original version and Even's version" (Springer)에서 원래 이름을 재확인.

전공자

알고리즘 스케치

DINITZ(G, s, t)
1. while true:
2.    level = BFS(Gf, s) // s-거리 계산
3.    if level[t] = ∞: return f // 도달 불가
4.    layer_graph = Gf에서 level[u]+1 = level[v]인 간선만
5.    f = f + blocking_flow(layer_graph, s, t)
6. return f

10 — 이분 매칭으로의 환원 (CLRS §24.3)

중급

문제 — 직원과 업무 배정

직원 N명, 업무 M개. 각 직원은 특정 업무만 할 수 있음(자격). 한 직원은 한 업무만, 한 업무엔 한 직원만. 가능한 최대 매칭은?

핵심 관찰: 이 문제는 max-flow로 환원할 수 있음!

3프레임 이분 매칭 환원: 이분 그래프 → source/sink 추가 (모든 간선 용량 1) → Max-Flow 실행 → 흐름이 1인 W-J 간선들이 매칭. 이 환원 덕에 max-flow 알고리즘이 사람 배정, 스케줄링, 결혼 문제를 모두 푼다.
고급

최적화 — Hopcroft-Karp 1973

이분 매칭 특화 알고리즘. Dinitz의 layer graph 아이디어를 매칭에 적용 → O(E√V). 일반 max-flow보다 빠름.

응용: 온라인 광고 입찰(AdX), 신장 이식 짝 맞추기(kidney exchange), 학교 배정 등 "양쪽 1:1" 문제.

11 — Push-Relabel — Goldberg-Tarjan 1988

전공자

증가 경로 패러다임을 버리다

Ford-Fulkerson, Edmonds-Karp, Dinitz 모두 "s→t 증가 경로 탐색" 기반. Goldberg-Tarjan 1988은 완전히 다른 길: preflow 허용.

  • Preflow: 각 정점에 "초과분"(excess, 들어옴 > 나감) 허용. 흐름 보존 법칙을 일시 완화.
  • Height 함수 h(v): "위→아래로만 push 가능". h(s) = V (고정), h(t) = 0.
  • Push(u, v): h(u) > h(v)이고 잔여 용량 있으면, u의 excess를 v로 넘김.
  • Relabel(u): push 불가 시 u의 높이를 올림.
  • 모든 excess가 s 또는 t로 돌아가면 종료.

복잡도: 기본 O(V²E), FIFO 변형 O(V³), highest-label selection O(V²√E).

장점: 국소 연산만 사용 → 병렬화 / GPU 구현에 자연스러움. 현대 고성능 max-flow 라이브러리의 표준.

12 — 응용 — 이미지 segmentation, 스포츠 순위

고급

이미지 Segmentation — Boykov-Kolmogorov

전경(foreground)과 배경(background) 분할 문제를 min-cut으로 환원.

  • 픽셀 = 정점. 이웃 픽셀 간 간선 가중치 = "둘이 같은 영역일 가능성".
  • 가상 source(전경), sink(배경) 추가. 각 픽셀과 s, t에 "전경/배경 확률" 가중치 간선.
  • Min-Cut = 전경-배경 경계 — 최적 분할.
  • Boykov-Kolmogorov 2004가 이미지 그래프에 특화한 max-flow 알고리즘 제안. OpenCV cv2.grabCut은 Rother–Kolmogorov–Blake 2004의 GrabCut 알고리즘이 본체이며, 내부 min-cut 해법으로 BK max-flow를 사용.
고급

스포츠 순위 — "우승 가능성"

시즌 중간에 팀 A가 수학적으로 우승 가능한가? 남은 경기를 max-flow로 모델링.

  • 남은 경기 (i,j)를 정점으로, source에서 해당 경기로 용량 = 남은 경기 수.
  • 경기 정점에서 팀 i, j로 용량 = ∞.
  • 팀 → sink 용량 = (A의 최대 가능 승수) − (현재 승수).
  • Max-Flow = 남은 경기 수이면 A가 우승 가능, 아니면 수학적으로 탈락.
  • MLB 정규 시즌 중 플레이오프 진출 판정에 실제 사용.
전공자

프로젝트 선택 (Picard 1976)

프로젝트 각각이 수익(양수) 또는 비용(음수). 프로젝트 간 의존성(어떤 것을 고르려면 다른 것도 골라야). 최대 이익?

Min-Cut 환원: 수익 프로젝트와 source 사이, 비용 프로젝트와 sink 사이 간선. 의존성 ∞ 간선. Min-Cut 값 = (총 수익) − (최대 이익).

13 — AI와 Network Flow

중급

ML/AI에서의 응용

  • Markov Random Field (MRF) inference: binary MRF의 MAP estimation → graph cut. 이미지 노이즈 제거, stereo matching.
  • 구조 학습: Bayesian network의 최적 구조 찾기에 min-cut 환원.
  • Optimal transport: 분포 간 최소 이동 비용 → 특수한 flow 문제.
  • Neural network pruning: 일부 알고리즘이 network flow로 뉴런 중요도 추정.
전공자

AI가 Max-Flow를 잘못 쓰는 패턴

  • 증가 경로 미선택 기준: "그냥 DFS"라고 제안 → 용량이 큰 수인 경우 느릴 수 있음. BFS(EK) 또는 Dinitz 권장.
  • 후방 간선 누락: "잔여 그래프에서 정방향만" — 잘못된 해. 반드시 후방 간선 허용 필요.
  • 방향 그래프 가정 누락: "무방향 max-flow" 질문 — 각 간선을 두 방향으로 바꿔서 해결.
  • 이분 매칭에 일반 max-flow: 작동은 하지만 Hopcroft-Karp O(E√V)가 훨씬 빠름.
  • min-cut 출력 요청에 max-flow만 반환: S = {s에서 잔여 그래프로 도달 가능한 정점}를 잊고 넘어감.

교정 질문: "후방 간선 업데이트 코드 보여줘", "이분 매칭이면 Hopcroft-Karp가 더 나을까?"

14 — 연습문제 (12문제)

WHY 문제 1

Ford-Fulkerson이 정수 용량에선 종료 보장되지만, 무리수 용량에선 왜 안 되나?

풀이 펼치기
답

정수 용량이면 매 증가 경로에 흐름 ≥ 1 → 최대 |f*|번 반복 후 종료. 무리수 용량에선 증가량이 점점 줄어들며 정답에 수렴만 할 뿐 종료 안 할 수 있음. Zwick 1995가 제시한 최소 반례 — 6개 정점에서 무한 루프 가능. BFS(EK) 쓰면 이 문제 해결.

PREDICT 문제 2

무방향 그래프의 max-flow는 어떻게 계산하나?

풀이 펼치기
답

각 무방향 간선 {u,v}(용량 c)를 두 개의 유향 간선 (u,v)와 (v,u) 각각 용량 c로 치환. 그 후 max-flow 알고리즘. 흐름은 한쪽 방향으로만 흐름 (두 방향 동시 양수가 최적에 포함될 수 없음 — 서로 상쇄 가능).

BUILD 문제 3

Max-Flow 종료 후 Min-Cut을 복원하는 방법.

풀이 펼치기
답

최종 잔여 그래프 Gf에서 s로부터 BFS → 도달 가능한 정점 집합 S. Min-Cut = (S, V−S). 컷을 건너는 원 그래프의 정방향 간선들이 "saturated edges" = min-cut의 간선들.

DESUGAR 문제 4

왜 Edmonds-Karp의 반복 횟수가 O(VE)로 제한되나?

풀이 펼치기
답

BFS로 최단 경로만 고르면 각 간선이 "포화"되는 횟수가 ≤ V/2. 왜? 포화된 후 역방향으로 다시 쓰려면 s-거리가 증가해야 함. 거리는 V보다 클 수 없음 → 각 간선 포화 ≤ V/2번. 총 포화 이벤트 ≤ E · V/2. 각 반복에 최소 1개 간선 포화 → 반복 ≤ O(VE). 각 BFS O(E) → 전체 O(VE²). CLRS Lemma 24.10-12 증명.

WHY 문제 5

이분 매칭 환원에서 왜 모든 간선의 용량을 1로 두나?

풀이 펼치기
답

매칭의 정의: "한 정점에 최대 1개 간선". s→W의 용량 1은 W가 1개 매칭에만 쓰임 보장. J→t의 용량 1은 J의 제약. W→J 자체도 "사용됐다/안 됐다"의 이분값이라 1. 정수 용량 + FF는 반드시 정수 흐름 반환 (integrality theorem).

BUILD 문제 6

"정점에 용량 제한"이 있는 max-flow를 푸는 기법.

풀이 펼치기
답

정점 분할(vertex splitting): 용량 c인 정점 v를 vin과 vout 두 정점으로 분할, 사이에 용량 c 간선. v로 들어오던 간선은 vin, 나가던 간선은 vout에 연결. 이제 일반 max-flow로 풀 수 있음. 정점 수 2배, 간선 수 +V.

PREDICT 문제 7

V=10⁵, E=10⁶ 그래프에 EK, Dinitz, Push-Relabel 중 어느 것이 실측 빠를까?

풀이 펼치기
답

이론적 상한: EK O(VE²)=10¹⁷, Dinitz O(V²E)=10¹⁶, PR O(V²E)=10¹⁶. 하지만 실측은 Push-Relabel이 보통 가장 빠름 (특히 highest-label 변형). Boost Graph Library, LEMON 등 C++ 라이브러리 기본. EK는 코드 간결성이 장점.

WHY 문제 8

Max-Flow Min-Cut 정리가 LP duality의 인스턴스라는 의미.

풀이 펼치기
답

Max-Flow는 선형 계획(LP)로 표현 가능: 목적 함수 max |f|, 제약 조건 용량과 보존. 이 LP의 쌍대 문제가 바로 Min-Cut. LP 강쌍대성 정리(Strong Duality)에 의해 primal 최적값 = dual 최적값 → Max-Flow = Min-Cut. 이 관점이 Edmonds-Karp 1972 논문의 시작점이기도.

BUILD 문제 9

다중 소스/싱크 max-flow를 하나의 소스/싱크로 환원.

풀이 펼치기
답

가상 super source s', super sink t' 추가. s'→각 원 소스에 용량 ∞, 각 원 싱크→t'에 용량 ∞. 일반 max-flow 실행. 이 테크닉은 "여러 발전소 → 여러 도시 전력 송출" 같은 실무 문제의 표준.

DESUGAR 문제 10

Dinitz의 blocking flow가 왜 각 phase에 s-t 거리를 증가시키는가?

풀이 펼치기
답

Layer graph는 정확히 현재 s-t 최단 거리 d의 경로만 포함. Blocking flow 후 layer graph에 s-t 경로 없음 → 잔여 그래프에서 s-t 경로는 길이 > d. 각 phase마다 거리 ≥ 1 증가, 거리는 V 이하 → 최대 V phase. 각 phase O(VE) → 전체 O(V²E).

AI 문제 11

AI가 "이분 매칭에 일반 Ford-Fulkerson" 제안. 더 나은 답은?

풀이 펼치기
질문
  1. "이분 매칭은 단위 용량 + 2부 그래프 — 특수 구조 아닌가?"
  2. "Hopcroft-Karp 1973이 O(E√V)로 일반 FF보다 훨씬 빠름."
  3. "N=10⁴ 정도까진 FF도 OK, 이상이면 HK 필수."
WHY 문제 12

Max-Flow 알고리즘이 이미지 segmentation에 어울리는 이유.

풀이 펼치기
답

이미지의 "전경 vs 배경" 이진 분류는 두 영역 사이 경계(min-cut) 찾기와 같음. 픽셀 간 "연결 비용"을 잘 설계하면, min-cut이 자연스런 object 경계. Max-Flow Min-Cut 쌍대성 덕에 빠르게 풀 수 있음. Boykov-Kolmogorov 2004는 이미지 특화로 최적화.
