ALGO 23

분기 한정 (Branch & Bound)
상한/하한으로 NP-hard를 다스리기

1960년 런던정경대(LSE)의 Ailsa Land와 Alison Doig, 두 여성 수학자가 Econometrica에 정수계획(Integer Programming)을 처음으로 자동으로 푸는 알고리즘을 발표합니다. 3년 뒤 MIT의 Little, Murty, Sweeney, Karel이 이를 TSP에 적용하며 "branch and bound"라는 이름을 지음(1963).
단순 백트래킹(Algo 22)에 "이 가지는 최적보다 좋을 수 없다"는 판단(bounding)을 더한 것. CPLEX, Gurobi, SCIP 같은 현대 상용 solver — 제조업, 항공 스케줄링, 금융 포트폴리오 최적화 — 의 엔진.
핵심 질문: "왜 '정수계획은 NP-hard'인데 실무 MIP solver가 수천만 변수 문제를 푸는가?"

01 — 이번 주 이야기: LSE의 두 여성 수학자

"The purpose of this paper is to present a simple numerical algorithm for the solution of programming problems in which some or all of the variables can take only discrete values."
— Land & Doig, Econometrica, 1960

1960 — London School of Economics

Ailsa Land(1927-2021)과 Alison Doig(1929-2023, 후일 Harcourt)가 LSE에서 공동 연구. Econometrica 논문 "An Automatic Method of Solving Discrete Programming Problems"(Econometrica 28(3):497-520, 1960)에서 역사상 첫 범용 정수계획 solver 제시.

이 논문이 없었다면 제조·물류·금융의 수학적 최적화는 지금과 완전히 달랐을 것. Nobel 경제학상은 못 받았지만, 경제학·OR 분야의 판도를 바꾼 논문. 2021년 Land 타계 시 OR Society와 INFORMS가 특별 추도 기사 게재.

1963 — MIT, "Branch and Bound" 명명

John Little, Katta Murty, Dura Sweeney, Caroline Karel이 Operations Research 11(6):972-989에 "An Algorithm for the Traveling Salesman Problem" 발표. Land-Doig의 아이디어를 TSP에 맞게 재설계하며 "branch and bound"라는 용어를 처음 사용.

핵심 기법:

  • Branching: 전체 해 공간을 쪼갬 (예: "i-j 간선 포함" vs "i-j 간선 제외").
  • Bounding: 각 부분집합의 하한(lower bound)을 쉽게 계산. 현재 best보다 나쁘면 포기.
  • 행/열 축약(row/column reduction): TSP 거리 행렬에서 각 행·열의 최솟값을 빼서 하한 계산.

1965 이후 — MIP의 시대

  • Dakin 1965: Land-Doig를 일반 MIP(Mixed Integer Programming)로 확장.
  • Gomory 1958-63: 절단면 기법(cutting planes). 이후 B&B와 결합 → "branch-and-cut".
  • 1988 CPLEX 출시: 상용 MIP solver의 원조 (Bob Bixby 창립).
  • 2000년대: presolve, heuristics, primal/dual bounds가 발달 — 수만~수십만 변수 MIP도 해결 가능한 시대.
  • 2024 Gurobi: 1988년 이후 ~100배 알고리즘 개선 + 하드웨어 발전 = MIP가 1000배 이상 빨라짐.
B&B의 계보 — Land-Doig 1960 첫 자동 IP 해법, Little 1963 TSP 적용 및 명명, 1988 CPLEX가 상용 MIP 시대 개막, Gurobi·HiGHS로 이어지는 현대 최적화 인프라.

02 — B&B의 핵심 — 분기 + 한정

중급

두 요소의 결합

  1. Branching(분기): 전체 해 공간을 서로 배타적인 부분집합으로 나눔. 예: "변수 x₁ = 0인 해" vs "x₁ = 1인 해".
  2. Bounding(한정): 각 부분집합의 목적함수 상한 또는 하한을 빠르게 계산. 이 값이 현재 best보다 나쁘면 부분집합 전체 폐기.

원리:

  • 최소화 문제: "하한이 현재 best 이상이면 포기". 하한 = 이 부분집합의 어떤 해도 이보다 좋을 수 없음.
  • 최대화 문제: "상한이 현재 best 이하이면 포기". 상한 = 이 부분집합의 어떤 해도 이보다 좋지 않음.
  • bound가 tight(실제에 가까움)할수록 가지치기 효과 커짐.
고급

백트래킹과의 차이 — 한 줄 요약

백트래킹: "규칙 위반인가?"만 체크. B&B: "규칙 위반 아니어도 최적보다 나쁠 수 없는가?"까지 체크.

  • 백트래킹: feasibility pruning만.
  • B&B: feasibility + optimality pruning.
  • B&B는 최적화 문제 전용, 백트래킹은 해 탐색 일반.

03 — 상한/하한의 역할

고급

Primal vs Dual Bound

(최대화 관점)

  • Primal bound (하한): 지금까지 찾은 실현 가능 해(feasible) 중 최고. 목적함수의 하한이자 current best.
  • Dual bound (상한): LP relaxation 등으로 계산한 "이론적 최대". 실제로 달성 불가능할 수도.
  • Gap = (Dual − Primal) / Dual. gap=0이면 최적 증명 완료.

실무 MIP solver는 두 bound를 계속 업데이트하며 gap을 줄임. "5% optimality gap에서 중단"이 많은 실무 사용법.

전공자

좋은 bound의 조건

  1. 계산 빠름: 매 노드에서 반복 계산 — 너무 느리면 득보다 실.
  2. Tight(실제에 가까움): 느슨하면 가지치기 못 함.
  3. 단조성(monotonicity): 하위 노드의 bound는 부모보다 나쁘거나 같음 (최대화 기준).

예:

  • 0/1 배낭: LP relaxation = 분수 배낭. O(n log n)으로 상한 계산.
  • TSP: 행/열 축약(Little 1963) 또는 1-tree 하한 (Held-Karp 1970).
  • 스케줄링: LP relaxation 또는 그리디 추정.

04 — 0/1 배낭 B&B

고급

아이디어

Algo 19/21의 0/1 배낭. 아이템을 v/w 내림차순 정렬 후, 각 아이템을 "포함/미포함"으로 분기. 각 노드에서:

  • 상한(upper bound) = 현재까지 누적 가치 + 분수 배낭으로 남은 용량을 채웠을 때의 가치.
  • 이 상한이 best보다 작거나 같으면 가지 포기.
  • 리프(모든 아이템 결정)에서 best 갱신.

왜 분수 배낭이 상한인가: 0/1 배낭은 분수 배낭의 제약 강화판이므로, 분수 배낭의 최적값 ≥ 0/1 배낭의 최적값. LP relaxation 원리.

5단계 B&B 탐색 트리: 루트 UB=240 → 분기 → 각 노드에서 UB 계산 → 최종 해 220 발견. 순수 DFS라면 2³=8 리프 모두 탐색, B&B는 UB 기반 가지치기로 일부만 탐색.

05 — 0/1 배낭 Python 구현

전공자

Best-First B&B (우선순위 큐)

knapsack_bb.py
import heapq

def upper_bound(idx, cur_w, cur_v, W, items):
    """남은 아이템을 분수 배낭으로 채운 가치 (LP relaxation)."""
    w, v = cur_w, cur_v
    for i in range(idx, len(items)):
        vi, wi = items[i]
        if w + wi <= W:
            w += wi; v += vi
        else:
            return v + vi * (W - w) / wi    # 분수
    return v


def knapsack_bb(values, weights, W):
    """0/1 배낭 Branch & Bound. 반환: 최적 가치."""
    # v/w 내림차순 정렬 — bound 추정이 유리
    items = sorted(zip(values, weights), key=lambda x: -x[0] / x[1])
    n = len(items)

    # (−UB, counter, idx, cur_w, cur_v) — counter는 §10의 tie-break 패턴
    root_ub = upper_bound(0, 0, 0, W, items)
    counter = 0
    heap = [(-root_ub, counter, 0, 0, 0)]
    best = 0

    while heap:
        neg_ub, _, idx, w, v = heapq.heappop(heap)
        ub = -neg_ub

        if ub <= best: break    # 남은 모두 최선보다 나쁨 (best-first 장점)
        if idx == n:
            best = max(best, v)
            continue

        vi, wi = items[idx]

        # 포함 가지
        if w + wi <= W:
            new_v = v + vi
            best = max(best, new_v)    # 부분 실현가 (이후 못 더해도 최소 new_v 확보)
            new_ub = upper_bound(idx + 1, w + wi, new_v, W, items)
            if new_ub > best:
                counter += 1
                heapq.heappush(heap, (-new_ub, counter, idx + 1, w + wi, new_v))

        # 제외 가지
        new_ub = upper_bound(idx + 1, w, v, W, items)
        if new_ub > best:
            counter += 1
            heapq.heappush(heap, (-new_ub, counter, idx + 1, w, v))

    return best


print(knapsack_bb([60, 100, 120], [10, 20, 30], 50))    # 220
  • Best-First 장점: 큐에서 UB 최대인 노드를 먼저 pop. UB ≤ best면 즉시 종료 (나머지 더 볼 필요 없음).
  • DFS 대비: 메모리 더 씀 (큐에 다수 노드) but 최적해 조기 발견.
  • DP(Algo 21)는 O(nW)로 작은 W에 빠름. B&B는 W가 크고 n이 작을 때 유리.

06 — LP Relaxation — 상한의 원천

전공자

정수 제약 풀어주기

정수계획(IP)의 제약 x ∈ {0, 1}을 0 ≤ x ≤ 1로 완화 → 선형계획(LP). LP는 simplex, interior-point 등 다항 시간에 풀림.

IP: max cᵀx    s.t. Ax ≤ b,   x ∈ {0, 1}ⁿ

LP relaxation: max cᵀx    s.t. Ax ≤ b,   0 ≤ x ≤ 1

LP 최적값 ≥ IP 최적값 (제약 완화 = 더 많은 해).
  • 0/1 배낭: LP relaxation = 분수 배낭. v/w 큰 것부터 채우기 O(n log n).
  • TSP: LP relaxation = "각 도시 방문 1회, 각 간선 0-1" 완화. 복잡한 LP지만 simplex로 해결.
  • Integrality Gap: LP 최적값 / IP 최적값. 1에 가까우면 좋은 bound, 크면 약함.

현대 MIP solver의 실제 동작: 각 노드에서 LP 풀기 → fractional 변수 하나를 골라 branching → LP 반복. 이 "LP 기반 B&B"가 1960년 Land-Doig의 핵심 아이디어.

전공자

Branch and Cut

B&B + Cutting Planes (Gomory 1958). LP relaxation의 fractional 해 주변에 새로운 선형 제약을 추가해 LP를 더 tight하게. 가지치기 효과 극대화.

  • Gomory cuts, Chvátal-Gomory cuts, lift-and-project 등.
  • 현대 CPLEX/Gurobi의 중심 기술. "Pure B&B"보다 수백 배 빠른 경우 흔함.
  • B&B + Cut + Heuristics = Branch-and-Cut-and-Price.

07 — TSP Branch & Bound (Little 1963)

전공자

행/열 축약(Reduction)

TSP 거리 행렬 C에서 하한을 싸게 계산하는 Little 1963의 방법:

  1. 각 행의 최솟값을 빼기 (행 축약). 합계를 하한에 더함.
  2. 각 열의 최솟값을 빼기 (열 축약). 합계를 하한에 더함.
  3. 결과 행렬은 각 행·열에 최소 0이 하나씩. 이 "축약 행렬"에서 최소 투어 ≥ 0.
  4. 따라서 원 최소 투어 ≥ 축약 합계(row_sum + col_sum).

이것이 현재 subtour 비용 + reduction 값 = 하한. 분기 시:

  • 간선 (i, j) 포함: 그 행·열의 다른 원소 무한대로. 서브투어 방지.
  • 간선 (i, j) 제외: c[i][j] = ∞로.
  • 각 선택마다 재축약 후 새 하한 계산.
전공자

Held-Karp 하한 (1970)

더 tight한 TSP 하한: 1-tree(노드 1을 제외하고 나머지 MST + 노드 1의 두 가장 가까운 간선). TSP 투어의 어떤 해도 1-tree보다 무거움 (tour − edge = spanning path ≤ 1-tree).

Lagrangian relaxation으로 더욱 개선 — Held-Karp 알고리즘. Grötschel 1980이 120-도시, Padberg-Rinaldi 1987이 2,392-도시까지 증명된 최적해 달성.

08 — TSP B&B Python 구현 (간단 버전)

전공자

DFS + 하한 가지치기

여기서 쓰는 하한은 Little 1963의 행/열 축약보다 단순한 버전 — "남은 각 미방문 도시에서 나가는 최소 간선"의 합. 구현이 짧고 교육용으로 충분. 실무 TSP는 §07의 행/열 축약 또는 Held-Karp 1-tree가 훨씬 tight.

tsp_bb.py
import math

def tsp_bb(dist):
    """n×n 거리 행렬. 시작 도시 0. 반환: (최소 투어 길이, 순서)."""
    n = len(dist)
    INF = float('inf')
    best = [INF]
    best_path = [None]

    def lower_bound(path, cur_cost):
        """남은 미방문 도시에서 나가는 최소 간선 합 + 현재 비용."""
        visited = set(path)
        bound = cur_cost
        for i in range(n):
            if i not in visited:
                # i에서 나갈 수 있는 최소 간선 (미방문 또는 시작점)
                m = min(dist[i][j] for j in range(n)
                        if j != i and (j not in visited or j == 0))
                bound += m
        return bound

    def dfs(path, cur_cost):
        if len(path) == n:
            total = cur_cost + dist[path[-1]][0]
            if total < best[0]:
                best[0] = total
                best_path[0] = path + [0]
            return

        if lower_bound(path, cur_cost) >= best[0]:
            return    # 가지치기!

        last = path[-1]
        for nxt in range(1, n):
            if nxt not in path:
                dfs(path + [nxt], cur_cost + dist[last][nxt])

    dfs([0], 0)
    return best[0], best_path[0]


# 테스트 (4 도시 대칭 TSP)
dist = [
    [0, 10, 15, 20],
    [10, 0, 35, 25],
    [15, 35, 0, 30],
    [20, 25, 30, 0],
]
print(tsp_bb(dist))    # (80, [0, 1, 3, 2, 0])
  • 복잡도: 최악 O(n!), 실제는 하한이 얼마나 tight한가에 달림.
  • 4-도시 → 즉시, 10-도시 → 수 초, 20-도시 → 수 시간.
  • 실무 TSP (대륙 규모 도시 수천~수만)은 Concorde solver(Applegate et al. 2006)가 정답 — B&B + Held-Karp + 휴리스틱 + cutting planes 결합.

09 — 탐색 전략 — DFS vs Best-First

고급

DFS vs BFS vs Best-First

전략자료구조장점단점
DFS스택(재귀)메모리 O(깊이) 적음best가 늦게 발견되면 가지치기 약함
BFS큐얕은 해 먼저메모리 폭발, 최적화엔 비효율
Best-First우선순위 큐가장 promising 노드 우선 → best 조기 발견힙 연산 오버헤드, 메모리 많음

실무 선택:

  • 메모리 제약 있으면 DFS (좌측 아이템부터 = depth-first). 초반에 좋은 best를 찾는 휴리스틱으로 보완.
  • 가장 빠른 최적 발견 → Best-First. CPLEX 기본.
  • 현대 solver는 혼합: Best-First로 노드 선택, DFS로 조기 실행 가능 해 탐색. "Dive" 전략.
같은 트리, 다른 순서: DFS는 좌측부터 깊게, Best-First는 "약속된 가치가 가장 큰" 노드부터. Best-First는 최적을 먼저 찾지만 메모리 사용량이 크다.

10 — 우선순위 큐 기반 B&B

전공자

구현 패턴

bb_template.py
import heapq

def branch_and_bound(root_state):
    best = float('-inf')    # 최대화 기준
    best_sol = None

    # heap: (-UB, counter, state) — counter는 tie-break
    heap = []
    counter = 0
    heapq.heappush(heap, (-upper_bound(root_state), counter, root_state))

    while heap:
        neg_ub, _, state = heapq.heappop(heap)
        ub = -neg_ub

        if ub <= best:
            break    # 최적성 증명 — 남은 모두 더 나쁨

        if is_complete(state):
            val = objective(state)
            if val > best:
                best, best_sol = val, state
            continue

        for child in branch(state):
            child_ub = upper_bound(child)
            if child_ub > best:
                counter += 1
                heapq.heappush(heap, (-child_ub, counter, child))

    return best, best_sol
  • Counter: heap 동률 시 tie-break. 없으면 state 비교 에러 가능.
  • 조기 종료: 큐 최상단의 UB ≤ best면 즉시 종료 — B&B의 가장 강력한 가지치기.
  • 실제 MIP solver는 추가로: warm start, restart, diving, feasibility pump 등.

전공자

현대 MIP solver의 구성 요소

  1. Presolve: 문제 단순화 (변수 제거, 제약 tightening). 50% 이상 성능 향상.
  2. LP relaxation: Simplex 또는 Interior-Point.
  3. Branching rules: "어떤 변수로 분기할까?" — strong branching, pseudocost 등.
  4. Cutting planes: Gomory, MIR, Clique, Cover cuts.
  5. Heuristics: feasibility pump, RINS, local branching — 좋은 primal bound 빨리 발견.
  6. Node selection: Best-first, DFS, hybrid.
  7. Parallelism: 최근 GPU·분산 버전도 등장.

1988 CPLEX vs 2024 Gurobi: 알고리즘만으로 ~100배 속도 향상 (하드웨어 발전 배제). 최적화는 "딥러닝 못지않은" 지난 35년의 혁명.

12 — 응용 — 스케줄링, 포트폴리오, 라우팅

고급

항공 승무원 스케줄링

Delta, United 같은 항공사가 매일 수만 명의 승무원 근무를 할당. 변수: (승무원 × 비행편) = 수백만. 제약: 노동법, 숙소, 자격. 목적함수: 총 비용 최소화. MIP로 모델링 → B&B solver. 연간 수억 달러 절감.

고급

포트폴리오 최적화

"어떤 자산에 얼마를 투자?" 기본은 Markowitz의 Quadratic Programming(QP). 하지만 실무엔 정수 제약:

  • 한 번에 살 수 있는 단위(lot size).
  • 보유할 자산 수 상한(cardinality).
  • 최소 보유량(threshold).

→ Mixed-Integer Quadratic Programming (MIQP). B&B + QP subproblem. BlackRock, Goldman Sachs의 실무 표준.

고급

공급망 & 제조

  • 로지스틱스: UPS, FedEx 차량 라우팅. VRP(Vehicle Routing Problem), TSP 변형.
  • 제조: 생산 라인 스케줄링, 설비 배치, 원자재 cutting.
  • 에너지: 발전소 가동 계획 (unit commitment), 송전망 최적화.
  • 반도체: 칩 설계에서 게이트 배치(VLSI placement) — 정수계획 + 휴리스틱.
전공자

AI가 B&B를 잘못 쓰는 패턴

  • Bound 계산 오버헤드: "복잡한 LP를 매 노드마다" → 가지치기 이득 < bound 계산 비용. 적절한 근사 필요.
  • DP로 풀 걸 B&B로: 정수 배낭 DP는 O(nW). W 작으면 DP가 100배 빠를 수 있음.
  • 휴리스틱 없이 순수 B&B: 초반 best가 없으면 가지치기 전혀 안 됨 → 전수 탐색. 그리디로 초기 best 얻기.
  • Heap counter 누락: (priority, state)만 넣으면 state 비교 TypeError. (priority, counter, state)가 표준.

교정 질문: "bound 계산이 O(1)~O(log n)인가?", "초기 best가 있나?", "DP 대안은?"

14 — 연습문제 (12문제)

WHY 문제 1

LP relaxation이 항상 IP의 상한(최대화)인 이유.

풀이 펼치기
답

IP 실현 가능 영역 ⊆ LP 실현 가능 영역 (정수 제약은 0 ≤ x ≤ 1의 특수 경우). 더 작은 영역에서의 최대 ≤ 더 큰 영역에서의 최대. 따라서 max(LP) ≥ max(IP). LP 최적값이 상한.

PREDICT 문제 2

0/1 배낭 DP O(nW) vs B&B, 언제 어느 게 빠를까?

풀이 펼치기
답

DP: 시간 O(nW), 공간 O(W). W가 작으면 매우 빠름 (100만 아이템 × 1000 용량 = 10억, 수 초). B&B: 최악 O(2ⁿ), 실제는 bound 성능에 따라. n이 작고 W가 크면 B&B가 유리 (W=10⁹이면 DP 불가). n이 크고 W가 작으면 DP가 유리.

BUILD 문제 3

TSP 4 도시 대칭 거리 행렬이 주어질 때 B&B 하한을 계산해봐라.

풀이 펼치기
답

간단한 하한: "각 도시에서 나가는 최소 간선 + 들어오는 최소 간선의 절반 합". Little 1963의 행/열 축약은 더 tight — 각 행 최솟값 뺀 뒤 열 최솟값 뺀 합. Held-Karp 1-tree는 더 강력. Bound tight일수록 가지치기 강력.

DESUGAR 문제 4

Best-First B&B가 "현재 큐 top의 UB ≤ best면 종료"할 수 있는 이유.

풀이 펼치기
답

Heap이 UB 최대 순으로 정렬. Top의 UB가 best 이하면 나머지 모든 노드 UB ≤ top UB ≤ best. 어느 노드도 best 초과 해 못 만듦 → 탐색 중단. DFS는 이 성질 없음 (top이 현재 브랜치의 UB일 뿐).

WHY 문제 5

heapq에 (priority, state)만 push하면 TypeError가 나는 이유.

풀이 펼치기
답

heap이 priority 동률일 때 두 번째 원소(state)를 비교. state가 dict, set, 사용자 class면 < 비교 불가 → TypeError. 해결: (priority, counter, state) 형식. counter는 고유 순번이라 tie-break 역할.

BUILD 문제 6

0/1 배낭에서 "초기 best를 그리디로" 얻는 효과.

풀이 펼치기
답

순수 B&B는 초기 best = 0 → 대부분 가지가 bound > 0 → 많이 탐색. 그리디 v/w 순으로 초기 best 얻으면 best = 160 정도 (예제). 이후 많은 가지 bound < 160 → 조기 가지치기 → 수~수백 배 빠름. 현대 MIP solver의 "primal heuristics"가 같은 아이디어.

PREDICT 문제 7

20 도시 TSP를 B&B로 풀면 예상 시간?

풀이 펼치기
답

naive O((n-1)!) = 19! ≈ 10¹⁷ — 불가능. 좋은 bound + 가지치기로 현대 컴퓨터에서 수 초~수 분. Concorde solver (Applegate-Bixby-Chvátal-Cook, ~1999-)는 2006년 pla85900 (85,900-노드 VLSI 설계 인스턴스)를 증명된 최적해로 풀었음. 주요 기법: LP relaxation + cutting planes + Held-Karp. (USA-13509는 1998년에 해결된 다른 인스턴스.)

WHY 문제 8

Integrality gap이 큰 문제가 B&B에 어려운 이유.

풀이 펼치기
답

Gap = LP 최적값 / IP 최적값. gap이 크면 LP bound가 실제와 멀어 가지치기 약함 → 트리 크기 커짐. 예: Set Cover는 integrality gap이 log(n) 가능 → 큰 인스턴스에 어려움. Cutting planes가 이런 경우 LP bound를 tighter하게 만들어 도움.

BUILD 문제 9

"프로젝트 선택" 문제(각 선택 수익/비용, 의존성)를 B&B로.

풀이 펼치기
답

이진 변수 x_i ∈ {0,1}. 의존성 x_i ≤ x_j (i 선택하려면 j 선택). 최대 ∑ profit_i · x_i. B&B: 변수 하나 골라 0/1 분기. LP relaxation으로 bound. Algo 18 §12에서 본 Picard 1976 방법 (min-cut)과 대안.

DESUGAR 문제 10

B&B에서 "strong branching"이 뭐고 왜 유용?

풀이 펼치기
답

현재 노드에서 각 후보 변수로 분기 후 LP 반복해 어느 쪽이 bound를 많이 개선하는지 실측. 가장 좋은 변수로 실제 분기. 비용 비싸지만 트리 크기 대폭 감소. Pseudocost는 과거 strong branching 결과를 저장해 추정 — 현대 solver 표준.

WHY 문제 12

Ailsa Land가 경제학자도 수학자도 "그 분야"가 아닌데 1960 논문이 중요한 이유.

풀이 펼치기
