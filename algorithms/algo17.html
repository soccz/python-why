ALGO 17

최소 신장 트리 (MST)
1926년 체코 전력망에서 태어난 그리디

1926년, 체코 모라비아 지방 전력망을 최저 비용으로 설계해달라는 요청을 받은 수학자 Otakar Borůvka는 최초의 MST 알고리즘을 발표합니다. 4년 뒤 같은 체코의 Jarník이 정점 중심 버전을, 1956년 Bell Labs의 Kruskal이 간선 정렬 버전을, 1957년 같은 Bell Labs의 Prim이 Jarník의 방법을 재발견.
네 명이 독립적으로 같은 그리디 원리에 도달. 그 원리는 지금도 네트워크 설계, 클러스터링, 이미지 segmentation의 기반.
핵심 질문: "왜 '가장 싼 간선만 욕심내는' 전략이 전역 최적을 보장하는가?"

01 — 이번 주 이야기: 네 수학자의 동시 발견

"In 1926, I was asked by a friend working for the West Moravian Electric Power Company to solve a problem: how to connect a given set of points in a region by a network of least total length?"
— Otakar Borůvka, 회고 (1977)

1926 — Otakar Borůvka, 체코 브르노

모라비아 지역 전력회사가 "작은 마을 N개를 최저 전선 길이로 잇는 방법"을 의뢰. Borůvka의 답은 "O jistém problému minimálním"(Práce Moravské Přírodovědecké Společnosti 3:37-58, 1926). 체코어 논문이라 세계에 알려지지 않음.

Borůvka의 아이디어: 모든 정점이 동시에 자기 최소 간선을 선택하고 merge. 병렬(parallel) 관점 — 현대 병렬 MST의 원조.

1930 — Vojtěch Jarník, 체코 프라하

Borůvka의 지도학생이 아니었지만 같은 체코에서 정점 중심 접근을 독립 발견. "한 정점에서 시작, 가장 싼 이웃 간선으로 확장, 반복". 하지만 영어권에 전달되지 못함.

1956 — Joseph Kruskal, Bell Labs

AT&T Bell Labs 소속. 네트워크 설계 맥락에서 간선 중심 접근 제안: "간선을 비용 오름차순으로 정렬, 사이클 만들지 않으면 추가". 논문 "On the Shortest Spanning Subtree of a Graph and the Traveling Salesman Problem"(Proc. AMS 7:48-50, 1956). 수학자 Harold W. Kuhn(TSP 연구자)이 제시한 문제에서 출발 — Kruskal은 Borůvka 1926을 원 논문 작성 시점에는 몰랐고, 후대 문헌 정리에서 체코 원조가 드러남.

1957 — Robert C. Prim, Bell Labs

같은 Bell Labs이지만 Kruskal과 독립적으로. Jarník의 정점 기반 알고리즘을 재발견. 논문 "Shortest Connection Networks and Some Generalizations"(Bell Syst. Tech. J. 36:1389-1401, 1957). 미국 교재가 "Prim's algorithm"으로 명명해 Jarník의 이름이 묻힘.

Dijkstra 1959도 Prim과 독립적으로 같은 알고리즘을 세 번째로 발견. 그래서 유럽권에선 Jarník–Prim–Dijkstra로 부르기도.

30년간 4번 독립 발견된 알고리즘. 체코 수학자들(Borůvka, Jarník)이 먼저였지만, 영어권 저널에 늦게 알려져 Kruskal/Prim 이름으로 교재에 남음. 현대 병렬 컴퓨팅은 Borůvka의 원안으로 돌아옴.

02 — MST란 — 문제 정의와 응용

초급

MST = "전봇대 잇기"

마을 N개를 전선으로 이어야 합니다. 전선은 비싸니까 최소 길이로. 단, 모든 마을이 연결되어야 하고 루프(사이클)는 낭비. 답: N-1개의 전선으로 이루어진 "트리".

  • N개 정점 → 정확히 N-1개 간선으로 된 트리.
  • 모든 정점이 서로 도달 가능.
  • 사이클 없음 (있으면 하나 빼도 여전히 연결됨).
  • 총 비용 최소.
중급

형식적 정의

무방향 가중 그래프 G = (V, E, w)에서, 모든 정점을 포함하고 사이클이 없는 부분그래프 T = (V, E') 중 ∑e∈E' w(e)가 최소인 것.

  • T는 신장 트리(spanning tree). 정점이 V 전부, 간선 수 |V|−1.
  • Minimum → 총 가중치 최소.
  • 해는 유일하지 않을 수 있음 (동일 가중치 간선이 있으면).
MST = 5개 간선(V-1), 사이클 없음, 모든 정점 연결, 총 비용 최소. 회색 점선 간선(9, 10)과 파란색 굵은 간선(7, 8)은 제외 — 더 싼 대안이 있기 때문.
고급

응용 분야

  • 네트워크 설계: 전력망, 통신망, 도로망 최저 비용 구축. MST 문제의 원형.
  • 클러스터링: Single-Link Clustering. 가장 싼 간선부터 이어가다 특정 임계값에서 멈추면 자연스런 클러스터.
  • 이미지 segmentation: 픽셀 간 그래프 MST → 유사 영역 탐지.
  • TSP 근사: MST의 2배가 Traveling Salesman의 상한.
  • 데이터 압축: 유전자 거리 행렬에서 계통수(phylogenetic tree) 추정.

03 — Cut Property — 모든 증명의 기반 (CLRS §21.1)

중급

"컷(cut)"이란

컷(S, V\S): 정점 집합 V를 두 부분 S와 V\S로 가르는 분할. 한쪽 끝이 S에, 다른 끝이 V\S에 있는 간선을 "컷을 건너는 간선"(crossing edge)이라 부름.

Cut Property (안전 간선 정리): 임의의 컷을 하나 잡았을 때, 그 컷을 건너는 간선 중 가장 가벼운(최소 가중치) 간선은 반드시 어떤 MST에 포함된다. 이 간선을 "safe edge"라 부름.

4프레임 Cut Property: 컷을 임의로 긋고 → 건너는 간선을 찾고 → 그중 최소 = 반드시 MST에 포함됨. 이 하나의 정리가 Kruskal, Prim, Borůvka 모두의 정당성을 증명.
고급

증명 (교환 논증, exchange argument)

컷 (S, V\S)를 건너는 최소 간선을 e라 하자. 어떤 MST T가 e를 포함하지 않는다고 가정.

  1. T는 모든 정점을 연결하므로, T 안에도 컷을 건너는 간선 e'가 반드시 존재 (그렇지 않으면 S와 V\S가 분리됨 → T가 연결 트리 아님).
  2. T에 e를 추가하면 사이클 C가 생김. C는 컷을 짝수번 건너야 하므로 C에 또 다른 crossing edge가 있음 — 바로 e'.
  3. T에서 e'를 빼고 e를 넣은 T' = T − e' + e도 여전히 spanning tree.
  4. w(e) ≤ w(e') (e가 건너는 간선 중 최소)이므로 w(T') ≤ w(T).
  5. T가 MST였으니 등호 성립 → T'도 MST이고 e 포함. □

이 논증은 exchange argument의 교과서적 예. 그리디 알고리즘 정당성 증명의 표준 기법.

📖 원서 인용 — CLRS 4판, Theorem 21.1

"Let G = (V, E) be a connected, undirected graph with a real-valued weight function w defined on E. Let A be a subset of E that is included in some minimum spanning tree for G, let (S, V−S) be any cut of G that respects A, and let (u,v) be a light edge crossing (S, V−S). Then, edge (u,v) is safe for A."

04 — Kruskal — 간선 정렬 + Union-Find

중급

아이디어 — "가장 싼 것부터, 사이클 피해서"

  1. 모든 간선을 가중치 오름차순으로 정렬.
  2. 순서대로 간선을 하나씩 보면서:
    • 두 끝점이 다른 컴포넌트에 있으면 → MST에 추가하고 두 컴포넌트 merge.
    • 같은 컴포넌트면 → 사이클 유발 → 건너뛰기.
  3. V-1개 간선을 골랐으면 완료.

왜 Cut Property와 연결되나: 현재까지 선택한 간선들이 이룬 컴포넌트 하나를 S, 나머지를 V\S로 보면, 정렬된 간선 중 가장 먼저 "S를 V\S와 잇는" 것이 건너는 최소 간선 = safe edge.

6단계 Kruskal 진행: 간선을 가중치 오름차순으로 처리 → 다른 컴포넌트면 merge, 같으면 skip. 4번째 간선 선택 후 V-1=4개 완료. Union-Find로 컴포넌트 merge를 빠르게.

05 — Kruskal Python 구현 (Union-Find)

고급

Union-Find (DSU) 자료구조

Kruskal의 핵심은 "두 정점이 같은 컴포넌트인가"를 빠르게 답하기. 이를 위한 자료구조가 Disjoint Set Union (DSU), 별칭 Union-Find.

  • find(x): x의 컴포넌트 대표(루트) 반환.
  • union(x, y): x와 y가 속한 두 컴포넌트 합침.
  • 두 최적화:
    • Path Compression: find 중 만난 노드를 전부 루트로 직접 연결 (다음 find를 O(1)에).
    • Union by Rank: 작은 트리를 큰 트리 밑에 붙이기 (트리 높이 제한).
  • 둘 다 쓰면 거의 O(α(N)) (Ackermann 역함수 — 실질 상수).
전공자

Python 구현

kruskal.py
class DSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n

    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]    # path halving (compression의 변형, 동일 O(α))
            x = self.parent[x]
        return x

    def union(self, x, y):
        rx, ry = self.find(x), self.find(y)
        if rx == ry: return False    # 이미 같은 컴포넌트
        if self.rank[rx] < self.rank[ry]:
            rx, ry = ry, rx
        self.parent[ry] = rx
        if self.rank[rx] == self.rank[ry]:
            self.rank[rx] += 1
        return True


def kruskal(V, edges):
    """edges: [(u, v, w), ...]. 반환: (MST 간선 리스트, 총 가중치)."""
    dsu = DSU(V)
    edges_sorted = sorted(edges, key=lambda e: e[2])
    mst = []
    total = 0

    for u, v, w in edges_sorted:
        if dsu.union(u, v):
            mst.append((u, v, w))
            total += w
            if len(mst) == V - 1:
                break    # 조기 종료

    return mst, total


# 테스트
edges = [(0,1,7),(0,2,3),(0,3,6),
         (1,2,4),(1,4,2),
         (2,3,1),(2,4,5)]
mst, total = kruskal(5, edges)
print(total, mst)
# 10 [(2,3,1),(1,4,2),(0,2,3),(1,2,4)]
  • 복잡도: 정렬 O(E log E) + V번의 Union-Find ≈ O(E log E). 정렬이 병목.
  • 간선 리스트가 입력으로 자연스러움. 인접 리스트 안 필요.
  • 조기 종료: V-1개 추가되면 그만.

06 — Prim — 정점에서 트리 확장

중급

아이디어 — "씨앗에서 싹트는 트리"

  1. 임의의 정점 r에서 시작. 트리 T = {r}.
  2. T에 포함된 정점 중 하나에서 T 밖 정점으로 가는 간선 중 가장 싼 것을 찾아 T에 추가.
  3. V-1번 반복.

Cut Property와의 연결: 현재 T를 S, 나머지를 V\S로 보면, 컷을 건너는 간선 중 최소 = 바로 safe edge. 이게 Prim의 그리디 선택.

Dijkstra와의 놀라운 유사성: 구현이 거의 동일 — heap, 완화, extract-min. 차이는 오직 하나:

  • Dijkstra: dist[v] = "시작점 s에서 v까지의 누적 거리".
  • Prim: key[v] = "T 밖 정점 v를 T에 잇는 단일 간선 최소 비용".
5단계 Prim 진행: A에서 시작 → 매 단계 T에서 밖으로 가는 가장 싼 간선 선택. Kruskal과 선택 순서는 다르지만 최종 합은 동일 (10). 둘 다 MST이므로.

07 — Prim Python 구현 (heapq)

고급

Dijkstra와 구현 쌍둥이

Prim은 Dijkstra와 거의 같은 코드. 차이는 heap 우선순위로 "누적 거리" 대신 "단일 간선 비용"을 쓰는 것.

prim.py
import heapq

def prim(V, graph, start=0):
    """graph: {u: [(v, w), ...]} 인접 리스트."""
    in_tree = [False] * V
    parent = [None] * V
    key = [float('inf')] * V
    key[start] = 0

    pq = [(0, start)]    # (간선 비용, 정점)
    mst_edges = []
    total = 0

    while pq:
        k, u = heapq.heappop(pq)
        if in_tree[u]: continue    # stale entry skip
        in_tree[u] = True
        total += k
        if parent[u] is not None:
            mst_edges.append((parent[u], u, k))

        for v, w in graph.get(u, []):
            if not in_tree[v] and w < key[v]:
                key[v] = w
                parent[v] = u
                heapq.heappush(pq, (w, v))

    return mst_edges, total


# 테스트
graph = {
    0: [(1,7),(2,3),(3,6)],
    1: [(0,7),(2,4),(4,2)],
    2: [(0,3),(1,4),(3,1),(4,5)],
    3: [(0,6),(2,1)],
    4: [(1,2),(2,5)],
}
mst, total = prim(5, graph, start=0)
print(total, mst)
# 10 [(0,2,3),(2,3,1),(2,1,4),(1,4,2)]
  • lazy deletion 패턴 (Algo 15 §05와 동일): 같은 정점의 옛 entry는 pop 시 in_tree 체크로 skip.
  • 복잡도: binary heap으로 O((V+E) log V). Fibonacci heap이면 O(E + V log V).
  • Kruskal vs Prim:
    • Kruskal: 간선 리스트에 자연스러움. 병렬화 어려움.
    • Prim: 인접 리스트에 자연스러움. dense 그래프에 유리.
📖 원서 인용 — CLRS 4판, §21.2 "The algorithms of Kruskal and Prim"

"Both algorithms can easily be made to run in time O(E lg V) using ordinary binary heaps. By using Fibonacci heaps, Prim's algorithm runs in time O(E + V lg V), which is an improvement over the binary-heap implementation if |V| is much smaller than |E|."

08 — 정확성 증명: Cut Property의 위력

고급

Kruskal 정당성

불변식: 현재까지 선택한 간선 집합 A는 어떤 MST의 부분집합.

  • Base: A = ∅은 빈 집합 → 임의 MST의 부분집합. ✓
  • Step: 다음에 추가할 최소 가중치 간선 e = (u,v)가 다른 컴포넌트 연결이라면:
    • 컷 정의: S = u가 속한 컴포넌트, V\S = 나머지. 이 컷은 A를 존중(respect) — A의 어떤 간선도 이 컷을 건너지 않음 (모두 A의 컴포넌트 내부).
    • e는 이 컷을 건너는 최소 가중치 간선 (정렬 순서). Cut Property → e는 safe.
    • A ∪ {e}도 어떤 MST의 부분집합. ✓
  • V-1 번 반복 후 A = MST. □
고급

Prim 정당성

불변식: 현재 트리 T는 어떤 MST의 부분트리.

  • 컷: S = T의 정점 집합, V\S = 나머지.
  • Prim은 이 컷을 건너는 최소 간선을 추가 → Cut Property.
  • Kruskal과 완전히 같은 논리. □

핵심 메시지: Kruskal, Prim, Borůvka는 모두 Cut Property의 서로 다른 실행 방식. 차이는 어느 컷을 고르는가:

  • Kruskal: 간선 순서에 따라 "암묵적으로" 컷이 결정됨.
  • Prim: (현재 트리, 나머지)로 컷이 명시적.
  • Borůvka: 각 컴포넌트를 S로 보고 병렬로 safe edge 선택.

09 — Borůvka — 병렬 MST의 원조

전공자

1926년의 병렬 아이디어

Borůvka는 "각 정점(컴포넌트)이 동시에 자기 최소 간선을 고르고 merge"라는 아이디어. 현대 병렬/분산 컴퓨팅에 딱 맞음.

  1. 각 정점을 별도 컴포넌트로 시작.
  2. 모든 컴포넌트가 병렬로 자기 밖으로 나가는 최소 간선 고름.
  3. 선택된 간선들을 모두 MST에 추가, 해당 컴포넌트들 merge.
  4. 컴포넌트가 1개가 될 때까지 반복.
  • 한 phase마다 컴포넌트 수가 최소 절반 → O(log V) phase.
  • 각 phase 내 간선 검사: O(E). 총 O(E log V).
  • 1976년 Cheriton & Tarjan (SIAM J. Comput. 5:724-742)이 Prim/Borůvka 혼합으로 O(E log log V) 달성.
Borůvka = 병렬 MST: 각 컴포넌트가 독립적으로 자기 light edge 선택 → 동시에 merge. 한 phase마다 컴포넌트 수 ≥ 절반 감소. 분산·GPU 환경에 자연스러움.
전공자

왜 한 phase에 컴포넌트 수가 절반 이하가 되나

각 컴포넌트는 자기 light edge를 "투표"로 고름. 모든 컴포넌트가 최소 1개의 외부 간선을 선택 → 병합 그래프에서 각 컴포넌트는 최소 1개의 이웃과 연결. 따라서 새 컴포넌트 수 ≤ 원래 컴포넌트 수 / 2.

→ O(log V) phase로 수렴. 병렬로 phase를 돌리면 이론적으로 O(log V × E/P) 시간 (P는 프로세서 수).

10 — 복잡도 비교

고급

세 알고리즘 복잡도 표

알고리즘 시간 복잡도 자료구조 언제 유리
Kruskal O(E log E) = O(E log V) 정렬 + Union-Find sparse, 간선 리스트 입력
Prim (binary heap) O((V+E) log V) heapq + 인접 리스트 dense, 인접 리스트 입력
Prim (Fibonacci) O(E + V log V) Fibonacci heap 이론상 최고, 실측 느림
Borůvka O(E log V) Union-Find 병렬 / 분산
Borůvka-Prim 혼합 O(E log log V) MST-REDUCE sparse, 경진 대회
Chazelle 2000 O(E · α(V, E)) soft heap 이론적 하한 근접

참고: MST의 이론적 하한은 여전히 미제(open problem). Chazelle의 O(Eα)는 deterministic 최고, Karger-Klein-Tarjan 1995은 randomized O(E) 달성.

전공자

실무에서의 선택

  • 경진 대회: 대부분 Kruskal + DSU. 코드 짧고 디버깅 쉬움.
  • dense 그래프 (E ≈ V²): Prim이 유리. Kruskal은 정렬만 O(V² log V).
  • 분산 환경 (맵리듀스): Borůvka. 각 worker가 자기 컴포넌트 독립 처리.
  • GPU / 병렬: Borůvka의 현대 변종 (Bader & Cong 2006).
  • Python 라이브러리: networkx.minimum_spanning_tree 기본은 Kruskal, scipy.sparse.csgraph.minimum_spanning_tree는 sparse matrix 위의 Prim 계열 변형 (scipy 소스 _min_spanning_tree.pyx).

11 — 변종 — 2nd-best MST, Bottleneck

전공자

Second-Best MST

"MST 다음으로 싼 스패닝 트리". MST와 정확히 하나의 간선만 다름.

알고리즘: MST 구한 뒤, MST에 없는 각 간선 e = (u,v)를 삽입했을 때 만들어지는 사이클에서 가장 무거운 간선을 뺌. 후보들 중 최소 교체 비용이 답. O(V² + E) with DFS.

전공자

Bottleneck Spanning Tree

최대 간선 가중치를 최소화하는 신장 트리. 일반 MST와 다른 목적 함수지만, 모든 MST는 bottleneck spanning tree다 (역은 성립 안 함).

증명 아이디어: 임의의 신장 트리 T를 잡고 그 최대 간선을 emax(T)라 하자. MST를 T*, 그 최대 간선을 e*라 할 때 e* ≤ emax(T)를 보이면 충분. T ≠ T*이면 T에 있고 T*에 없는 간선 f를 하나 잡고, f를 T*에 넣어 생긴 사이클에서 T*의 간선 g를 뺀 T*' = T* − g + f. T*가 MST였으므로 w(f) ≥ w(g), 따라서 e* ≤ max(w(f), e*) ≤ emax(T). 즉 모든 신장 트리의 최대 간선은 MST의 최대 간선 이상. □

전공자

Steiner Tree (NP-hard)

"부분 정점만 연결하되 중간 정점(Steiner point)을 추가로 쓸 수 있는" MST. MST와 달리 NP-hard. 2-approximation은 MST로 얻어짐.

12 — 실무 — 네트워크 설계, 클러스터링

중급

전력망 / 통신망 설계

원래 Borůvka 1926의 문제 그대로. 모라비아 전력망은 실제로 Borůvka 알고리즘으로 설계됨. 현대에도 해저 케이블, 5G 백홀 네트워크 등에서 MST 변형이 사용.

고급

Single-Link Clustering

데이터 포인트 사이 거리를 그래프로 모델링. Kruskal을 돌리되 특정 임계값보다 큰 간선은 추가하지 않으면 자연스런 클러스터가 형성됨.

  • scipy.cluster.hierarchy.linkage의 method='single'이 이 방법.
  • Chameleon, CURE 등 계층적 클러스터링 알고리즘의 기본 빌딩 블록.
  • 한계: "체인 효과"(chaining) — 긴 가느다란 클러스터에 약함. Ward, complete-link가 보완.
고급

이미지 Segmentation

픽셀을 정점, 이웃 픽셀 사이 색 차이를 간선 가중치로. Felzenszwalb-Huttenlocher 2004 알고리즘이 MST 기반. OpenCV의 cv2.ximgproc.createGraphSegmentation 내부.

  • Kruskal 돌리면서 컴포넌트 내부 최대 간선(internal difference) 추적.
  • 두 컴포넌트를 이을 간선이 internal difference보다 크면 merge 안 함 → 자연스런 영역 분리.

13 — AI와 MST

중급

MST는 어디 쓰이나

  • k-means 초기화: MST 기반 분할로 초기 centroid 선정 (k-means++의 변형).
  • 계통수 추론: 유전자 거리 → MST → phylogenetic tree 근사.
  • 그래프 기반 반지도 학습: MST로 라벨 전파의 backbone 구성.
  • 이상 탐지: MST 평균 간선 길이 대비 이상한 간선은 outlier.
  • 임베딩 시각화: t-SNE, UMAP 결과의 백본을 MST로 그리기.
전공자

AI가 MST를 잘못 쓰는 패턴

  • DAG에 MST: MST는 무방향 그래프에 정의. 방향 그래프는 Edmonds 알고리즘(arborescence)이 별개.
  • 음수 가중치 + MST: 문제 없음 (음수 사이클이 MST에 영향 없음) — 하지만 AI가 혼동해 BF를 쓰라고 할 때 주의.
  • Union-Find 생략: "사이클 검사는 DFS로"라고 제안 → O(VE) 폭발.
  • 정렬 O(n²) 버전 Kruskal: 초보가 bubble sort 쓰는 경우. 반드시 O(E log E) 정렬 쓰라고 강조.

교정 질문: "MST 정의에 방향성 있나?", "Union-Find 없이 사이클 체크는 얼마나 느려지나?"

14 — 연습문제 (12문제)

WHY 문제 1

왜 MST는 항상 정확히 V-1개 간선을 갖나?

풀이 펼치기
답

트리의 정의: "V개 정점을 사이클 없이 연결". 유도: V개 정점을 연결하려면 최소 V-1개 간선 필요 (안 그러면 분리됨). V-1개 초과하면 반드시 사이클 (단순한 귀납 증명). 따라서 정확히 V-1.

PREDICT 문제 2

간선 가중치가 모두 다르다면 MST는 유일한가?

풀이 펼치기
답

유일하다. 증명: MST가 둘(T₁, T₂)이라고 가정. T₁에만 있는 최소 간선 e를 잡으면, T₂에 e를 넣으면 사이클 C. C 중 T₂에만 있는 간선 f를 빼고 e를 넣으면 T₂'는 더 가벼움 → T₂가 MST가 아니라는 모순 (가중치 유일하므로 w(e) ≠ w(f)).

BUILD 문제 3

Union-Find에서 path compression 없이 union by rank만 쓰면 find의 최악 시간은?

풀이 펼치기
답

O(log N). Union by rank만으로도 트리 높이가 log N으로 제한됨 (작은 것을 큰 것에 붙이므로). Path compression을 추가하면 amortized O(α(N)) (실질 상수).

DESUGAR 문제 4

Prim과 Dijkstra의 코드는 왜 이렇게 비슷한가?

풀이 펼치기
답

둘 다 "heap에서 현재 key 최소 정점 꺼내기 + 이웃 relax"의 구조. 차이는 key의 의미: Dijkstra는 dist[v] = s에서 v까지 누적 거리, Prim은 key[v] = v를 T에 잇는 단일 간선 최소 비용. relax 식도 Dijkstra는 dist[u]+w(u,v), Prim은 그냥 w(u,v). 구조적으로 쌍둥이 — 둘 다 heap 기반 그리디.

WHY 문제 5

MST에 음수 가중치 간선이 있어도 알고리즘 작동하나?

풀이 펼치기
답

문제 없음. MST는 "간선을 V-1개 고르는" 문제라 가중치의 부호와 무관. Cut Property도 부호와 무관. Dijkstra와 달리 MST는 음수 간선에도 안전. (직관: 음수 간선은 거의 반드시 MST에 들어감 — 작을수록 좋으니까.)

BUILD 문제 6

Maximum Spanning Tree (MaxST) 구하는 법.

풀이 펼치기
답

모든 간선 가중치 부호 반전 후 MST → 원복. 또는 Kruskal을 내림차순 정렬. Prim도 max-heap으로 바꾸면 됨. 응용: 신뢰도 네트워크에서 "가장 튼튼한 경로"의 핵심.

PREDICT 문제 7

V=10⁶, E=10⁷ 그래프에 Kruskal과 Prim — 어느 게 빠를까?

풀이 펼치기
답

둘 다 O(E log V) = 10⁷ × 20 = 2×10⁸. 비슷함. 하지만 Kruskal은 정렬 상수가 작아 실측 약간 빠름. 단 메모리: Prim은 인접 리스트 O(V+E), Kruskal은 간선 리스트 O(E)만. 입력 형식이 결정 요인.

WHY 문제 8

Borůvka의 각 phase에서 컴포넌트 수가 최소 절반으로 줄어드는 이유.

풀이 펼치기
답

각 컴포넌트가 ≥ 1개의 외부 간선 선택. 선택된 간선들이 만드는 그래프에서 각 컴포넌트의 degree ≥ 1. 따라서 이 "phase 그래프"의 각 컴포넌트는 최소 2개의 원래 컴포넌트를 합침 → 새 컴포넌트 수 ≤ 원래/2.

BUILD 문제 9

그래프가 연결되어 있지 않으면 Kruskal은 어떻게 동작하나?

풀이 펼치기
답

연결 컴포넌트 개수 k라면 MST 대신 Minimum Spanning Forest가 됨. 간선 V-k개. Kruskal 그대로 돌리면 자연스럽게 이런 forest가 나옴. Prim은 하나의 컴포넌트만 커버 — 시작점이 속한 컴포넌트만.

DESUGAR 문제 10

Cut Property가 "최소 light edge를 여러 개 고르면" 어떻게 되나? (동률 간선)

풀이 펼치기
답

동률이 있으면 서로 다른 MST가 존재할 수 있음. Cut Property는 "어떤 최소 간선 하나는 safe"를 보장 (모두가 아님). 동률을 어떻게 tie-break 하느냐에 따라 다른 MST가 나올 수 있지만 총 합은 같음.

AI 문제 11

AI가 "유향 그래프에 Kruskal 추천"했다. 어떻게 교정?

풀이 펼치기
질문
  1. "MST의 정의가 무향 그래프인지 확인해봐."
  2. "유향 MST = Minimum Arborescence. Edmonds 알고리즘(Chu-Liu 1965)이 별개로 존재."
  3. "Edmonds는 기본 O(VE). Gabow-Galil-Spencer-Tarjan 1986이 Fibonacci heap으로 O(E + V log V)까지 개선."
WHY 문제 12

Single-link clustering이 왜 "체인 효과"에 약한가?

풀이 펼치기
답

Kruskal은 "가장 짧은 간선"만 봄 → 긴 가느다란 클러스터가 한 점으로 연결돼도 merge. 예: 두 뚜렷한 클러스터 사이에 점 하나만 중간에 있어도 전체가 한 클러스터로 합쳐짐. complete-link (최대 거리)나 Ward (분산 증가 최소)가 이 문제에 강함.

네트워크 플로우 — Ford-Fulkerson, Edmonds-Karp

"파이프로 물을 얼마나 흘릴 수 있는가" — 최단 경로와 MST를 넘어선 그래프 세 번째 축.

  • Max-Flow Min-Cut 정리 — 쌍대성의 아름다움.
