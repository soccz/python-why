01 — 이번 주 이야기: Levenshtein의 1965년 논문

"Let us consider the problem of correcting deletions, insertions, and reversals which occur in transmission."
— Vladimir Levenshtein, 1965

1965 — 모스크바, Vladimir Levenshtein

소비에트 수학자 Levenshtein(1935-2017)이 오류 정정 부호 연구 중. 논문 "Binary codes capable of correcting deletions, insertions, and reversals"(Doklady Akademii Nauk SSSR 163:845-848, 1965)에서 "문자열 간 최소 편집 수"를 정의. 이것이 지금 모두가 아는 Levenshtein distance.

Levenshtein 자신은 알고리즘을 제시하진 않음 — 수학적 정의였을 뿐. 이를 DP로 푸는 실용 알고리즘은 9년 뒤에 나옴.

1974 — Robert A. Wagner & Michael J. Fischer

"The String-to-String Correction Problem"(J. ACM 21:168-173, 1974). Levenshtein의 정의를 O(mn) DP로 효율 계산하는 알고리즘 정립. 오늘날 모든 교과서의 편집거리 알고리즘 = Wagner-Fischer.

1975 — Daniel S. Hirschberg

"A linear space algorithm for computing maximal common subsequences"(CACM 18:341-343, 1975). Wagner-Fischer의 O(mn) 공간을 O(n)으로. 분할 정복 + DP 혼합. 큰 유전자 서열 정렬에 필수.

고전 DP 5종이 만든 세상

  • diff / git: LCS 알고리즘이 파일 diff의 핵심.
  • 맞춤법 검사: Levenshtein으로 사전에서 가장 가까운 단어.
  • 유전자 정렬 (BLAST, Needleman-Wunsch 1970): 편집 거리의 생물학 버전.
  • GPT BLEU/ROUGE 점수: LCS 기반 기계번역 평가.
  • 자동완성 / 검색 fuzzy matching: 편집 거리 ≤ k 제약.
  • 배낭 문제: 암호학(knapsack cryptosystem), 자원 배분, 투자 최적화.
DP 고전의 황금기 — 1957 Bellman 기초 → 1960-70년대 편집거리, LIS, 유전자 정렬 폭발. 1986 Myers의 O(ND) diff 알고리즘이 git의 기반.

02 — 0/1 배낭 — 2D DP 정석

고급

문제 재정의

Algo 19 §10의 문제: n개 아이템 (value, weight), 용량 W. 쪼갤 수 없음. 최대 가치는?

점화식: dp[i][w] = 첫 i개 아이템을 고려하고 용량 w 이하에서 최대 가치.

dp[i][w] = max( dp[i-1][w],   dp[i-1][w − weights[i]] + values[i] )
  • 첫 번째 항: 아이템 i를 안 담음.
  • 두 번째 항: 아이템 i를 담음 (무게 감소 + 가치 추가).
  • 조건: w ≥ weights[i]여야 두 번째 항 가능.
  • 기저: dp[0][w] = 0.
4단계 DP 테이블: 아이템을 하나씩 추가하며 각 용량에서의 최적값 갱신. dp[3][50]=220 — 그리디(Algo 19)가 놓친 정답. O(nW) 시간, 2D 배열 O(nW) 공간.

03 — 배낭 공간 최적화 — 역순의 마법

고급

2D → 1D 축소

dp[i][w] = max(dp[i-1][w], dp[i-1][w-weights[i]] + values[i]). 점화식이 이전 행만 참조 → 1D로 충분.

핵심: 1D로 축소 시 w를 역순으로 순회해야 함. 이유:

  • 정순 순회 시 dp[w - weights[i]]가 이번 i에서 이미 갱신된 값 → 아이템 i를 여러 번 쓴 것처럼 됨 (완전 배낭이 됨).
  • 역순 순회면 dp[w - weights[i]]는 i-1 상태 → 0/1 성질 유지.
knapsack_01_1d.py
def knapsack_01(values, weights, W):
    n = len(values)
    dp = [0] * (W + 1)
    for i in range(n):
        # 역순 — 0/1 성질 보존
        for w in range(W, weights[i] - 1, -1):
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
    return dp[W]


def knapsack_unbounded(values, weights, W):
    """완전 배낭 — 같은 아이템 무한 사용."""
    n = len(values)
    dp = [0] * (W + 1)
    for i in range(n):
        for w in range(weights[i], W + 1):    # 정순
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
    return dp[W]


print(knapsack_01([60,100,120], [10,20,30], 50))    # 220
print(knapsack_unbounded([60,100,120], [10,20,30], 50))    # 300 (A 5개)
  • 공간: O(nW) → O(W).
  • 시간: 동일 O(nW).
  • 0/1 vs 완전: 순회 방향 차이 한 줄이 문제를 완전히 바꿈.

04 — LCS — 최장 공통 부분수열 (CLRS §14.4)

중급

문제 정의

두 문자열 X, Y. 둘 모두의 부분수열(subsequence)인 가장 긴 수열.

  • 부분수열: 원래 순서 유지하면서 일부 문자만 고른 것. 연속할 필요 없음.
  • 예: X = "ABCBDAB", Y = "BDCABA". LCS = "BCBA" 또는 "BDAB" (길이 4).
  • 부분수열 ≠ 부분문자열(substring — 연속해야 함).
고급

점화식

dp[i][j] = X[0..i-1]와 Y[0..j-1]의 LCS 길이.

dp[i][j] = 0          (i=0 또는 j=0)
dp[i][j] = dp[i-1][j-1] + 1    (X[i-1] = Y[j-1])
dp[i][j] = max(dp[i-1][j], dp[i][j-1])   (X[i-1] ≠ Y[j-1])
  • 같은 글자면: 그 글자를 LCS에 포함 → 이전 상태 +1.
  • 다른 글자면: "X의 마지막 글자 빼기" 또는 "Y의 마지막 글자 빼기" 중 더 긴 쪽.
  • 답: dp[|X|][|Y|].
  • 복잡도: O(mn) 시간, O(mn) 공간 (Hirschberg로 O(n) 공간).
4단계 LCS 테이블: X="ACBE", Y="ABCE"의 LCS 길이 3. 대각선 이동(같은 글자, 노랑) 또는 수평/수직 최대값(다른 글자, 파랑). 복잡도 O(mn).

05 — LCS Python 구현

고급

2D DP 기본 구현

lcs.py
def lcs_length(X, Y):
    m, n = len(X), len(Y)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if X[i-1] == Y[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])

    return dp[m][n]


print(lcs_length("ABCBDAB", "BDCABA"))    # 4 (BCBA 또는 BDAB)
print(lcs_length("ACBE", "ABCE"))    # 3


# 공간 최적화: 2행만 유지 (O(n) 공간)
def lcs_space(X, Y):
    m, n = len(X), len(Y)
    prev = [0] * (n + 1)
    for i in range(1, m + 1):
        curr = [0] * (n + 1)
        for j in range(1, n + 1):
            if X[i-1] == Y[j-1]:
                curr[j] = prev[j-1] + 1
            else:
                curr[j] = max(prev[j], curr[j-1])
        prev = curr
    return prev[n]
  • 복잡도: O(mn) 시간, O(mn) 공간 (기본) → O(n) 공간 (롤링 배열).
  • O(n) 공간 + 경로 복원: 불가능. Hirschberg 1975의 분할 정복 트릭 필요 (섹션 12).

06 — LCS 문자열 복원

전공자

테이블에서 역추적

dp 테이블이 완성되면, 오른쪽 아래에서 시작해 역추적:

  1. X[i-1] == Y[j-1]: LCS에 포함, i/j 둘 다 감소.
  2. dp[i-1][j] >= dp[i][j-1]: i 감소 (위로).
  3. 아니면: j 감소 (왼쪽으로).
lcs_reconstruct.py
def lcs_reconstruct(X, Y):
    m, n = len(X), len(Y)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if X[i-1] == Y[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])

    # 역추적
    result = []
    i, j = m, n
    while i > 0 and j > 0:
        if X[i-1] == Y[j-1]:
            result.append(X[i-1])
            i -= 1; j -= 1
        elif dp[i-1][j] >= dp[i][j-1]:
            i -= 1
        else:
            j -= 1
    return "".join(reversed(result))


print(lcs_reconstruct("ABCBDAB", "BDCABA"))    # "BCBA" 또는 "BDAB"

주의: LCS가 유일하지 않을 수 있음. 역추적 순서(위 vs 왼쪽)에 따라 다른 LCS가 나옴. 모두 유효.

07 — 편집 거리 (Levenshtein 1965)

고급

문제

두 문자열 X, Y. 한쪽을 다른 쪽으로 바꾸는 최소 연산 수. 연산 세 가지:

  • 삽입 (Insert): 글자 하나 삽입.
  • 삭제 (Delete): 글자 하나 삭제.
  • 치환 (Substitute): 글자 하나를 다른 글자로.

예: "kitten" → "sitting" = 3 (k→s 치환, e→i 치환, +g 삽입).

고급

점화식 (Wagner-Fischer 1974)

dp[i][j] = X[0..i-1]를 Y[0..j-1]로 바꾸는 최소 편집 수.

dp[i][0] = i,   dp[0][j] = j

X[i-1] = Y[j-1]이면:   dp[i][j] = dp[i-1][j-1]

아니면: dp[i][j] = 1 + min(
    dp[i-1][j],   // 삭제
    dp[i][j-1],   // 삽입
    dp[i-1][j-1] // 치환
)
  • 기저: 빈 문자열로 바꾸려면 그냥 지우기.
  • 같은 글자면 비용 없이 넘어감.
  • 다르면 세 연산 중 최소.
4단계 편집 거리: "CAT" → "BAT" 는 C→B 치환 한 번 = 거리 1. 매치 시 대각선(비용 0), 불일치 시 세 방향 중 최소 +1. O(mn).

08 — 편집 거리 Python 구현

고급

Wagner-Fischer 구현

edit_distance.py
def edit_distance(X, Y):
    m, n = len(X), len(Y)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    # 기저
    for i in range(m + 1): dp[i][0] = i
    for j in range(n + 1): dp[0][j] = j

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if X[i-1] == Y[j-1]:
                dp[i][j] = dp[i-1][j-1]
            else:
                dp[i][j] = 1 + min(
                    dp[i-1][j],      # 삭제
                    dp[i][j-1],      # 삽입
                    dp[i-1][j-1]     # 치환
                )
    return dp[m][n]


print(edit_distance("kitten", "sitting"))    # 3
print(edit_distance("Saturday", "Sunday"))    # 3
print(edit_distance("", "abc"))    # 3


# 연산 순서 복원
def edit_operations(X, Y):
    m, n = len(X), len(Y)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(m + 1): dp[i][0] = i
    for j in range(n + 1): dp[0][j] = j

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if X[i-1] == Y[j-1]:
                dp[i][j] = dp[i-1][j-1]
            else:
                dp[i][j] = 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1])

    ops = []
    i, j = m, n
    while i > 0 or j > 0:
        if i > 0 and j > 0 and X[i-1] == Y[j-1]:
            i -= 1; j -= 1    # 매치
        elif i > 0 and j > 0 and dp[i][j] == dp[i-1][j-1] + 1:
            ops.append(f"sub {X[i-1]}→{Y[j-1]}"); i -= 1; j -= 1
        elif i > 0 and dp[i][j] == dp[i-1][j] + 1:
            ops.append(f"del {X[i-1]}"); i -= 1
        else:
            ops.append(f"ins {Y[j-1]}"); j -= 1
    return dp[m][n], list(reversed(ops))


print(edit_operations("kitten", "sitting"))
# (3, ['sub k→s', 'sub e→i', 'ins g'])
  • 맞춤법 검사: 사전에서 편집 거리 ≤ 2인 단어 제시. Python의 difflib.get_close_matches.
  • Damerau-Levenshtein: 인접 글자 교환도 1 연산으로 허용 (타자 실수).
  • 가중 편집 거리: 각 연산에 다른 비용 (DNA: transition vs transversion).

09 — LIS — 최장 증가 부분수열 O(n²)

중급

문제

정수 배열에서 엄격히 증가하는 가장 긴 부분수열의 길이.

예: [10, 9, 2, 5, 3, 7, 101, 18]. LIS 중 하나 = [2, 3, 7, 18] 또는 [2, 3, 7, 101]. 길이 4.

고급

DP O(n²) 풀이

dp[i] = nums[i]로 끝나는 LIS의 길이.

dp[i] = 1 + maxj < i, nums[j] < nums[i] dp[j]
lis_n2.py
def lis_n2(nums):
    n = len(nums)
    if n == 0: return 0
    dp = [1] * n
    for i in range(1, n):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    return max(dp)


print(lis_n2([10, 9, 2, 5, 3, 7, 101, 18]))    # 4

O(n²) 시간. n=104까지는 OK, 105부터는 느림. 더 빠른 방법이 있음 — 다음 섹션.

10 — LIS Patience Sort — O(n log n)

전공자

"인내심 카드 게임" 아이디어

카드 게임 Patience에서 영감을 받은 기법. Aldous & Diaconis가 1999년에 소개했지만 수학적 뿌리는 Robinson-Schensted 1961.

  1. 빈 리스트 tails로 시작. tails[k] = "길이 k+1짜리 LIS의 가능한 가장 작은 마지막 원소".
  2. 각 num에 대해:
    • tails에서 num보다 같거나 큰 첫 위치를 이진 탐색(bisect_left).
    • 그 위치에 num 대입 (없으면 append).
  3. 최종 len(tails) = LIS 길이.

핵심: tails는 항상 정렬 상태. 이진 탐색으로 O(log n) 업데이트 → 전체 O(n log n).

lis_nlogn.py
from bisect import bisect_left

def lis_nlogn(nums):
    tails = []
    for num in nums:
        pos = bisect_left(tails, num)
        if pos == len(tails):
            tails.append(num)
        else:
            tails[pos] = num
    return len(tails)


print(lis_nlogn([10, 9, 2, 5, 3, 7, 101, 18]))    # 4
길이만 같음.
8단계 Patience 처리: 각 원소를 tails에 적절한 위치에 삽입/교체. 최종 길이가 LIS 길이. Aldous & Diaconis 1999로 대중화된 기법. 실제 수열 복원엔 parent 추적 추가 필요.

11 — 행렬 체인 곱셈 (CLRS §14.2)

전공자

문제

A1 × A2 × ... × An을 계산. 행렬 곱셈은 결합 법칙이 성립하지만 계산 비용은 괄호 배치에 따라 크게 다름.

예: A(10×100) × B(100×5) × C(5×50)

  • (A·B)·C: (10·100·5) + (10·5·50) = 5000 + 2500 = 7500
  • A·(B·C): (100·5·50) + (10·100·50) = 25000 + 50000 = 75000
  • → 10배 차이!
전공자

DP 점화식

p[i-1] × p[i] 크기 행렬 Ai. m[i][j] = Ai..Aj의 최소 곱셈 수.

m[i][j] = mini ≤ k < j ( m[i][k] + m[k+1][j] + p[i-1]·p[k]·p[j] )
  • m[i][i] = 0 (단일 행렬).
  • 복잡도: O(n³) — 상태 O(n²), 전이 O(n).
  • 실제로는 Hu & Shing 1981가 O(n log n)을 제시했지만 실무엔 O(n³)이 표준.
matrix_chain.py
def matrix_chain(p):
    """p: [p0, p1, ..., pn] (n+1 크기). A_i = p[i-1] × p[i]."""
    n = len(p) - 1
    m = [[0] * (n + 1) for _ in range(n + 1)]

    for length in range(2, n + 1):    # 구간 길이
        for i in range(1, n - length + 2):
            j = i + length - 1
            m[i][j] = float('inf')
            for k in range(i, j):
                cost = m[i][k] + m[k+1][j] + p[i-1] * p[k] * p[j]
                if cost < m[i][j]:
                    m[i][j] = cost
    return m[1][n]

12 — Hirschberg — O(n) 공간 LCS

전공자

1975년의 분할 정복 + DP

Wagner-Fischer의 O(mn) 공간은 큰 문자열(DNA, 텍스트 파일)에 치명적. m=n=106이면 1012 셀 = 수 TB.

Hirschberg의 트릭:

  1. LCS 길이만 필요하면 O(n) 공간은 쉬움 (롤링 배열).
  2. LCS 문자열까지 필요할 때가 문제. Hirschberg는 분할 정복으로 해결.
  3. X를 중간에서 두 개로 나눔: XL, XR. XL과 Y의 LCS를 정방향 + XR과 Y의 LCS를 역방향으로 계산해 각 k에서 LCS(X_L, Y[0..k]) + LCS(X_R, Y[k..n])이 최대가 되는 k를 찾음.
  4. 재귀적으로 각 반쪽에 대해 반복.
  • 시간: O(mn) (기본 DP와 같음).
  • 공간: O(min(m, n)). 혁신.
  • 응용: BLAST 같은 대규모 유전자 정렬 도구의 근간.

13 — 응용 — diff, git, DNA, GPT

중급

Unix diff / git diff

두 파일의 공통 줄을 보여주는 diff는 기본적으로 LCS 문제. 줄 단위로 X, Y로 보고 LCS를 찾은 뒤 없는 부분 = 추가/삭제된 줄.

  • Myers 1986: "An O(ND) Difference Algorithm". D = 편집 횟수. git, GNU diff, svn이 이 알고리즘 기반. Python difflib은 별도로 Ratcliff-Obershelp gestalt pattern matching을 씀 (Ratcliff & Metzener 1988).
  • Hunt-McIlroy 1976: 이전 세대 diff, O(mn + m log m).
고급

유전자 서열 정렬

  • Needleman-Wunsch 1970: 전역 정렬. 편집 거리의 일반화 (치환/삭제/삽입에 가중치 score matrix).
  • Smith-Waterman 1981: 지역 정렬. DP의 변형 — 음수 점수는 0으로 reset해 "가장 유사한 부분 구간" 찾기.
  • BLAST: 휴리스틱으로 빠르게. k-mer seed + 확장.
고급

NLP 평가 지표

  • BLEU (기계번역): n-gram 겹침 기반. LCS 직접 사용 안 하지만 유사 아이디어.
  • ROUGE-L (요약): LCS 기반. "생성 요약과 레퍼런스의 최장 공통 부분수열".
  • Levenshtein ratio: 1 - distance/max(|X|,|Y|). 유사도 0-1.
  • GPT 모델 평가, 자동완성 평가에 실제로 사용.
전공자

AI가 DP 고전을 잘못 쓰는 패턴

  • LCS와 substring 혼동: 연속 여부 다름. Longest Common Substring(부분문자열)은 별개 DP — dp[i][j]가 "반드시 i, j에서 끝나야 함".
  • 0/1 vs 완전 배낭 순회 방향: 역순이 0/1. 정순이면 같은 아이템 무한 사용.
  • LIS에 O(n²) 고집: n=105이면 O(n log n) 필수.
  • 편집 거리 초기화 실수: dp[0][j] = j 빼먹으면 모든 값이 망가짐.
  • 공간 최적화 없이 큰 입력: m=n=105에 O(mn) 공간 = 10 GB.

0/1 배낭에서 W=50, 아이템 (v=60,w=10), (v=100,w=20), (v=120,w=30)의 최적값은?

길이 4. 가능한 LCS: "BCBA", "BDAB", "BCAB". 역추적 순서에 따라 하나가 나옴. LCS 개수 세기는 또 다른 DP 문제.

BUILD 문제 3

편집 거리("horse", "ros") 계산 + 최적 연산 시퀀스.

풀이 펼치기
답

거리 3. 예: horse → rorse (h→r), rorse → rose (delete r), rose → ros (delete e). 또는: horse → hrse (delete o), hrse → rrse → ros 등 여러 최적 경로.

DESUGAR 문제 4

LIS Patience 기법에서 왜 tails 자체는 실제 LIS가 아닐 수 있는가?

풀이 펼치기
답

예: [1, 5, 2, 3]. Patience 진행: tails=[1]→[1,5]→[1,2]→[1,2,3]. 최종 tails=[1,2,3] ← 이것이 실제 LIS (우연히). 하지만 [3, 1, 2]의 경우: tails=[3]→[1]→[1,2]. 결과 tails=[1,2]이지만 실제 LIS는 [1,2] (OK 이경우에도 맞음). 반례: [1, 3, 2, 4]: tails=[1]→[1,3]→[1,2]→[1,2,4]. 이건 실제 LIS와 같음. 더 복잡한 예: [3, 1, 4, 1, 5, 9, 2, 6]. 최종 tails 내용물이 반드시 LIS 원래 순서가 아닐 수 있음. 길이만 맞음, 실제 원소 배치는 별개. 실제 LIS 복원은 parent 배열로.

WHY 문제 5

LCS 길이 L과 편집 거리 D의 관계를 식으로.

풀이 펼치기
답

D ≤ |X| + |Y| − 2L. 직관: LCS L만큼은 "공통"이라 건드릴 필요 없음. 나머지 X의 |X|−L 글자는 삭제, Y의 |Y|−L 글자는 삽입 → indel-only 거리 |X|+|Y|−2L. 표준 Levenshtein(치환 허용)은 이보다 작거나 같음 — 치환 한 번이 "삭제+삽입" 두 연산을 대체할 수 있기 때문. 예: "CAT"→"BAT": L=2, indel-only 거리 = 2, Levenshtein = 1 (C→B 치환). 1 ≤ 2 ✓.

BUILD 문제 6

편집 거리에서 치환 비용이 2라면?

풀이 펼치기
답

치환 = 삭제 + 삽입과 동등해짐. 이 경우 편집 거리 = |X|+|Y|−2L (LCS 기반). 가중 편집 거리는 연산마다 다른 비용을 정의해 일반화한 것.

PREDICT 문제 7

n=105에 LIS O(n²)와 O(n log n) 예상 시간.

풀이 펼치기
답

O(n²) = 1010. CPU 109ops/s면 10초 정도 (제한시간 초과 가능). O(n log n) = 105×17 ≈ 2×106. 0.01초. 경진대회/실무에 O(n log n) 필수.

DESUGAR 문제 8

배낭 공간 최적화에서 정순 순회로 바꾸면 어떤 문제를 푸나?

풀이 펼치기
답

완전 배낭(unbounded knapsack). 같은 아이템을 여러 번 쓰는 문제. 정순 순회 시 dp[w-weights[i]]가 이번 i에 이미 갱신된 값 → 아이템 i가 중복 카운트됨. 예: 동전 교환 변형 — 각 종류 동전 무한 사용 가능.

WHY 문제 9

행렬 체인에서 3개 행렬인데 왜 괄호에 따라 10배 차이?

풀이 펼치기
답

A(10×100) × B(100×5) × C(5×50):

  • (AB)C: AB는 10×5, 곱셈 10·100·5=5000. (AB)C는 10·5·50=2500. 총 7500.
  • A(BC): BC는 100×50, 곱셈 100·5·50=25000. A(BC)는 10·100·50=50000. 총 75000.

작은 중간 행렬을 만들수록 이후 곱셈 비용 감소. 차원 100이 "나쁜 차원"이므로 먼저 흡수하는 게 나음.

WHY 문제 12

편집 거리가 "metric"(거리 공리 만족)인 이유.

풀이 펼치기
답

거리 공리:

  • 비음수성: d(X,Y) ≥ 0. ✓
  • 대칭성: d(X,Y) = d(Y,X). ✓ (삽입↔삭제 대칭).
  • 삼각부등식: d(X,Z) ≤ d(X,Y) + d(Y,Z). ✓ (X→Y→Z 경로가 하나의 옵션).
  • d(X,Y) = 0 ⟺ X = Y. ✓

→ Levenshtein은 정식 metric. k-최근접이웃, 클러스터링에 쓸 수 있음. 반면 가중 편집 거리는 가중치 설정에 따라 대칭성이 깨질 수 있음 (주의).

백트래킹 — N-Queens, 스도쿠, 가지치기

DP가 "모든 경우 기억"이라면, 백트래킹은 "가능성 탐색 + 실패 시 되돌림". 깊이 우선 탐색의 꽃.

  • 스도쿠 — 제약 전파(constraint propagation)와 결합.
  • 부분집합 합 / 순열 생성 — 백트래킹의 교과서 예.
  • 가지치기(pruning) — "살릴 수 없는" 경로 조기 포기. 상한/하한 계산.
  • 이전 주와의 연결 — DP에 Optimal Substructure 없는 경우가 백트래킹의 영역.
