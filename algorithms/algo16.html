01 — 이번 주 이야기: 세 사람의 DP

"The tree of solutions, growing larger and larger, is a thing of wonder and of joy. Its fruit is the solution of the problem."
— Richard Bellman, Dynamic Programming (1957)

1956년 — Lester Ford Jr., RAND Corporation

냉전기 미 공군이 RAND에 의뢰한 문제: "네트워크 흐름(network flow)". Ford는 이를 풀면서 부수적으로 최단 경로 알고리즘의 첫 윤곽을 잡음 (Ford, L.R. 1956, Network Flow Theory, RAND Paper P-923). "경로 완화(relaxation)"라는 표현이 이때 등장.

1958년 — Richard Bellman, Quarterly of Applied Mathematics

Bellman은 막 Dynamic Programming(1957) 책을 쓴 상태. 라우팅 문제를 DP 관점에서 재포장한 논문 "On a Routing Problem"(Q. Appl. Math. 16:87-90, 1958)에서 오늘날 우리가 아는 형태로 알고리즘을 정립. "Bellman 방정식": dk(v) = minu(dk-1(u) + w(u,v)).

왜 DP였나: Bellman은 CS가 아니라 OR(운영 과학)·제어 이론에서 왔다. 그에게 최단 경로는 단지 순차 결정 문제의 특수한 경우일 뿐. 이 시각이 이후 강화 학습(Bellman equation)의 뿌리가 된다.

1962년 — Floyd & Warshall, 각자 독립적으로

같은 해, 같은 저널(CACM)에 두 논문이 연달아 게재:

  • Stephen Warshall (1962): "A Theorem on Boolean Matrices". 이진 행렬의 transitive closure. "도달 가능성"을 묻는 알고리즘.
  • Robert W. Floyd (1962): "Algorithm 97: Shortest Path". Warshall의 Boolean 행렬을 실수 가중치로 확장. 겨우 5줄.
  • Peter Ingerman (1962): 3중 for-loop 형태로 정식화. 오늘날 교과서에서 보는 형태.

그리고 사실은 Bernard Roy(1959)가 먼저 같은 알고리즘을 발표했지만 불어 저널이라 묻혔다 — 그래서 유럽에선 가끔 Roy-Floyd-Warshall로도 불림.

WHY — 세 이름이 왜 하나의 주제로 묶이는가

Bellman-Ford와 Floyd-Warshall은 겉보기엔 다른 문제를 풀지만(SSSP vs APSP), 같은 DP 구조를 씁니다:

  • BF: "최대 i개 간선을 쓰는 최단 경로" → i를 1부터 V-1까지.
  • FW: "정점 {1..k}만 중간에 허용하는 최단 경로" → k를 1부터 V까지.

둘 다 "허용 집합을 키워가며 부분 문제를 저장" — DP의 정의. Dijkstra의 그리디와 대조적.

02 — Dijkstra의 한계 재확인

중급

Algo 15 복습 — 음수 간선이 깨뜨리는 것

Dijkstra의 불변식: "pop된 노드의 dist는 확정". 음수 간선이 있으면 이 보장이 깨짐 — 미래의 음수 간선이 과거 확정 값을 뒤집을 수 있기 때문.

예: A→B: 4,   A→C: 2,   B→C: −3
교과서 Dijkstra: dist[C] = 2 (A→C 직행).
실제 최단: A→B→C = 4 + (−3) = 1. ❌
고급

음수 간선은 왜 현실에서 등장하는가

  • 환율 차익거래: USD→KRW 환율의 −log를 가중치로 두면, 음의 사이클 = 차익. Floyd-Warshall로 감지.
  • 물리 시뮬레이션: 에너지 감소 경로.
  • 네트워크 비용: 일부 간선에 인센티브(음수 비용).
  • DP 변형: 게임 상태 그래프에서 "보상" 간선.
감지 ✓ 단순 (DP, 이중 loop) ✗ O(VE) — 느림 사용: RIP, 환율 차익, DP 교재
선택 기준 — 음수 간선이 없고 빠른 쿼리가 필요 → Dijkstra. 음수 간선이 있거나 사이클 감지가 필요 → Bellman-Ford.

03 — Bellman-Ford 아이디어: V-1번 relax

중급

DP 관점 — "i개 간선으로 도달 가능한 최단 거리"

핵심 정의: di(v) = "시작점에서 v까지, 간선을 최대 i개 써서 도달하는 최단 거리".

점화식:

di(v) = min( di-1(v),   min(u→v)∈E [ di-1(u) + w(u,v) ] )
  • 초기: d0(s)=0, 나머지는 ∞.
  • 종료: i = V-1 (임의의 simple path는 최대 V-1개 간선).
  • 구현 관점: i를 증가시키며 모든 간선을 한 번씩 relax. 총 V-1번의 "라운드".
고급

Dijkstra와의 본질적 차이

  • Dijkstra: "매 반복 최적 노드 확정"(greedy). 노드 중심.
  • Bellman-Ford: "매 반복 모든 간선 한 번씩 완화". 간선 중심. 노드는 여러 번 갱신될 수 있음.

BF는 "확정" 개념이 없기에 음수 간선에도 안전. 대신 모든 간선을 V-1번 훑어야 해서 O(VE).

Bellman-Ford = V−1번의 "전체 방송". 매 라운드 모든 간선을 한 번씩 relax. 간선 수가 k개일 때 i번째 라운드엔 "최대 i개 간선 경로"가 확정.

04 — BF 의사코드 (CLRS §22.1)

고급

CLRS 표준 의사코드

BELLMAN-FORD(G, w, s)
1. INITIALIZE-SINGLE-SOURCE(G, s)
2. for i = 1 to |G.V| − 1
3.     for each edge (u, v) ∈ G.E
4.        RELAX(u, v, w)
5. for each edge (u, v) ∈ G.E // 음수 사이클 체크
6.     if v.d > u.d + w(u, v)
7.        return FALSE
8. return TRUE
  • 라인 2-4: V−1 라운드. 매 라운드 모든 간선 relax.
  • 라인 5-7: 한 번 더 돌아서 갱신이 일어나면 음수 사이클 존재. 최단 경로 없음.
  • 반환: 음수 사이클 없으면 TRUE + dist 배열; 있으면 FALSE.
📖 원서 인용 — CLRS 4판, §22.1 "The Bellman-Ford algorithm"

"The Bellman-Ford algorithm solves the single-source shortest-paths problem in the general case in which edge weights may be negative. ... The algorithm returns a boolean value indicating whether or not there is a negative-weight cycle that is reachable from the source."

전공자

조기 종료 최적화

라운드 한 번에 아무 갱신도 없으면 이후 라운드도 무변화 (RELAX의 멱등성). updated 플래그로 확인하고 즉시 break 가능. 평균적으로 훨씬 빠름 (특히 층화된 그래프).

05 — BF Python 구현 + 시각화

전공자

간선 리스트 기반 구현

bellman_ford.py
def bellman_ford(V, edges, src):
    """
    V: 정점 수
    edges: [(u, v, w), ...] — 간선 리스트
    src: 시작 정점
    반환: (dist, parent, ok)  ok=False이면 음수 사이클
    """
    INF = float('inf')
    dist = [INF] * V
    parent = [None] * V
    dist[src] = 0

    # V-1 라운드
    for i in range(V - 1):
        updated = False
        for u, v, w in edges:
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                parent[v] = u
                updated = True
        if not updated:
            break    # 조기 종료

    # 음수 사이클 감지 (한 번 더 relax 시도)
    for u, v, w in edges:
        if dist[u] + w < dist[v]:
            return dist, parent, False

    return dist, parent, True


# 테스트 — 간선 순서를 "멀리→가까이"로 배치해서 V-1 라운드 필요성 시연
edges = [
    (1, 3, 2),     # A → C: 2   (먼저 시도 — dist[A]=∞라 무효)
    (2, 3, -4),    # B → C: -4  (먼저 시도 — dist[B]=∞라 무효)
    (0, 1, 3),     # s → A: 3   (라운드 1에서 dist[A] = 3)
    (0, 2, 5),     # s → B: 5   (라운드 1에서 dist[B] = 5)
]
dist, parent, ok = bellman_ford(4, edges, 0)
print(ok, dist)
# True [0, 3, 5, 1]  (라운드 1: [0,3,5,∞], 라운드 2: [0,3,5,1])
# 간선 순서가 반대였다면 라운드 1에 완성 — BF 최악(V-1) vs 최선(1) 라운드 차이.
  • 간선 리스트: 인접 리스트가 아닌 (u, v, w) 튜플 리스트가 자연스러움 (모든 간선 순회).
  • INF 덧셈 주의: INF + (-10)는 파이썬에서 여전히 inf. 다른 언어에선 오버플로우 가능.
  • 조기 종료: updated 플래그로 라운드 한 번에 변화 없으면 break.
5단계 애니메이션: 초기화 → 라운드 1 → 라운드 2 → 라운드 3 (무변화) → 완료. BF는 "최대 i개 간선 경로"를 라운드 i에 완성.

06 — 정확성 증명: 왜 V-1번인가

고급

핵심 보조정리 (Path-Relaxation Property)

보조정리: 시작점 s에서 노드 v까지의 최단 경로가 s = v0 → v1 → … → vk = v일 때, 만약 간선 (v0,v1), (v1,v2), …, (vk-1,vk)가 이 순서대로 relax되면, relax 종료 후 dist[v] = δ(s,v).

증명 스케치 (귀납법):

  1. Base: dist[s] = 0 = δ(s,s). ✓
  2. Hypothesis: dist[vi-1] = δ(s, vi-1).
  3. 간선 (vi-1, vi) relax 시: dist[vi] ≤ dist[vi-1] + w = δ(s,vi-1) + w(vi-1,vi) = δ(s,vi) (최단 경로의 부분 경로 또한 최단 — subpath optimality).
  4. 항상 dist[vi] ≥ δ(s,vi) (관찰에 의해). 따라서 등호. ✓
전공자

V-1번으로 충분한 이유

음수 사이클이 없다면 임의의 simple path는 최대 V-1개 간선을 가짐 (V개 정점을 반복 없이 거침). BF는 매 라운드 모든 간선을 relax하므로, 최단 경로의 k개 간선이 "순서대로" relax되려면 k 라운드면 충분. k ≤ V-1.

⚠️ "순서"의 의미

한 라운드에 간선들이 어떤 순서로 relax되든 상관없다. 왜냐면 라운드 i가 끝난 시점에는 "i개 이하 간선으로 도달 가능한 최단 거리"가 모두 이미 dist에 반영되어 있기 때문. 증명의 "순서대로"는 라운드 간의 순서이지 라운드 내 순서가 아님.

📖 CLRS Theorem 22.4 (4판) — Bellman-Ford 정당성

"After |V|−1 passes of relaxation, d[v] = δ(s,v) for all vertices v reachable from s, provided the graph contains no negative-weight cycles reachable from s."

07 — 음수 사이클 감지

고급

왜 V번째 라운드가 필요한가

음수 사이클이 s로부터 도달 가능하면, 그 사이클을 돌 때마다 경로 비용이 감소 → 최단 경로 값이 −∞. 정의 불가.

감지 방법: V-1 라운드 후에 한 번 더 relax 시도. 추가로 갱신되는 간선이 있으면 음수 사이클 존재. 왜? V-1 라운드로 모든 simple path는 수렴했어야 하는데 더 감소한다면 반드시 사이클을 돈 것.

5프레임 애니메이션: 초기화 → R1 → R2(V−1 완료) → R3(또 감소 ⚠️) → 사이클 확정. 핵심 관찰 — V−1 라운드 이후 추가 갱신 = 사이클.
전공자

사이클 복원

V번째 라운드에서 갱신된 노드 x를 하나 잡고, parent를 V번 따라가면 반드시 사이클 위의 노드에 도달. 거기서부터 parent 따라 돌면서 사이클 노드 집합 수집.

find_negative_cycle.py
def find_negative_cycle(V, edges, src):
    INF = float('inf')
    dist = [INF] * V; dist[src] = 0
    parent = [None] * V

    x = -1    # 마지막 갱신된 노드 (V번째 라운드에서 감지용)
    for i in range(V):
        x = -1
        for u, v, w in edges:
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                parent[v] = u
                x = v

    if x == -1:
        return None    # 음수 사이클 없음

    # 사이클 진입점 찾기: V번 parent 따라가기
    y = x
    for _ in range(V):
        y = parent[y]

    # y부터 사이클 복원
    cycle = []
    cur = y
    while True:
        cycle.append(cur)
        cur = parent[cur]
        if cur == y and len(cycle) > 1:
            break
    return cycle[::-1]

이 트릭은 환율 차익거래에서 바로 쓰인다: 환율 r(u,v)를 −log r(u,v)로 치환하면, 음수 사이클 = 차익 거래 기회.

08 — SPFA (Shortest Path Faster Algorithm)

전공자

Moore 1959의 변형 — "큐 기반 BF"

동기: BF는 매 라운드 모든 간선을 검사하지만, 현실에선 "갱신된 노드의 이웃"만 재검토하면 충분. 큐로 이걸 구현.

아이디어:

  1. 갱신된 노드 u를 큐에 넣기.
  2. 큐에서 u를 꺼내 u의 이웃을 relax.
  3. 갱신된 이웃이 큐에 없으면 넣기.
  4. 큐가 비면 종료.
spfa.py
from collections import deque

def spfa(V, graph, src):
    """graph: {u: [(v, w), ...]} 형태의 인접 리스트."""
    INF = float('inf')
    dist = [INF] * V; dist[src] = 0
    in_queue = [False] * V
    count = [0] * V    # 큐 삽입 횟수 (음수 사이클 감지용)
    q = deque([src]); in_queue[src] = True

    while q:
        u = q.popleft(); in_queue[u] = False
        for v, w in graph.get(u, []):
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                if not in_queue[v]:
                    q.append(v); in_queue[v] = True
                    count[v] += 1
                    if count[v] >= V:
                        return None    # 음수 사이클!
    return dist

복잡도: 평균 O(kE) (k는 작은 상수, 실측 2-3). 최악은 여전히 O(VE).

⚠️ SPFA는 "빠를 수 있는" 알고리즘

중국 경쟁 프로그래밍(competitive programming)에서 유행. 하지만 최악 그래프 설계로 O(VE)가 되는 경우(예: lattice + 음수 간선)가 경진대회에서 "SPFA 죽이기"로 유명. "SPFA 죽었다(SPFA已死)"는 농담도 있음. 실무에선 음수 간선 없으면 Dijkstra, 있으면 BF가 더 안전.

09 — Floyd-Warshall: DP on 중간 정점 (CLRS §23.2)

중급

All-Pairs Shortest Paths (APSP)

지금까지는 한 출발점에서 최단 경로. Floyd-Warshall은 모든 쌍의 최단 경로를 한 번에.

언제 쓰는가: 작은 그래프 (V ≤ 수백), 많은 쿼리. 행렬 형태로 결과 저장.

고급

DP 정의 — "중간 정점 집합"

정점 번호를 1..V로 매김. d(k)(i,j) = "i에서 j까지, 중간 정점이 {1..k}만 허용될 때의 최단 거리".

d(k)(i,j) = min( d(k-1)(i,j),   d(k-1)(i,k) + d(k-1)(k,j) )
  • 초기 d(0)(i,j) = w(i,j) (간선 직접) 또는 ∞.
  • 해: d(V)(i,j) = 전 정점 허용.
  • 의미: "k를 중간에 끼우면 더 짧아지는가" — 매 k마다 모든 (i,j) 점검.

이 점화식의 우아함: k 차원 배열이 필요할 것 같지만, 실제로는 2D 배열 하나를 in-place 업데이트하면 됨 (다음 섹션에서 증명).

📖 원서 인용 — CLRS 4판, §23.2 "The Floyd-Warshall algorithm"

"The Floyd-Warshall algorithm considers the 'intermediate' vertices of a shortest path, where an intermediate vertex of a simple path p = ⟨v1, v2, …, vℓ⟩ is any vertex of p other than v1 or vℓ."

Floyd-Warshall의 DP 축: k번째 단계는 "{1..k} 경유 최단 거리"로 완성. k=V가 되면 전 정점 허용 = 진짜 최단 거리.

10 — FW Python 구현 (5줄)

고급

In-place 업데이트의 정당성

점화식이 d(k)(i,j) = min(d(k-1)(i,j), d(k-1)(i,k) + d(k-1)(k,j))인데, 2D 배열을 쓰면 d[i][k]와 d[k][j]가 이미 이번 k에서 갱신됐을 수 있음.

핵심 관찰: d[i][k]는 이번 k에 갱신되어도 값이 바뀌지 않는다. 왜? d(k)(i,k) = min(d(k-1)(i,k), d(k-1)(i,k) + d(k-1)(k,k))이고 d(k-1)(k,k) ≥ 0 (음수 사이클 없다는 전제) → 두 번째 항이 더 크거나 같아 min이 변하지 않음. 같은 논리로 d[k][j]도 불변.

따라서 단일 2D 배열 in-place 업데이트 OK. 이게 3중 루프 5줄의 마법이 가능한 이유.

전공자

전설의 5줄 코드

floyd_warshall.py
def floyd_warshall(V, edges):
    """edges: [(u, v, w), ...]. 반환: V×V 거리 행렬."""
    INF = float('inf')
    d = [[INF] * V for _ in range(V)]
    for i in range(V): d[i][i] = 0
    for u, v, w in edges: d[u][v] = min(d[u][v], w)

    # --- 핵심 3중 루프 (Floyd 1962, Algorithm 97) ---
    for k in range(V):
        for i in range(V):
            for j in range(V):
                if d[i][k] + d[k][j] < d[i][j]:
                    d[i][j] = d[i][k] + d[k][j]
    return d


# 경로 복원 버전
def floyd_warshall_path(V, edges):
    INF = float('inf')
    d = [[INF]*V for _ in range(V)]
    nxt = [[None]*V for _ in range(V)]
    for i in range(V): d[i][i] = 0
    for u, v, w in edges:
        if w < d[u][v]:
            d[u][v] = w; nxt[u][v] = v

    for k in range(V):
        for i in range(V):
            for j in range(V):
                if d[i][k] + d[k][j] < d[i][j]:
                    d[i][j] = d[i][k] + d[k][j]
                    nxt[i][j] = nxt[i][k]
    return d, nxt

def reconstruct(nxt, u, v):
    if nxt[u][v] is None: return None
    path = [u]
    while u != v:
        u = nxt[u][v]; path.append(u)
    return path
  • 복잡도: O(V³) 시간, O(V²) 공간. V=500까지는 파이썬에서도 ~수 초.
  • 음수 사이클 감지: 종료 후 d[i][i] < 0인 i가 있으면 i 포함 음수 사이클 존재.
  • numpy 가속: d = np.minimum(d, d[:,k:k+1] + d[k:k+1,:]) 로 한 줄 브로드캐스트.
📖 Floyd 1962 원문 — CACM 5(6):345 "Algorithm 97: Shortest Path"

"The following algorithm computes the shortest path between every pair of nodes of a directed graph, assuming that no cycles of negative weight exist."

Floyd의 원 논문은 단 반 페이지. ALGOL로 작성된 5줄짜리 프로시저. Floyd는 이 업적을 포함한 알고리즘 연구로 1978년 Turing Award를 수상.

11 — Johnson 알고리즘 (sparse APSP)

전공자

Johnson 1977 — BF + Dijkstra의 조합

문제: FW는 O(V³). sparse 그래프(E = O(V))에서는 낭비. V개 시작점마다 Dijkstra 돌리면 O(VE log V)가 더 빠르지만, 음수 간선이 있으면 Dijkstra 못 씀.

Johnson의 트릭: 음수 간선을 "재가중(reweighting)"으로 제거. 각 정점에 potential h(v)를 부여해 새 가중치 w'(u,v) = w(u,v) + h(u) − h(v) ≥ 0이 되도록. 그 다음 V번 Dijkstra.

  1. 가상 정점 q 추가, q→모든 정점에 비용 0 간선.
  2. BF로 q부터의 거리 계산 → h(v)로 사용.
  3. 모든 간선 가중치를 w'(u,v) = w(u,v) + h(u) − h(v)로 치환. 최단경로 부등식 (CLRS Lemma 22.10, "triangle inequality for shortest paths") h(v) ≤ h(u) + w(u,v)로부터 w'(u,v) ≥ 0이 성립.
  4. 각 시작점마다 Dijkstra.
  5. 결과 거리 δ'(s,t)를 원래로 복원: δ(s,t) = δ'(s,t) − h(s) + h(t).
  • 복잡도: O(VE + V² log V). sparse에서 O(V³)의 FW보다 빠름.
  • 전제: 음수 사이클 없음 (BF가 감지).
💡 potential의 의미

h(v)는 물리의 "포텐셜 에너지"와 같은 역할. 간선 가중치에 "높이 차"를 더하면 경로 총비용은 경로의 시작-끝 높이 차만큼 상수 offset 됨 → 최단 경로 순서는 보존. 이 아이디어는 linear programming duality에서 유래.

13 — 실무: RIP, 환율 차익거래, AI

중급

RIP (Routing Information Protocol)

분산 Bellman-Ford. 1988년 RFC 1058에 표준화. 각 라우터가 주기적으로 "내 거리 테이블" 방송 → 이웃이 받아 자신의 테이블 갱신 → 반복.

  • 라우터 각자가 로컬 BF를 수행하고 결과를 공유 — distributed BF.
  • hop count ≤ 15로 제한 → 네트워크 지름 제한 (BF의 V-1 수렴 보장 차용).
  • 1990년대 OSPF (Dijkstra 기반, link-state)로 대체되며 쇠퇴. 지금도 소규모 LAN엔 사용.
고급

환율 차익거래 감지

통화 그래프를 구성: 정점 = 통화, 간선 = 환전 비율. 간선 가중치 w(u,v) = −log(rate(u,v)).

통화 시퀀스 c1 → c2 → … → c1의 총 환전 비율
= r(c1,c2) · r(c2,c3) · … · r(cn,c1)
= exp(−∑ w)    (w = −log r)

차익 = 비율 > 1   ⟺   ∑w < 0   ⟺   음수 사이클

알고리즘: BF로 음수 사이클 감지 후 복원. 실시간 FX 시스템에서 사이클 패턴 감지에 응용.

4프레임 차익거래 시뮬레이션: USD → KRW → JPY → USD로 한 바퀴 돌면 4% 차익. 환율을 −log로 변환하면 합이 음수 → 음수 사이클 ≡ 차익 기회. Bellman-Ford로 실시간 감지 가능.
전공자

AI: Bellman 방정식과 강화학습

RL의 핵심 식(Bellman 방정식):

V*(s) = maxa [ R(s,a) + γ · ∑s' P(s'|s,a) V*(s') ]

이게 Bellman-Ford의 확률적 일반화. 그래프 대신 MDP, 간선 대신 (s,a,s') 전이, 가중치 대신 기대 보상. Value Iteration은 BF의 MDP 버전.

  • DeepMind DQN, AlphaGo의 뿌리.
  • GPT의 RLHF도 변종 Bellman 업데이트.
  • Bellman 본인은 1950년대 OR 연구자. CS 출신 아님.
전공자

AI가 BF/FW를 잘못 쓰는 패턴

  • 음수 사이클 미점검: "음수 간선 있는데 Bellman-Ford로 최단경로 계산" 뒤 사이클 체크 생략 → 무한 루프 또는 −∞ 결과.
  • FW에 거대 그래프: V=10만이면 1015번 연산. AI는 가끔 "FW는 O(V³)이니까 괜찮다"는 식으로 제안.
  • in-place 업데이트 순서 오해: k-loop이 가장 바깥에 와야 하는 이유 (불변식)를 모르고 i/j/k 순서 뒤섞음.
  • SPFA 신봉: "SPFA가 BF보다 빠르다"고 단정. 최악 O(VE) 여전.

교정 질문: "음수 사이클 있으면 결과가 정의되나?", "V³이 실행 가능한가 (V 크기는)?", "k를 바깥 loop에 둔 이유는?"

14 — 연습문제 (12문제)

WHY 문제 1

Bellman-Ford가 V-1 라운드가 아니라 V 라운드를 돌리면 어떻게 되나? 더 정확해지나?

풀이 펼치기
답

음수 사이클이 없으면 V-1 라운드 이후로는 어떤 라운드도 갱신 없음. V번째 라운드는 낭비 또는 음수 사이클 감지용. 더 정확해지지 않음. CLRS가 V-1로 규정한 이유 — "simple path 최대 간선 수 = V-1".

PREDICT 문제 2

V=1000, E=5000인 그래프에 BF와 FW를 돌렸을 때 예상 시간? (CPU 10⁸ ops/sec 가정)

풀이 펼치기
답

BF: O(VE) = 1000 × 5000 = 5·10⁶ ops ≈ 0.05초. FW: O(V³) = 10⁹ ops ≈ 10초. 희소 그래프엔 BF가 압도적. 대신 BF는 SSSP, FW는 APSP — 용도가 다름. V번 BF = O(V²E) = 5·10⁹ — FW보다 더 느림. 희소 APSP엔 Johnson이 맞는 답.

BUILD 문제 3

BF를 이용해 그래프의 가장 "싼" 경로(노드 리스트)를 출력. parent 배열 활용.

풀이 펼치기
힌트

Relax 시 parent[v] = u 기록. 종료 후 목표 t에서 parent 따라가며 리스트 작성, 마지막에 reverse. Algo 15와 같은 패턴.

DESUGAR 문제 4

FW의 in-place 업데이트가 왜 올바른지 설명하라. 특히 "d[i][k]가 이번 k 라운드에 갱신되면 후속 계산에 영향 없나?"

풀이 펼치기
답

d(k)(i,k) = min(d(k-1)(i,k), d(k-1)(i,k) + d(k-1)(k,k)). 음수 사이클 없으면 d(k-1)(k,k) ≥ 0이라 두 번째 항이 첫째 항보다 크거나 같음 → min 동일. 따라서 d[i][k]는 이번 k에 변하지 않음. 마찬가지로 d[k][j]도 불변. 그래서 2D 배열을 in-place 업데이트해도 점화식 의미 보존.

PREDICT 문제 5

FW의 3중 루프 순서를 i, j, k 순서로 바꾸면? 결과가 달라지나?

풀이 펼치기
답

결과가 틀림. k가 가장 바깥에 와야 하는 이유 — DP 축이 k ("중간 정점 집합 확장"). k가 안쪽이면 d[i][k], d[k][j]가 아직 갱신되지 않아 옛 k-1 값 참조 실패. 점화식의 의미가 깨짐. 이 "루프 순서 버그"는 FW에서 가장 흔한 실수.

BUILD 문제 6

FW를 써서 그래프의 직경(diameter) (두 정점 사이 최장 최단거리) 구하기.

풀이 펼치기
답

FW 후 max(d[i][j]). ∞ 제외. 6-degree-of-separation 실험(Milgram 1967) 같은 거 시뮬레이션에 표준.

WHY 문제 7

음수 사이클 감지를 위해 BF의 V번째 라운드만 돌리면 되나, 전체 V라운드 돌려야 하나?

풀이 펼치기
답

V번째 라운드만 추가로 돌리면 됨 (V-1 라운드 기반 dist 위에). 추가 라운드에 갱신되는 간선이 있으면 음수 사이클 존재. 왜? V-1 라운드로 모든 simple path 수렴 완료 → 더 감소 = 사이클을 돈 것.

BUILD 문제 8

3통화 환율에 차익 기회 감지: USD→KRW=1300, KRW→JPY=0.1, JPY→USD=0.008.

풀이 펼치기
답

비율 곱: 1300 × 0.1 × 0.008 = 1.04 > 1 → 차익 있음! 그래프로: w = −log(rate). 간선 가중치 합 = −log(1.04) ≈ −0.039 < 0. 음수 사이클. BF로 감지 가능. (실제 FX는 스프레드/수수료로 실제 차익은 더 작음.)

PREDICT 문제 9

SPFA의 최악 시나리오는?

풀이 펼치기
답

lattice 그래프 + 음수 간선 패턴. 각 노드가 큐에 V번 들어가게 조작 가능 → O(VE). "SPFA killer" 테스트케이스로 알려짐. 경진대회에선 stream-test 패턴, 격자 그래프 → SPFA 대신 BF나 Dijkstra 택해야.

DESUGAR 문제 10

Johnson 알고리즘의 "재가중(reweighting)"이 왜 최단 경로 순서를 보존하나?

풀이 펼치기
답

경로 s→v1→…→vk=t의 새 총비용:
∑ w'(vi-1,vi) = ∑ [w(vi-1,vi) + h(vi-1) − h(vi)]
= ∑ w + h(s) − h(t).
즉 경로 중간의 h는 전부 소거(telescoping)되고 오직 시작-끝만 남음. 따라서 경로 순서는 그대로, 최단 경로가 최단 경로로 유지.

풀이 펼치기
질문

WHY 문제 12

Bellman 방정식이 강화학습의 핵심인 이유 — BF와의 연결.

풀이 펼치기
답

RL에선 상태 s, 행동 a, 다음 상태 s'를 그래프의 노드·간선으로 볼 수 있음. Value Iteration = BF의 확률적 버전. "i번 반복 후 i-step 예측 최적 가치 확정" 구조 동일. Bellman이 1950년대 OR 연구 중 발견한 점화식이 현대 DeepRL (DQN, AlphaGo)의 뿌리인 이유.

반복 정점 없는 경로. V개 정점 그래프에선 최대 V-1 간선.
Negative Cycle
총 가중치 < 0인 사이클. 닿을 수 있으면 최단 경로 −∞ = 정의 불가.
Moore 1959
BF의 큐 기반 변형. SPFA의 원조.
SPFA
Shortest Path Faster Algorithm. 큐 기반 BF. 평균 빠름, 최악 O(VE).
Floyd-Warshall
1962 Floyd + Warshall. APSP. 3중 for-loop, O(V³).
Robert W. Floyd
1936-2001. 1978 Turing Award. "Algorithm 97" (CACM 1962).
Stephen Warshall
1935-2006. Transitive closure 알고리즘 (1962). Floyd와 독립 발견.
Bernard Roy
1934-2017. 1959 동일 알고리즘을 불어 저널에 먼저 발표. 유럽에선 Roy-Floyd-Warshall로도.
APSP
All-Pairs Shortest Paths. 모든 정점 쌍 간 최단 거리.
SSSP
Single-Source Shortest Paths. 한 출발점에서 전 정점까지.
Transitive Closure
"i에서 j로 도달 가능한가" 행렬. Warshall의 원 문제.
Johnson 알고리즘
1977 Johnson. BF + V·Dijkstra. sparse APSP. O(VE + V² log V).
Reweighting
potential h(v)로 간선 가중치 재정의. 음수 제거. Johnson 핵심.
Potential (h-function)
노드 v의 "높이". w'(u,v) = w + h(u) − h(v) ≥ 0 만들기용.
Telescoping Sum
∑(h(vi-1) − h(vi)) = h(s) − h(t). Johnson의 경로 순서 보존 근거.
RIP
Routing Information Protocol (RFC 1058). Distributed BF. hop ≤ 15.
Bellman Equation
V(s) = maxa[R + γ·E[V(s')]]. RL의 핵심. BF의 확률적 일반화.
Value Iteration
Bellman 방정식을 반복 적용해 최적 가치 함수 찾기. BF의 MDP 버전.
Arbitrage Detection
환율 그래프의 음수 사이클 = 차익. BF로 감지. 실시간 FX 시스템.

최소 신장 트리 (MST) — Prim, Kruskal

"그래프 전체를 가장 싸게 연결하는 방법" — 최단 경로와 다른 또 하나의 그리디.

  • Kruskal (1956) — 간선 정렬 + Union-Find. O(E log E).
  • Prim (1957, Jarník 1930) — 정점에서 시작, heap으로 확장. O((V+E) log V).
  • Borůvka (1926) — 병렬 MST의 원조, 체코 전력망 설계.
  • Cut Property — 모든 MST 알고리즘의 정당성 기반.
  • 응용 — 네트워크 설계, 클러스터링, 이미지 segmentation.
